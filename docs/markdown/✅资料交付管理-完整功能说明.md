## 🎯 使用流程

### 第一步：配置项目交付要求

1. **进入配置页面**
   - 侧边栏 → "资料交付管理" → "交付配置"

2. **添加配置**
   - 点击"添加配置"按钮
   - 选择项目
   - 选择交付周期：
     - **按月交付**：每月生成一条记录
     - **按季度交付**：每季度生成一条记录
   - 选择交付方式：
     - **快递交付**：需上传快递单号
     - **电子推送**：需上传电子文件
   - 添加资料清单（可选）
   - 保存

### 第二步：系统自动生成记录

**定时任务自动执行：**

#### 每月1日早上8:00
- 系统自动为"按月交付"的项目生成本月交付记录
- 系统检查"按季度交付"的项目，满3个月则生成新记录
- 自动发送提醒通知给经办

#### 每月最后一天晚上20:00
- 检查本月待交付记录
- 向经办发送未交付提醒

### 第三步：经办提交交付

1. **查看待办**
   - 侧边栏 → "资料交付管理" → "交付记录"
   - 筛选状态："待交付"

2. **提交交付**
   - 点击记录的"提交交付"按钮

**快递方式：**
   - 填写快递单号
   - 选择寄出日期
   - 填写资料说明
   - 提交

**电子方式：**
   - 上传电子资料文件（可多个）
   - 填写资料说明（可选）
   - 提交

3. **标记完成**
   - 已提交的记录可以"标记完成"
   - 完成后状态变为"已完成"

### 第四步：查看历史记录

- 在"交付记录"页面可以查看所有历史
- 支持按项目、期间、状态筛选
- 点击"详情"查看完整信息

## 🗂️ 数据库表结构

### 1. project_delivery_configs（项目交付配置）
```sql
id                    主键ID
account_set_id        账套ID
project_id            项目ID（唯一）
delivery_cycle        交付周期（monthly/quarterly）
delivery_method       交付方式（express/electronic）
required_documents    资料清单（JSON数组）
is_active             是否启用
created_by/updated_by 创建人/更新人
created_at/updated_at 时间戳
```

### 2. document_deliveries（资料交付记录）
```sql
id                    主键ID
account_set_id        账套ID
project_id            项目ID
delivery_cycle        交付周期
delivery_method       交付方式
delivery_period       交付期间（YYYY-MM 或 YYYY-Q1）
status                状态（pending/submitted/completed）
required_documents    应交付资料（JSON）
submitted_documents   资料说明
express_number        快递单号
express_date          寄出日期
submitted_by          提交人
submitted_at          提交时间
completed_by          完成人
completed_at          完成时间
remarks               备注
created_at/updated_at 时间戳
```

### 3. document_delivery_attachments（附件）
```sql
id            主键ID
delivery_id   交付记录ID
filename      文件名
file_path     文件路径
file_size     文件大小
mime_type     文件类型
uploaded_by   上传人
created_at    上传时间
```

### 4. document_delivery_reminders（提醒记录）
```sql
id              主键ID
account_set_id  账套ID
delivery_id     交付记录ID
reminder_type   提醒类型（new_period/not_submitted）
recipient_id    接收人
is_read         是否已读
created_at      提醒时间
```

## 🔧 定时任务说明

### 任务1：生成交付记录
**命令**: `php artisan delivery:generate`  
**调度**: 每月1日 08:00  
**功能**:
- 为所有"按月交付"的项目生成当月记录
- 为满足条件的"按季度交付"项目生成季度记录
- 发送新周期提醒

**手动执行测试**:
```bash
php artisan delivery:generate
# 或指定类型
php artisan delivery:generate --type=monthly
php artisan delivery:generate --type=quarterly
```

### 任务2：检查未交付
**命令**: `php artisan delivery:check-pending`  
**调度**: 每月最后一天 20:00  
**功能**:
- 检查所有待交付记录
- 向经办发送提醒
- 记录提醒历史

**手动执行测试**:
```bash
php artisan delivery:check-pending
```

## ⚠️ 注意事项

### 1. 项目配置
- 每个项目只能配置一次
- 编辑时无法更改项目
- 有交付记录的配置建议禁用而不是删除

### 2. 交付周期
- **按月**：每月1日生成记录
- **按季度**：每3个月生成一次记录

### 3. 交付方式
- **快递**：必须填写快递单号和寄出日期
- **电子**：必须上传至少一个附件文件

### 4. 文件上传
- 单个文件最大50MB
- 支持所有常见格式
- 文件保存在 `public/document_deliveries/` 目录

### 5. 定时任务
- 需要配置系统cron任务
- 每分钟执行一次 `php artisan schedule:run`
- 可以手动执行命令测试
