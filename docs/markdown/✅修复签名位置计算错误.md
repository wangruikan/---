# ✅ 修复签名位置计算错误

## 问题描述

签名出现在PDF顶部，而不是底部的"本人签名："位置。

## 根本原因

**pdf-lib的坐标系统：**
- 原点 (0, 0) 在**左下角**
- X轴向右为正
- Y轴向上为正

**错误的计算：**
```javascript
// ❌ 错误
const sigY = position.from_bottom ? height - position.y : position.y
// 当position.y=60时，计算结果：842 - 60 = 782
// 这是从底部向上782点，接近顶部！
```

**正确的计算：**
```javascript
// ✅ 正确
const sigY = position.from_bottom ? position.y : (height - position.y)
// 当position.y=140时，结果就是140
// 这是从底部向上140点，在底部区域！
```

## 坐标系统说明

### pdf-lib坐标系

```
Y轴 ↑
    |
800 | ──────────────────  ← 顶部（距底部800点）
    |
    |  PDF 内容区域
    |
140 | 本人签名：[签名]   ← 签名位置（距底部140点）
    |
  0 | ──────────────────  ← 底部（原点）
    └────────────────→ X轴
    0                  595
```

### 位置计算

**from_bottom = true（从底部计算）：**
- `y = 140` → 距底部140点
- 直接使用：`sigY = 140`

**from_bottom = false（从顶部计算）：**
- `y = 100` → 距顶部100点
- 需要转换：`sigY = height - 100`

### 2. 签名位置参数调整 ✅

**文件：** `app/Http/Controllers/EmployeeController.php`

**新参数：**
```php
$signaturePosition = [
    'x' => 360,           // X坐标（在"本人签名："右侧）

    'from_bottom' => true // 从底部计算
];
```

**位置说明：**
- X=360：大约在"本人签名："文字右侧
- Y=140：距离页面底部140点
- 尺寸：100×40像素

## 坐标系对比

### 不同坐标系统

| 坐标系 | 原点位置 | Y轴方向 | 使用场景 |
|--------|---------|---------|---------|
| pdf-lib | 左下角 | 向上为正 | PDF生成、签名合成 |
| Canvas | 左上角 | 向下为正 | 网页绘图 |
| CSS | 左上角 | 向下为正 | 网页布局 |
| mpdf | 左上角 | 向下为正 | 后端PDF生成 |

### 转换公式

**从顶部坐标转换为pdf-lib坐标：**
```javascript
pdfLibY = pageHeight - topY
```

**从底部坐标转换为pdf-lib坐标：**
```javascript
pdfLibY = bottomY  // 直接使用
```

## 常见问题

### Q: 签名太靠上/太靠下？

**A:** 修改后端的 `y` 值：
- 增大Y值 = 向上移动（距底部更远）
- 减小Y值 = 向下移动（距底部更近）

### Q: 签名太靠左/太靠右？

**A:** 修改后端的 `x` 值：
- 增大X值 = 向右移动
- 减小X值 = 向左移动

### Q: 如何确定"本人签名"的准确位置？

**A:** 方法：
1. 在PDF中测量"本人签名："的位置
2. 使用PDF编辑工具查看坐标
3. 或通过多次测试调整参数

### Q: 不同内容的PDF签名位置会变吗？

**A:** 不会！因为使用 `from_bottom = true`，签名始终距离底部固定距离，无论内容多少。
