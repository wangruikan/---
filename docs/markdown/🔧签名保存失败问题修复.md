# 🔧 签名保存失败问题修复

## 问题现象

1. ❌ 点击"完成签名"后，签名没有真正保存
2. ❌ 再次进入页面，Canvas为空，签名丢失
3. ❌ PC端显示"该员工尚未签名"

## 问题根源

### 核心问题：Canvas尺寸未正确设置

**Canvas需要明确的width和height属性！**

```vue
<!-- 错误：只有CSS样式 -->
<canvas class="signature-canvas"></canvas>

<style>
.signature-canvas {
    width: 100%;
    height: 400rpx;  /* CSS样式不等于Canvas真实尺寸 */
}
</style>
```

在uni-app中，Canvas的绘制尺寸和显示尺寸是分开的：
- **CSS样式** - 控制Canvas在页面上的显示大小
- **width/height属性** - 控制Canvas内部的绘制分辨率

如果只设置CSS样式而不设置width/height属性，Canvas的默认绘制尺寸是300x150px，导致：
- 绘制时坐标系不对
- 导出图片尺寸错误
- 恢复显示时尺寸不匹配

### 次要问题：异步保存未完成

`saveSignature()`是异步方法，但`finishSignature()`立即显示"签名已保存"，实际可能还没保存完成。

## 完整解决方案

### 1. 计算并设置Canvas尺寸

```javascript
// data中添加尺寸变量
data() {
    return {
        canvasWidth: 0,
        canvasHeight: 0
    }
}

// onLoad时计算实际尺寸
onLoad() {
    const systemInfo = uni.getSystemInfoSync()
    const screenWidth = systemInfo.windowWidth
    this.canvasWidth = screenWidth - 60  // 减去padding和border
    this.canvasHeight = 200  // 固定高度200px
}
```

### 2. 动态设置Canvas尺寸

```vue
<canvas 
    canvas-id="signatureCanvas"
    :style="{ width: canvasWidth + 'px', height: canvasHeight + 'px' }"
    @touchstart="handleTouchStart"
    @touchmove="handleTouchMove"
    @touchend="handleTouchEnd"
></canvas>
```

### 3. 导出时使用正确尺寸

```javascript
saveSignature(callback) {
    uni.canvasToTempFilePath({
        canvasId: 'signatureCanvas',
        fileType: 'png',
        quality: 1,
        width: this.canvasWidth,          // ✅ 指定导出宽度
        height: this.canvasHeight,        // ✅ 指定导出高度
        destWidth: this.canvasWidth * 2,  // ✅ 提高导出质量
        destHeight: this.canvasHeight * 2,
        success: (res) => {
            uni.getFileSystemManager().readFile({
                filePath: res.tempFilePath,
                encoding: 'base64',
                success: (data) => {
                    this.formData.signature = 'data:image/png;base64,' + data.data
                    if (callback) callback(true)  // ✅ 回调通知成功
                },
                fail: (error) => {
                    if (callback) callback(false)  // ✅ 回调通知失败
                }
            })
        }
    }, this)
}
```

### 4. 完成签名时等待异步完成

```javascript
finishSignature() {
    if (!this.hasSignature) {
        uni.showToast({ title: '请先签名', icon: 'none' })
        return
    }
    
    // 等待保存完成后再提示
    this.saveSignature((success) => {
        if (success) {
            uni.showToast({ title: '签名已保存', icon: 'success' })
        } else {
            uni.showToast({ title: '签名保存失败', icon: 'none' })
        }
    })
}
```

### 5. 恢复签名时使用正确尺寸

```javascript
restoreSignatureToCanvas(signatureBase64) {
    if (!this.signatureCtx) {
        this.initSignatureCanvas()
    }
    
    // 使用正确的Canvas尺寸绘制图片
    this.signatureCtx.drawImage(
        signatureBase64, 
        0, 0, 
        this.canvasWidth,   // ✅ 正确的宽度
        this.canvasHeight   // ✅ 正确的高度
    )
    this.signatureCtx.draw(false)
}
```

### 6. 清除时使用正确尺寸

```javascript
clearSignature() {
    if (this.signatureCtx) {
        // 使用正确的尺寸清除
        this.signatureCtx.clearRect(0, 0, this.canvasWidth, this.canvasHeight)
        this.signatureCtx.draw(false)
        
        // 重新设置绘制样式
        this.signatureCtx.setStrokeStyle('#000000')
        this.signatureCtx.setLineWidth(3)
        this.signatureCtx.setLineCap('round')
        this.signatureCtx.setLineJoin('round')
    }
}
```

### 7. 更新CSS样式

```css
.signature-canvas {
    background: #fafafa;
    border-radius: 8rpx;
    display: block;
    /* 不再设置固定的width和height */
}
```

## 技术要点

### Canvas尺寸的三个层面

1. **CSS显示尺寸**
   - 通过CSS设置
   - 控制Canvas在页面上的显示大小
   - 如果和绘制尺寸不一致，会导致缩放

2. **Canvas绘制尺寸**
   - 通过width/height属性设置
   - 控制Canvas内部的像素数量
   - 影响绘制坐标系和导出质量

3. **导出图片尺寸**
   - 通过canvasToTempFilePath的参数设置
   - destWidth/destHeight可以提高导出质量
   - 应该和绘制尺寸匹配

### 正确的做法

```javascript
// 1. 计算实际尺寸（px）
const width = systemInfo.windowWidth - padding
const height = 200

// 2. 设置Canvas显示尺寸
<canvas :style="{ width: width + 'px', height: height + 'px' }"></canvas>

// 3. 导出时使用相同尺寸
uni.canvasToTempFilePath({
    width: width,
    height: height,
    destWidth: width * 2,    // 2倍质量
    destHeight: height * 2
})

// 4. 恢复时使用相同尺寸
ctx.drawImage(image, 0, 0, width, height)

// 5. 清除时使用相同尺寸
ctx.clearRect(0, 0, width, height)
```

## 完整工作流程

### 首次签名

1. **onLoad** → 计算Canvas尺寸
2. **用户绘制** → touchStart/Move/End在Canvas上绘制
3. **点击"完成签名"** → 
   - 调用`saveSignature(callback)`
   - 使用正确尺寸导出Canvas为base64
   - 保存到`formData.signature`
   - 回调显示"签名已保存"
4. **提交表单** → signature数据一起提交到后端

### 再次进入

1. **onLoad** → 计算Canvas尺寸
2. **loadSavedData** → 从后端加载表单数据
3. **检测到signature** → 
   - 设置`hasSignature = true`
   - 调用`restoreSignatureToCanvas(signature)`
   - 使用正确尺寸将图片绘制到Canvas
4. **用户看到** → 之前的签名显示在Canvas上

### PC端查看

1. 后端返回员工数据，包含signature字段
2. PC端检测signature有值
3. 显示签名图片：`<img :src="employee.signature" />`
