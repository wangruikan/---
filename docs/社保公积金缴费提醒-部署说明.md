# 社保公积金缴费提醒功能 - 部署说明

## 功能概述

社保公积金缴费提醒功能用于自动提醒相关人员及时上传缴费回执。

### 核心特性

1. **全局配置**：配置不分账套，所有账套共享同一套配置
2. **按月配置缴费日期**：为1-12月每个月分别设置缴费日期，每年都按此配置执行
3. **统一提醒时间**：提醒时间配置全局通用，不需要每个月单独设置
4. **按条提醒**：有几条未回执就生成几条提醒
5. **提醒对象**：
   - 社保：提醒财务人员
   - 公积金：提醒业务发起人
6. **提醒位置**：首页提醒事项 + 右上角通知

## 部署步骤

### 1. 执行数据库脚本

执行 `docs/sql/create_payment_reminder_tables.sql` 创建以下表：

- `payment_due_date_configs` - 缴费日期配置表（按月份）
- `payment_reminder_configs` - 提醒时间配置表
- `payment_reminder_logs` - 提醒记录表（用于防重复）

脚本会自动插入默认配置：
- 社保和公积金每月15号缴费
- 缴费前一天上午10点、下午5点提醒
- 缴费当天上午10点提醒

### 2. 配置定时任务

在服务器的 crontab 中添加定时任务，每小时执行一次：

```bash
0 * * * * cd /path/to/project && php artisan payment:check-reminders >> /path/to/logs/payment-reminders.log 2>&1
```

或者更频繁（每15分钟）：

```bash
*/15 * * * * cd /path/to/project && php artisan payment:check-reminders >> /path/to/logs/payment-reminders.log 2>&1
```

### 3. 前端部署

前端文件已自动包含在构建中：
- 页面：`src/views/Settings/PaymentReminder.vue`
- 路由：已添加到 `src/router/index.js`
- 菜单：已添加到 `src/config/menuConfig.js`

重新构建前端：

## 使用说明

### 配置缴费日期

1. 进入"系统设置" -> "缴费提醒配置"
2. 选择"社保"或"公积金"标签页
3. 可以：
   - 批量设置：输入缴费日，点击"批量应用到所有月份"
   - 单独设置：直接修改每个月的缴费日
4. 点击"保存社保配置"或"保存公积金配置"

### 配置提醒时间

1. 在"提醒时间配置"区域点击"添加提醒"
2. 选择缴费类型（社保/公积金）
3. 设置提前天数（0表示当天）
4. 设置提醒时间（如：10:00:00）
5. 保存

### 提醒规则

系统会在每个提醒时间点检查：

1. 根据当前日期和提前天数，计算目标缴费月份
2. 查询该月份的缴费日期配置
3. 查找所有未完成回执的付款申请（状态为approved，发票状态为pending_invoice或invoice_uploaded）
4. 为每条未完成的付款申请生成一条提醒
5. 提醒对象：
   - 社保：该账套的财务人员
   - 公积金：付款申请的发起人

### 查看提醒

用户可以在以下位置看到提醒：

1. **首页提醒事项**：显示所有待处理的缴费提醒
2. **右上角通知**：实时显示新的提醒通知

## 注意事项

1. **全局配置**：配置对所有账套生效，修改后会影响所有账套
2. **每年通用**：缴费日期配置按月份设置，每年都按此配置执行
3. **防重复机制**：同一条付款申请在同一天的同一时间点只会提醒一次
4. **提醒记录**：提醒记录表仅用于防止重复提醒，不在前端显示
5. **定时任务**：确保定时任务正常运行，否则提醒功能无法工作

## 故障排查

### 提醒没有发送

1. 检查定时任务是否正常运行：
   ```bash
   # 查看定时任务日志
   tail -f /path/to/logs/payment-reminders.log
   ```

2. 检查配置是否正确：
   ```sql
   -- 检查缴费日期配置
   SELECT * FROM payment_due_date_configs WHERE payment_type = 'social_security';
   
   -- 检查提醒时间配置
   SELECT * FROM payment_reminder_configs WHERE is_active = 1;
   ```

3. 检查是否有未完成回执的付款申请：
   ```sql
   SELECT * FROM payment_requests 
   WHERE payment_type = 'insurance' 
   AND status = 'approved' 
   AND invoice_status IN ('pending_invoice', 'invoice_uploaded');
   ```

### 提醒重复发送

检查 `payment_reminder_logs` 表，确认防重复机制是否正常工作：

```sql
SELECT * FROM payment_reminder_logs 
WHERE payment_request_id = [付款申请ID]
ORDER BY created_at DESC;
```

### 找不到提醒对象

1. 社保提醒：检查账套是否有财务角色的用户
   ```sql
   SELECT u.* FROM users u
   JOIN account_set_users asu ON u.id = asu.user_id
   WHERE asu.account_set_id = [账套ID]
   AND (u.role = 'finance' OR u.role LIKE '%财务%');
   ```

2. 公积金提醒：检查付款申请的发起人是否存在
   ```sql
   SELECT * FROM users WHERE id = [发起人ID];
   ```
