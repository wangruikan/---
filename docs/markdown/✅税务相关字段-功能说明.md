# 税务相关字段 - 功能说明

## 📝 新增字段

为工资表添加了3个税务相关字段：

| 字段名 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| **累计其他应纳税项（合并扣税）** | DECIMAL(10,2) | 0.00 | 用于记录其他应纳税收入 |
| **应补（退）税额** | DECIMAL(10,2) | 0.00 | 应补缴或应退还的税额 |
| **本人签字** | VARCHAR(255) | NULL | 员工签字确认 |

---

## 🎯 字段说明

### 1️⃣ 累计其他应纳税项（合并扣税）

**用途**：记录除工资外的其他应税收入，例如：
- 年终奖
- 兼职收入
- 劳务报酬
- 稿酬
- 特许权使用费

**计算规则**：
- 目前默认为 `0.00`
- 可以根据实际情况手动修改
- 在计算个人所得税时会累加到应纳税所得额中

**示例**：
```
员工当年除工资外还有：
- 年终奖：10,000元
- 兼职收入：5,000元

累计其他应纳税项 = 10,000 + 5,000 = 15,000元
```

---

### 2️⃣ 应补（退）税额

**用途**：记录年度汇算清缴时需要补缴或退还的个人所得税。

**计算公式**：
```
应补（退）税额 = 全年应纳税额 - 全年已预缴税额
```

**值的含义**：
- **正数**：需要补缴税款（显示为红色）
- **负数**：可以退税（显示为绿色）
- **零**：不补不退

**示例**：
```
情况1：需要补税
- 全年应纳税额：12,000元
- 全年已预缴税额：10,000元
- 应补（退）税额：12,000 - 10,000 = +2,000元（需要补缴）

情况2：可以退税
- 全年应纳税额：8,000元
- 全年已预缴税额：10,000元
- 应补（退）税额：8,000 - 10,000 = -2,000元（可以退税）
```

---

### 3️⃣ 本人签字

**用途**：记录员工对工资条的确认签字。

**使用场景**：
- 员工查看工资条后，确认无误后签字
- 可以填写姓名、日期或"已确认"等
- 作为工资发放的凭证

**示例**：
```
- "张三 2025-10-15"
- "已确认"
- "同意"
```

## 🔧 实现细节

### 数据库字段
```sql
-- 1. 累计其他应纳税项（合并扣税）
ALTER TABLE `salaries` 
ADD COLUMN `cumulative_other_taxable` DECIMAL(10, 2) DEFAULT 0.00 
COMMENT '累计其他应纳税项（合并扣税）' 
AFTER `taxable_income`;

-- 2. 应补（退）税额
ALTER TABLE `salaries` 
ADD COLUMN `tax_payable_or_refundable` DECIMAL(10, 2) DEFAULT 0.00 
COMMENT '应补（退）税额' 
AFTER `cumulative_other_taxable`;

-- 3. 本人签字
ALTER TABLE `salaries` 
ADD COLUMN `employee_signature` VARCHAR(255) DEFAULT NULL 
COMMENT '本人签字' 
AFTER `tax_payable_or_refundable`;
```

## 🎯 使用场景

### 年度汇算清缴
```
场景：2025年度个税汇算清缴

步骤：
1. 查看员工全年工资记录
2. 计算全年应纳税额和已预缴税额
3. 填写"应补（退）税额"
4. 员工确认后在"本人签字"栏签字
5. 完成汇算清缴流程
```

### 其他应税收入管理
```
场景：员工有兼职收入需要合并计税

步骤：
1. 在"累计其他应纳税项"中填写兼职收入
2. 系统重新计算应纳税额
3. 更新"应补（退）税额"
4. 员工确认签字
```
