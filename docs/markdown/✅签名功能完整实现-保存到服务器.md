# ✅ 签名功能完整实现 - 保存到服务器

## 核心思路

**签名图片保存到服务器，数据库存储路径，前端显示完整URL**

### 完整流程

```
1. 用户在Canvas上签名（多笔）
   ↓
2. 点击"完成签名"
   ↓
3. Canvas导出base64图片
   ↓
4. 提交表单时，base64一起提交到后端
   ↓
5. 后端接收base64，保存为图片文件
   ↓
6. 数据库存储相对路径：signatures/xxx.png
   ↓
7. 再次加载时，后端返回完整URL
   ↓
8. 前端显示图片预览
```

```php
private function saveSignatureImage($base64Image)
{
    // 去掉base64前缀
    $image = preg_replace('/^data:image\/\w+;base64,/', '', $base64Image);
    $imageData = base64_decode($image);
    
    // 生成文件名
    $filename = 'signatures/' . uniqid() . '_' . time() . '.png';
    
    // 保存到 storage/app/public/signatures/
    Storage::disk('public')->put($filename, $imageData);
    
    return $filename; // 返回相对路径：signatures/xxx.png
}
```

#### 1.3 返回完整URL

**MiniController.php - getMyOnboardingForm方法**

```php
public function getMyOnboardingForm(Request $request)
{
    try {
        $employee = Employee::find($request->user()->id);
        $form = OnboardingForm::where('employee_id', $employee->id)->first();

        // 将签名路径转换为完整URL
        if ($form && $form->signature) {
            $formData = $form->toArray();
            $formData['signature'] = asset('storage/' . $formData['signature']);
            // asset() 生成：http://127.0.0.1:8000/storage/signatures/xxx.png
            $form = $formData;
        }

        return response()->json([
            'success' => true,
            'data' => $form
        ]);
    } catch (\Exception $e) {
        // ...
    }
}
```

### 2. 前端修改

#### 2.1 Canvas绘制和保存

**ht_qshu/pages/onboarding/index.vue**

```javascript
// 完成签名
finishSignature() {
    if (!this.hasSignature) {
        uni.showToast({ title: '请先签名', icon: 'none' })
        return
    }
    
    // 保存签名为base64（异步）
    this.saveSignature((success) => {
        if (success) {
            uni.showToast({ title: '签名已保存', icon: 'success' })
            // formData.signature 现在是 base64 字符串
        } else {
            uni.showToast({ title: '签名保存失败', icon: 'none' })
        }
    })
}

// 保存签名为base64
saveSignature(callback) {
    uni.canvasToTempFilePath({
        canvasId: 'signatureCanvas',
        fileType: 'png',
        quality: 1,
        width: this.canvasWidth,
        height: this.canvasHeight,
        destWidth: this.canvasWidth * 2,  // 提高质量
        destHeight: this.canvasHeight * 2,
        success: (res) => {
            uni.getFileSystemManager().readFile({
                filePath: res.tempFilePath,
                encoding: 'base64',
                success: (data) => {
                    // 保存为base64字符串
                    this.formData.signature = 'data:image/png;base64,' + data.data
                    if (callback) callback(true)
                }
            })
        }
    }, this)
}
```

## 文件存储结构

```
storage/
  ├── app/
  │   └── public/
  │       └── signatures/           # 签名图片目录
  │           ├── abc123_1234567890.png
  │           ├── def456_1234567891.png
  │           └── ...
  └── ...

public/
  └── storage/  → symlink to storage/app/public/
      └── signatures/
          ├── abc123_1234567890.png
          └── ...
```

**注意：** 需要运行 `php artisan storage:link` 创建符号链接！

## 数据库存储

**onboarding_forms表 - signature字段：**

```
| id | employee_id | signature                      | created_at | updated_at |
|----|-------------|--------------------------------|------------|------------|
| 1  | 10          | signatures/abc123_1234567890.png | ...        | ...        |
| 2  | 11          | signatures/def456_1234567891.png | ...        | ...        |
```

- ✅ 存储**相对路径**，不是完整URL
- ✅ 便于服务器迁移（域名变化无影响）
- ✅ 返回时动态拼接完整URL

## 关键技术点

### 1. asset() 函数

```php
asset('storage/signatures/xxx.png')
// 生成：http://127.0.0.1:8000/storage/signatures/xxx.png
```

### 2. storage_path() 函数

```php
storage_path('app/public/signatures/xxx.png')
// 生成：/var/www/storage/app/public/signatures/xxx.png
// 用于本地文件系统访问（PDF生成）
```

### 3. Storage::disk('public')

```php
Storage::disk('public')->put('signatures/xxx.png', $imageData)
// 保存到：storage/app/public/signatures/xxx.png
```

### 4. 符号链接

```bash
php artisan storage:link
# 创建：public/storage -> storage/app/public
```

这样浏览器访问 `/storage/signatures/xxx.png` 就能读取到文件。

## 总结

### 核心改进

**之前方案（错误）：**
- ❌ 保存base64字符串到数据库
- ❌ 字段类型TEXT，数据库臃肿
- ❌ 前端需要Base64数据才能显示
- ❌ PDF生成困难

**现在方案（正确）：**
- ✅ 保存图片文件到服务器
- ✅ 数据库只存路径，轻量高效
- ✅ 前端显示图片URL，标准做法
- ✅ PDF直接读取本地文件

### 技术优势

1. **数据库轻量**：只存路径字符串
2. **性能优异**：图片由Web服务器直接返回
3. **易于管理**：所有签名集中在一个目录
4. **支持CDN**：可以轻松迁移到OSS/CDN
5. **标准实践**：符合文件上传的最佳实践
