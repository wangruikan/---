# ✅ 修复 - 基数调差记录自动创建功能

## 修复时间
2025-11-21 14:30

## 问题描述

劳动合同审批通过后，基数调差管理中没有自动创建记录，显示"暂无数据"。

## 根本原因

### 问题 1：状态不匹配
- **代码检查**：`status === 'completed'`
- **数据库实际**：`status = 'complete'`
- **结果**：条件判断失败，自动创建功能从未执行

### 问题 2：ENUM 类型错误
- **代码使用**：`status = 'effective'`
- **数据库允许**：`enum('pending', 'approved', 'applied', 'cancelled')`
- **结果**：插入失败，报错 "Data truncated for column 'status'"

### 问题 3：时间格式未格式化
- **问题**：历史记录对话框的时间字段直接显示原始值
- **结果**：时间显示格式不统一，可读性差

## 修复方案

**文件**：`app/Services/ApprovalService.php`

```

**说明**：同时支持 `complete` 和 `completed` 两种状态，确保兼容性。

<!-- 生效日期 -->
<el-table-column prop="social_security_effective_date" label="社保生效时间" width="120" align="center">

```

**修改后**：
```vue
<!-- 生效日期 -->
<el-table-column prop="social_security_effective_date" label="社保生效时间" width="120" align="center">
  <template #default="{ row }">
    {{ formatDate(row.social_security_effective_date) }}
  </template>
</el-table-column>

<!-- 创建时间 -->
<el-table-column prop="created_at" label="创建时间" width="160" align="center">
  <template #default="{ row }">
    {{ formatDateTime(row.created_at) }}
  </template>
</el-table-column>

<!-- 实际生效时间 -->
<el-table-column prop="applied_at" label="实际生效时间" width="160" align="center">
  <template #default="{ row }">
    {{ formatDateTime(row.applied_at) }}
  </template>
</el-table-column>
```

**新增方法**：
```javascript
// 格式化日期时间
const formatDateTime = (dateTimeString) => {
  if (!dateTimeString) return '-'
  
  // 如果已经是 YYYY-MM-DD HH:mm:ss 格式，直接返回
  if (/^\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}$/.test(dateTimeString)) {
    return dateTimeString
  }
  
  // 处理其他格式
  const date = new Date(dateTimeString)
  if (isNaN(date.getTime())) return dateTimeString // 无效日期，返回原值
  
  const year = date.getFullYear()
  const month = String(date.getMonth() + 1).padStart(2, '0')
  const day = String(date.getDate()).padStart(2, '0')
  const hours = String(date.getHours()).padStart(2, '0')
  const minutes = String(date.getMinutes()).padStart(2, '0')
  const seconds = String(date.getSeconds()).padStart(2, '0')
  
  return `${year}-${month}-${day} ${hours}:${minutes}:${seconds}`
}
```

**说明**：
- 生效日期使用 `formatDate()`，格式：`2025-11-21`
- 创建时间和实际生效时间使用 `formatDateTime()`，格式：`2025-11-21 14:30:00`

## 功能说明

### 自动创建 vs 手动调整

#### 相同点
1. **数据表**：都存储在 `base_adjustments` 表
2. **状态**：都是 `applied`（已生效）
3. **字段结构**：完全相同
4. **生效逻辑**：都会立即应用到员工档案

#### 不同点

| 对比项 | 自动创建（入职） | 手动调整 |
|--------|------------------|----------|
| 触发方式 | 劳动合同审批通过 | 用户手动创建 |
| 旧基数 | 0（新员工） | 员工当前基数 |
| 新基数 | 员工创建时填写的基数 | 用户调整后的基数 |
| 调整原因 | "劳动合同审批通过，自动导入员工基数" | 用户填写的原因 |
| 生效时间 | 立即生效（当天） | 用户指定的生效日期 |
| 状态 | `applied`（已生效） | `pending`（待生效）或 `applied`（已生效） |

### 时间格式说明

| 字段类型 | 格式化方法 | 显示格式 | 示例 |
|----------|-----------|----------|------|
| 生效日期 | `formatDate()` | `YYYY-MM-DD` | `2025-11-21` |
| 创建时间 | `formatDateTime()` | `YYYY-MM-DD HH:mm:ss` | `2025-11-21 14:30:00` |
| 实际生效时间 | `formatDateTime()` | `YYYY-MM-DD HH:mm:ss` | `2025-11-21 14:30:00` |

## 注意事项

1. **状态匹配**：确保合同审批完成后的状态为 `complete` 或 `completed`
2. **基数必填**：员工创建时必须填写基数，否则不会自动创建记录
3. **合同类型**：只有劳动合同（`contract_type = 'labor'`）才会触发自动创建
4. **不重复创建**：如果员工已有基数调差记录，不会重复创建
5. **时间格式**：历史记录中的时间会自动格式化显示
