# ✅ 发票申请 - 驳回改为红冲状态

## 📋 需求

1. **审批被驳回时**：状态直接改为"红冲"（不再显示"已驳回"）
2. **红冲状态**：可以点击"重新发起"按钮，创建新申请并启动新的审批流程

#### 文件：`app/Services/ApprovalService.php`

```php
case '发票申请':
case '发票申请（重新提交）':
    // 更新发票申请状态
    $invoiceApplication = \App\Models\InvoiceApplication::find($businessId);
    if ($invoiceApplication) {
        if ($status === 'completed') {
            // 审批通过
            $invoiceApplication->update([
                'status' => 'approved',
            ]);
        } elseif ($status === 'rejected') {
            // 驳回时直接改为红冲状态
            $invoiceApplication->update([
                'status' => 'red_flushed',
            ]);
        } elseif ($status === 'in_approval') {
            // 审批中
            $invoiceApplication->update([
                'status' => 'pending',
            ]);
        }
    }
    break;
```

### 2. 模型 - 修改权限判断方法

```php
public function canEdit()
{
    return in_array($this->status, [self::STATUS_DRAFT, self::STATUS_REJECTED]);
}

public function canResubmit()
{
    return $this->status === self::STATUS_REJECTED;
}
```

**修改后：**
```php
public function canEdit()
{
    // 只有草稿状态可以编辑（红冲状态不可编辑）
    return in_array($this->status, [self::STATUS_DRAFT]);
}

public function canResubmit()
{
    // 只有红冲状态可以重新发起
    return $this->status === self::STATUS_RED_FLUSHED;
}
```

### 3. 后端控制器 - 修改重新发起逻辑

#### 文件：`app/Http/Controllers/InvoiceApplicationController.php`

**修改 `resubmit()` 方法**：

```php
/**
 * 重新发起（红冲后）
 */
public function resubmit($id)
{
    $application = InvoiceApplication::with('items')->find($id);

    // 验证状态
    if (!$application->canResubmit()) {
        return response()->json([
            'success' => false,
            'message' => '只有红冲状态的申请才能重新发起'
        ], 400);
    }

    DB::beginTransaction();
    try {
        // 创建新的申请（复制原红冲申请的数据）
        $newApplication = InvoiceApplication::create([
            'account_set_id' => $application->account_set_id,
            'application_no' => InvoiceApplication::generateApplicationNo(),
            'task_name' => $application->task_name,
            'year' => $application->year,
            'month' => $application->month,
            'project_name' => $application->project_name,  // 复制项目名称
            'status' => InvoiceApplication::STATUS_PENDING,
            'submitter_id' => $application->submitter_id,
            'submitted_at' => now(),
            'attachments' => $application->attachments,
            'created_by' => Auth::id(),
        ]);

        // 复制明细项...
        // 创建审批实例...

        return response()->json([
            'success' => true,
            'message' => '重新发起成功，已创建新的申请并提交审批',
            'data' => $newApplication->load(['items', 'approvalInstance'])
        ]);
    }
}
```

**关键改动**：
- ✅ 复制时包含 `project_name` 字段
- ✅ 不再需要手动将原申请改为红冲（审批驳回时已经是红冲）
- ✅ 错误提示改为"只有红冲状态的申请才能重新发起"

```javascript
// 重新发起
const handleResubmit = async (row) => {
  try {
    await ElMessageBox.confirm(
      '将基于此红冲申请创建新的申请并发起审批流程，确定要继续吗？',
      '重新发起申请',
      {
        confirmButtonText: '确定',
        cancelButtonText: '取消',
        type: 'warning'
      }
    )

    const response = await resubmitInvoiceApplication(row.id)

    if (response.success) {
      ElMessage.success(response.message || '重新发起成功')
      loadData()
    }
  } catch (error) {
    if (error !== 'cancel') {
      console.error('重新发起失败', error)
      ElMessage.error(error.response?.data?.message || '重新发起失败')
    }
  }
}
```

## 🎯 功能说明

### 1. 审批被驳回

**触发时机**：审批人在审批流程中点击"驳回"

**系统行为**：
1. 审批实例状态改为 `rejected`
2. **发票申请状态直接改为 `red_flushed`（红冲）**
3. 记录驳回原因

**用户看到**：
- 列表中状态显示"红冲"标签（红色）
- 操作列显示"重新发起"按钮

---

### 2. 重新发起申请

**触发时机**：业务人员点击"重新发起"按钮

**系统行为**：
1. 验证原申请状态必须是 `red_flushed`
2. 创建新的发票申请：
   - 生成新的申请单号
   - 复制原申请的所有数据（任务名称、年份、月份、**项目名称**、明细项、附件）
   - 状态设为 `pending`（审批中）
3. 创建新的审批实例
4. 原红冲申请保持不变（保留历史记录）

**用户看到**：
- 提示"重新发起成功，已创建新的申请并提交审批"
- 列表中出现新的申请记录（状态：审批中）
- 原红冲申请仍在列表中（状态：红冲）

## 📋 状态说明

| 状态代码 | 显示文本 | 何时出现 | 可以操作 |
|---------|---------|---------|---------|
| `draft` | 草稿 | 创建任务后 | 编辑、提交审批、删除 |
| `pending` | 审批中 | 提交审批后 | 查看 |
| `approved` | 已通过 | 审批完成 | 查看 |
| `red_flushed` | 红冲 | **审批被驳回** | 查看、**重新发起** |

## 🔄 业务流程

### 场景1：审批通过
1. 审批人创建开票任务
2. 业务人员填写项目、添加明细、上传附件
3. 提交审批
4. 审批人逐级审批
5. **全部通过** → 状态变为"已通过"

### 场景2：审批被驳回（新逻辑）
1. 审批人创建开票任务
2. 业务人员填写项目、添加明细、上传附件
3. 提交审批
4. 某个审批人点击"驳回"
5. **申请状态变为"红冲"**（不再是"已驳回"）
6. 业务人员点击"重新发起"
7. 系统创建新申请（复制所有数据）并自动提交审批
8. 新申请进入审批流程
9. 原红冲申请保留在系统中作为历史记录
