# ✅ 修复 - 正常状态不能添加明细和上传附件

## 🔍 问题描述

在发票申请详情页（编辑模式）中：
- ❌ 看不到"添加明细"按钮
- ❌ 看不到"上传附件"按钮
- ❌ 不能编辑、删除明细项

## 📋 原因分析

前端的 `canEdit` 计算属性判断条件**没有更新**，还是旧的逻辑：

**旧逻辑**（错误）：
```javascript
const canEdit = computed(() => {
  return isEditMode.value && 
         (currentApplication.value.status === 'draft' || 
          currentApplication.value.status === 'rejected')
})
```

这个逻辑只允许 `draft` 和 `rejected` 状态编辑，**不包含** `normal` 和 `red_flushed` 状态！

## ✅ 解决方案

### 文件：`src/views/InvoiceApplications/index.vue`

**修改前**：
```javascript
const canEdit = computed(() => {
  return isEditMode.value && 
         (currentApplication.value.status === 'draft' || currentApplication.value.status === 'rejected')
})
```

**修改后**：
```javascript
const canEdit = computed(() => {
  return isEditMode.value && 
         ['draft', 'normal', 'red_flushed'].includes(currentApplication.value.status)
})
```

## 🎯 修改说明

### 现在可以编辑的状态

| 状态 | 可以编辑 | 说明 |
|------|---------|------|
| `draft` | ✅ | 草稿状态（保留用于兼容） |
| **`normal`** | ✅ | **正常状态（新增）** |
| `pending` | ❌ | 审批中，不可编辑 |
| `approved` | ❌ | 已通过，不可编辑 |
| `rejected` | ❌ | 已驳回（已废弃，改用red_flushed） |
| **`red_flushed`** | ✅ | **红冲状态（新增）** |

### 编辑模式下的功能

当 `canEdit = true` 时，会显示：

1. **基本信息**
   - ✅ 项目名称输入框（可编辑）

2. **扣除明细**
   - ✅ "添加明细"按钮
   - ✅ 每行的"编辑"和"删除"按钮

3. **附件上传**
   - ✅ "上传附件"按钮
   - ✅ 每个附件的"删除"按钮

4. **底部操作**
   - ✅ "提交审批"按钮（需满足条件）

## 📊 与后端的一致性

### 后端模型权限判断

```php
// app/Models/InvoiceApplication.php
public function canEdit()
{
    return in_array($this->status, [
        self::STATUS_DRAFT, 
        self::STATUS_NORMAL, 
        self::STATUS_RED_FLUSHED
    ]);
}
```

### 前端权限判断（已修复）

```javascript
// src/views/InvoiceApplications/index.vue
const canEdit = computed(() => {
  return isEditMode.value && 
         ['draft', 'normal', 'red_flushed'].includes(currentApplication.value.status)
})
```

✅ **现在前后端逻辑一致了！**
