**执行 `add_position_to_employees.sql`**：

```sql

ADD COLUMN position VARCHAR(100) NULL COMMENT '岗位' AFTER name;
```

```sql
DESC employees;

-- 应该看到 position 字段
```

## 🚀 立即部署

### 1. 执行SQL
```sql
-- 添加岗位字段
SOURCE add_position_to_employees.sql;
```

### 2. 刷新前端
```
Ctrl + F5 强制刷新浏览器
```

### 3. 生成工资表
```
1. 进入"薪金计算"页面
2. 点击"生成工资表"
3. 选择期间和项目
4. 点击"确定生成"
```

### 4. 查看明细
```
1. 点击"查看明细"按钮
2. 点击每行前面的展开箭头 ">"
3. 查看详细的保险明细
```

## 📊 数据流向

```
员工档案 (employees)
  ↓
  - 姓名
  - 部门
  - 岗位 ✨（新字段）
  - 基础工资
  - 社保基数
  - 公积金基数
  - 大额医疗基数
  
参保人员表 (insurance_personnel)
  ↓
  - 社保险种配置（JSON）
    - 养老：单位16% + 个人8%
    - 医疗：单位10% + 个人2%
    - 失业：单位0.8% + 个人0.2%
    - 工伤：单位0.2% + 个人0%
    
  - 公积金配置ID
  
公积金配置表 (housing_fund_configs)
  ↓
  - 单位比例：12%
  - 个人比例：12%
  
        ↓ 自动计算
        
工资表 (salaries)
  ↓
  - 基本信息（部门、岗位）
  - 基础工资
  - 单位保险合计
  - 个人保险合计
  - 实发工资
  
        ↓ 查看明细时
        
保险明细（实时计算）
  ↓
  - 社保明细（每个险种的基数、比例、金额）
  - 公积金明细（基数、比例、金额）
  - 大额医疗明细（基数、金额）
```
