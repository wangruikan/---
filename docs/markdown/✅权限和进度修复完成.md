## 📋 权限控制详情

### 🔍 审批流程结构
- **ApprovalInstance** - 审批实例（包含总步数、当前步骤）
- **ApprovalRecord** - 审批记录（包含步骤顺序、审批人ID）
- **step_order** - 审批步骤顺序（1、2、3、4）

### ✅ 可以访问的用户
1. **超级管理员** (`super_admin`)
2. **审批节点2的用户** - 在`ApprovalRecord`中`step_order = 2`
3. **审批节点3的用户** - 在`ApprovalRecord`中`step_order = 3`
4. **审批节点4的用户** - 在`ApprovalRecord`中`step_order = 4`
5. **备用角色** - `admin`, `manager`, `senior_manager`
6. **备用级别** - `approval_level >= 2`

### ❌ 无法访问的用户
1. **第1审批节点用户** - 只在`step_order = 1`的用户
2. **普通员工** - 没有审批权限的用户
3. **无审批记录的用户** - 从未参与过审批流程的用户

## 🔧 技术实现

### 后端权限检查
```php
private function canViewProcessRecords($user)
{
    // 超级管理员
    if ($user->role === 'super_admin') return true;
    
    // 审批节点检查
    $userApprovalSteps = ApprovalRecord::where('approver_id', $user->id)
        ->distinct('step_order')->pluck('step_order')->toArray();
    
    $allowedSteps = [2, 3, 4];
    return !empty(array_intersect($userApprovalSteps, $allowedSteps));
}
```

### 前端权限控制
```javascript
const canViewProcessRecords = computed(() => {
  const user = userStore.userInfo
  if (!user) return false
  
  // 超级管理员
  if (user.role === 'super_admin') return true
  
  // 角色和级别检查（简化版）
  const allowedRoles = ['admin', 'manager', 'senior_manager']
  if (allowedRoles.includes(user.role)) return true
  
  return user.approval_level >= 2
})
```
