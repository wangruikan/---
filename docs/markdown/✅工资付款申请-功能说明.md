# 工资付款申请功能说明

## 功能概述

工资付款申请功能允许在工资表审批通过后，发起付款申请并进入审批流程。该功能与保险汇总付款申请类似，支持附件上传和多级审批，审批通过后自动跳过发起人。

## 主要特性

1. **工资表审批通过后发起付款**
   - 只有审批状态为"已审批"的工资表才能发起付款申请
   - 发起付款时需要上传附件（至少1个文件）
   - 自动计算付款金额（所有员工的实发工资总和）

2. **附件管理**
   - 支持上传PDF、Word、Excel、图片等格式
   - 单个文件最大50MB
   - 最多上传5个文件
   - 附件存储在 `public/payment_requests/{id}/` 目录

3. **审批流程**
   - 集成到通用审批管理系统
   - 自动跳过发起人（第一个审批节点）
   - 支持多级审批
   - 审批记录可在"审批管理"模块查看

4. **付款申请分类**
   - **工资付款申请**：从工资表发起的付款申请
   - **保险汇总付款申请**：从保险汇总发起的付款申请（原有功能）

## 使用流程

### 1. 工资表审批

1. 在"工资管理"页面生成工资表
2. 点击"提交审批"按钮
3. 选择盖章方式（线上/线下）
4. 上传附件（至少1个文件）
5. 提交审批

### 2. 审批流程

1. 审批流程自动创建，跳过发起人
2. 第二个审批节点的审批人在"审批管理 > 待办"中查看
3. 审批人审批通过或驳回
4. 审批通过后，工资表状态变为"已审批"

### 3. 发起付款申请

1. 在"工资管理"页面，审批通过的工资表显示"发起付款"按钮
2. 点击"发起付款"按钮
3. 填写备注信息
4. 上传附件（至少1个文件）
5. 提交付款申请

### 4. 付款审批

1. 付款审批流程自动创建，跳过发起人
2. 审批人在"审批管理 > 待办"中查看付款申请
3. 审批人审批通过或驳回
4. 审批通过后，付款申请状态变为"已批准"

### 5. 查看付款申请

1. 在"付款申请"模块查看所有付款申请
2. 付款申请分为两类：
   - **工资付款申请**：从工资表发起
   - **保险汇总付款申请**：从保险汇总发起

## 注意事项

1. **附件存储**
   - 附件存储在 `public/payment_requests/{id}/` 目录
   - 确保该目录有写入权限
   - 建议定期清理已删除的付款申请的附件

2. **审批流程**
   - 发起人会自动跳过第一个审批节点
   - 确保审批流程配置正确（至少2个审批节点）

3. **付款金额**
   - 付款金额自动计算为所有员工的实发工资总和
   - 金额只读，不可修改

4. **重复提交**
   - 每个工资表只能发起一次付款申请
   - 如需重新发起，需先删除原付款申请记录

5. **文件大小限制**
   - 单个文件最大50MB
   - 最多上传5个文件
   - 确保服务器 `upload_max_filesize` 和 `post_max_size` 配置足够大

## 常见问题

### Q1: 发起付款后，审批流程没有创建？
**A**: 检查以下几点：
1. 确保 `approval_instance_id` 字段已添加到 `payment_requests` 表
2. 检查审批流程配置是否正确
3. 查看 Laravel 日志文件 `storage/logs/laravel.log`

### Q2: 附件上传失败？
**A**: 检查以下几点：
1. 确保 `public/payment_requests` 目录存在并有写入权限
2. 检查文件大小是否超过限制
3. 检查服务器 PHP 配置 `upload_max_filesize` 和 `post_max_size`

### Q3: 付款申请列表中看不到工资付款申请？
**A**: 检查以下几点：
1. 确保 `payment_type` 字段值为 `'salary'`
2. 检查账套过滤条件
3. 查看浏览器控制台是否有错误

### Q4: 审批通过后，状态没有更新？
**A**: 检查以下几点：
1. 确保 `ApprovalService::updateBusinessStatus()` 中有 `case '工资付款申请'`
2. 清除缓存：`php artisan cache:clear`
3. 查看 Laravel 日志
