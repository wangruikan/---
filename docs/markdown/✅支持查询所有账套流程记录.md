# ✅ 支持查询所有账套流程记录

## 🎯 需求说明

用户希望在"流程记录管理"页面查看**所有账套**的流程记录，而不是只看当前选择账套的数据。

## 📊 问题分析

### 原始数据分布
```
账套1（默认套账）: 165条流程记录 ✅
账套3（账套2）:     0条流程记录  ❌
```

### 问题现象
- 用户选择"账套2"时，显示"暂无数据"
- 所有165条流程数据都在"账套1"中
- 用户想看到所有账套的完整数据

## 🔧 解决方案

### 1. 移除账套ID强制要求
**修改前：**
```php
if (!$accountSetId) {
    return response()->json([
        'success' => false,
        'message' => '请选择账套'
    ], 400);
}

$query = ApprovalInstance::where('account_set_id', $accountSetId);
```

**修改后：**
```php
// 不再要求必须有account_set_id
$query = ApprovalInstance::with(['creator', 'records', 'accountSet']);

// 如果提供了账套ID，则只查询该账套的数据
if ($accountSetId) {
    $query->where('account_set_id', $accountSetId);
}
```

### 2. 添加accountSet关系

在 `ApprovalInstance.php` 模型中添加：
```php
/**
 * 所属账套
 */
public function accountSet()
{
    return $this->belongsTo(AccountSet::class, 'account_set_id');
}
```

### 3. 在返回数据中包含账套信息

```php
return [
    'id' => $instance->id,
    'account_set_id' => $instance->account_set_id,
    'account_set_name' => $instance->accountSet ? $instance->accountSet->name : '未知账套',
    // ... 其他字段
];
```

### 4. 前端显示账套列

在表格中添加账套列：
```vue
<el-table-column prop="account_set_name" label="所属账套" width="120" />
```

### 5. 更新权限检查逻辑

```php
private function canViewProcessRecords($user, $accountSetId = null)
{
    // 超级管理员可以查看
    if ($user->role === 'super_admin') {
        return true;
    }

    // 管理员角色可以查看
    $allowedRoles = ['admin', 'manager', 'senior_manager'];
    if (in_array($user->role, $allowedRoles)) {
        return true;
    }

    // 如果指定了账套ID，检查该账套权限
    if ($accountSetId) {
        $approver = \DB::table('account_set_users')
            ->where('account_set_id', $accountSetId)
            ->where('user_id', $user->id)
            ->whereIn('approval_level', [2, 3, 4])
            ->first();
        
        if ($approver) {
            return true;
        }
    } else {
        // 未指定账套时，检查用户在任意账套中是否有2/3/4级别权限
        $approver = \DB::table('account_set_users')
            ->where('user_id', $user->id)
            ->whereIn('approval_level', [2, 3, 4])
            ->first();
        
        if ($approver) {
            return true;
        }
    }

    return false;
}
```

### 6. 统计数据支持

统计接口也支持查询所有账套：
```php
$baseQuery = \App\Models\ApprovalInstance::query();

// 如果提供了账套ID，则只统计该账套
if ($accountSetId) {
    $baseQuery->where('account_set_id', $accountSetId);
}

$stats = [
    'total' => (clone $baseQuery)->count(),
    'pending' => (clone $baseQuery)->where('status', 'pending')->count(),
    // ...
];
```

### 使用方式

**方式1：查看所有账套**
- 不选择特定账套（或传空account_set_id）
- 显示所有账套的流程记录
- 统计卡片显示总计数据

**方式2：筛选特定账套**
- 选择特定账套
- 只显示该账套的流程记录
- 统计卡片只统计该账套数据

## 🔑 关键改进

1. **灵活查询** - 支持查询所有账套或特定账套
2. **权限合理** - 有审批权限的用户可以查看所有流程
3. **信息完整** - 显示每条记录所属的账套
4. **向后兼容** - 仍然支持按账套筛选

## 📋 权限说明

可以查看流程记录的用户：
- ✅ 超级管理员（super_admin）
- ✅ 管理员角色（admin, manager, senior_manager）
- ✅ 在任意账套中审批级别为2/3/4的用户

不能查看的用户：
- ❌ 只有审批级别1（经办）的用户
- ❌ 没有任何账套关联的用户
