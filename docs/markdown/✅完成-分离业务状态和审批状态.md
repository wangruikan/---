# ✅ 完成 - 分离业务状态和审批状态

## 📋 需求

将发票申请的状态分为两个独立的字段：

1. **业务状态（status）**：只有"正常"和"红冲"两种
   - 初始为"正常"
   - 被驳回后变成"红冲"
   - **一旦变成红冲就永久保持，不再恢复到正常**

2. **审批状态（approval_status）**：表示审批流程状态
   - 未提交：`NULL`
   - 审批中：`pending`
   - 已通过：`approved`
   - 已驳回：`rejected`

---

## ✅ 实现方案

### 核心设计

```
业务状态（status）      审批状态（approval_status）
    ↓                        ↓
  正常/红冲              NULL/pending/approved/rejected
    ↓                        ↓
  用于业务判断          用于流程判断
```

### 状态流转

```
创建申请
  ├─ 业务状态: 正常
  └─ 审批状态: NULL（未提交）

提交审批
  ├─ 业务状态: 正常（不变）
  └─ 审批状态: pending

审批通过
  ├─ 业务状态: 正常（不变）
  └─ 审批状态: approved

审批驳回
  ├─ 业务状态: 红冲（改变！）
  └─ 审批状态: rejected

重新发起
  ├─ 业务状态: 红冲（保持！）
  └─ 审批状态: pending

再次通过
  ├─ 业务状态: 红冲（保持！）
  └─ 审批状态: approved
```

**关键点**：
- ✅ 业务状态一旦变成"红冲"就永久保持
- ✅ 审批状态可以在 pending/approved/rejected 之间流转
- ✅ 业务状态和审批状态完全独立

### 1. 数据库 - 添加审批状态字段

#### 迁移文件
`database/migrations/2025_11_03_000004_add_approval_status_to_invoice_applications.php`

```php
public function up(): void
{
    Schema::table('invoice_applications', function (Blueprint $table) {
        // 添加审批状态字段
        if (!Schema::hasColumn('invoice_applications', 'approval_status')) {
            $table->string('approval_status', 20)
                ->nullable()
                ->after('status')
                ->comment('审批状态：pending-审批中，approved-已通过，rejected-已驳回');
        }
    });
    
    // 数据迁移：将现有的审批状态从 status 迁移到 approval_status
    DB::statement("
        UPDATE invoice_applications 
        SET approval_status = CASE 
            WHEN status = 'pending' THEN 'pending'
            WHEN status = 'approved' THEN 'approved'
            WHEN status = 'rejected' THEN 'rejected'
            ELSE NULL
        END
        WHERE status IN ('pending', 'approved', 'rejected')
    ");
    
    // 将 status 字段的审批状态值改为业务状态
    DB::statement("
        UPDATE invoice_applications 
        SET status = CASE 
            WHEN status IN ('pending', 'approved') THEN 'normal'
            WHEN status = 'rejected' THEN 'red_flushed'
            ELSE status
        END
        WHERE status IN ('pending', 'approved', 'rejected')
    ");
}
```

**数据迁移说明**：
- 将现有的 `status='pending/approved/rejected'` 迁移到 `approval_status`
- 将 `status` 统一改为业务状态（`normal` 或 `red_flushed`）

### 2. 模型 - 定义新状态常量和访问器

#### 文件：`app/Models/InvoiceApplication.php`

**状态常量**：
```php
// 业务状态常量（只有2种）
const STATUS_NORMAL = 'normal';            // 正常
const STATUS_RED_FLUSHED = 'red_flushed';  // 红冲

// 审批状态常量
const APPROVAL_STATUS_PENDING = 'pending';    // 审批中
const APPROVAL_STATUS_APPROVED = 'approved';  // 已通过
const APPROVAL_STATUS_REJECTED = 'rejected';  // 已驳回
```

**添加字段**：
```php
protected $fillable = [
    // ...
    'status',           // 业务状态
    'approval_status',  // 审批状态
    // ...
];

protected $appends = [
    'status_text',           // 业务状态文本
    'approval_status_text',  // 审批状态文本
    'can_resubmit',
];
```

**状态文本访问器**：
```php
/**
 * 获取业务状态文本
 */
public function getStatusTextAttribute()
{
    $statusMap = [
        self::STATUS_NORMAL => '正常',
        self::STATUS_RED_FLUSHED => '红冲',
    ];
    return $statusMap[$this->status] ?? $this->status;
}

/**
 * 获取审批状态文本
 */
public function getApprovalStatusTextAttribute()
{
    if (!$this->approval_status) {
        return '-';
    }
    
    $statusMap = [
        self::APPROVAL_STATUS_PENDING => '审批中',
        self::APPROVAL_STATUS_APPROVED => '已通过',
        self::APPROVAL_STATUS_REJECTED => '已驳回',
    ];
    return $statusMap[$this->approval_status] ?? $this->approval_status;
}
```

**权限判断（基于审批状态）**：
```php
/**
 * 是否可以编辑（基于审批状态判断）
 */
public function canEdit()
{
    // 没有审批状态（未提交）或已驳回时可以编辑
    return !$this->approval_status || $this->approval_status === self::APPROVAL_STATUS_REJECTED;
}

/**
 * 是否可以提交审批（基于审批状态判断）
 */
public function canSubmit()
{
    return (!$this->approval_status || $this->approval_status === self::APPROVAL_STATUS_REJECTED) && 
           $this->items()->count() > 0 &&
           !empty($this->attachments);
}

/**
 * 是否可以重新提交（重新发起）
 */
public function canResubmit()
{
    // 业务状态是红冲 且 审批状态是已驳回
    return $this->status === self::STATUS_RED_FLUSHED && 
           $this->approval_status === self::APPROVAL_STATUS_REJECTED;
}
```

### 4. 审批服务 - 区分业务状态和审批状态更新

#### 文件：`app/Services/ApprovalService.php`

```php
case '发票申请':
case '发票申请（重新提交）':
    $invoiceApplication = \App\Models\InvoiceApplication::find($businessId);
    if ($invoiceApplication) {
        if ($status === 'completed') {
            // 审批通过：审批状态=已通过，业务状态保持不变
            $invoiceApplication->update([
                'approval_status' => 'approved',
            ]);
        } elseif ($status === 'rejected') {
            // 驳回：审批状态=已驳回，业务状态=红冲
            $invoiceApplication->update([
                'approval_status' => 'rejected',
                'status' => 'red_flushed',  // 业务状态改为红冲
            ]);
        } elseif ($status === 'in_approval') {
            // 审批中：审批状态=审批中，业务状态保持不变
            $invoiceApplication->update([
                'approval_status' => 'pending',
            ]);
        }
    }
    break;
```

**关键逻辑**：
- ✅ 审批通过：只更新审批状态，业务状态不变
- ✅ 审批驳回：更新审批状态 + 业务状态改为红冲
- ✅ 审批中：只更新审批状态
