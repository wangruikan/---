# ✅ 签名功能最终方案 - 图片显示

## 方案思路

**签名完成后直接显示图片，不再用Canvas显示！**

### 核心逻辑

1. **未签名时** → 显示Canvas，让用户签名
2. **签名完成后** → 显示图片预览（img标签）
3. **再次进入时** → 如果已有签名，直接显示图片
4. **重新签名** → 清除图片，切换回Canvas

## 完整实现

### 1. 模板结构

```vue
<view class="signature-wrapper">
    <text class="label">本人签名：</text>
    
    <!-- 未签名：显示Canvas -->
    <view v-if="!formData.signature" class="signature-canvas-box">
        <canvas 
            canvas-id="signatureCanvas"
            :style="{ width: canvasWidth + 'px', height: canvasHeight + 'px' }"
            @touchstart="handleTouchStart"
            @touchmove="handleTouchMove"
            @touchend="handleTouchEnd"
        ></canvas>
        <view class="canvas-tips">请在上方空白区域手写签名</view>
        <view class="signature-buttons">
            <button @click="clearSignature">清除重写</button>
            <button @click="finishSignature" v-if="hasSignature">完成签名</button>
        </view>
    </view>
    
    <!-- 已签名：显示图片 -->
    <view v-else class="signature-preview-box">
        <image :src="formData.signature" mode="aspectFit" class="signature-image"></image>
        <button @click="clearSignature">重新签名</button>
    </view>
</view>
```

**关键点：**
- ✅ 用`v-if="!formData.signature"`判断是否已签名
- ✅ 未签名显示Canvas
- ✅ 已签名显示图片

### 2. 数据定义

```javascript
data() {
    return {
        formData: {
            signature: ''  // base64签名图片
        },
        signatureCtx: null,
        isDrawing: false,
        lastPoint: null,
        hasSignature: false,
        canvasWidth: 0,
        canvasHeight: 0
    }
}
```

### 3. 初始化

```javascript
onLoad() {
    // 计算Canvas尺寸
    const systemInfo = uni.getSystemInfoSync()
    this.canvasWidth = systemInfo.windowWidth - 60
    this.canvasHeight = 200
    
    // 初始化Canvas
    this.$nextTick(() => {
        this.initSignatureCanvas()
    })
    
    // 加载数据
    this.loadSavedData()
}
```

### 4. 加载数据

```javascript
async loadSavedData() {
    const res = await getMyOnboardingForm()
    if (res.success && res.data) {
        this.formData = { ...this.formData, ...res.data }
        
        // 如果有签名，标记状态（图片会通过v-else自动显示）
        if (res.data.signature) {
            this.hasSignature = true
        }
    }
}
```

**关键点：**
- ✅ 不需要恢复到Canvas
- ✅ 有签名时v-else会自动显示图片

### 5. 完成签名

```javascript
finishSignature() {
    if (!this.hasSignature) {
        uni.showToast({ title: '请先签名', icon: 'none' })
        return
    }
    
    // 保存签名为base64
    this.saveSignature((success) => {
        if (success) {
            uni.showToast({ title: '签名已保存', icon: 'success' })
            // formData.signature有值后，v-if会切换到图片显示
        } else {
            uni.showToast({ title: '签名保存失败', icon: 'none' })
        }
    })
}

saveSignature(callback) {
    uni.canvasToTempFilePath({
        canvasId: 'signatureCanvas',
        width: this.canvasWidth,
        height: this.canvasHeight,
        destWidth: this.canvasWidth * 2,
        destHeight: this.canvasHeight * 2,
        success: (res) => {
            uni.getFileSystemManager().readFile({
                filePath: res.tempFilePath,
                encoding: 'base64',
                success: (data) => {
                    // 保存base64到formData
                    this.formData.signature = 'data:image/png;base64,' + data.data
                    if (callback) callback(true)
                }
            })
        }
    }, this)
}
```

**关键点：**
- ✅ 保存成功后`formData.signature`有值
- ✅ v-if自动切换到图片显示

### 6. 清除签名

```javascript
clearSignature() {
    // 清除签名数据
    this.formData.signature = ''
    this.isDrawing = false
    this.lastPoint = null
    this.hasSignature = false
    
    // v-if切换到Canvas后，清空Canvas
    this.$nextTick(() => {
        if (this.signatureCtx) {
            this.signatureCtx.clearRect(0, 0, this.canvasWidth, this.canvasHeight)
            this.signatureCtx.draw(false)
            
            // 重新设置绘制样式
            this.signatureCtx.setStrokeStyle('#000000')
            this.signatureCtx.setLineWidth(3)
            this.signatureCtx.setLineCap('round')
            this.signatureCtx.setLineJoin('round')
        } else {
            this.initSignatureCanvas()
        }
    })
    
    uni.showToast({ title: '已清除，请重新签名', icon: 'none' })
}
```

**关键点：**
- ✅ 清除`formData.signature`
- ✅ v-if切换回Canvas
- ✅ 用`$nextTick`等待DOM更新后清空Canvas

## 完整流程

### 首次签名流程

```
1. 进入页面
   ↓
2. formData.signature = ''
   ↓
3. v-if显示Canvas
   ↓
4. 用户在Canvas上签名（多笔）
   ↓
5. hasSignature = true，显示"完成签名"按钮
   ↓
6. 点击"完成签名"
   ↓
7. Canvas导出为base64
   ↓
8. formData.signature = base64数据
   ↓
9. v-if切换，显示图片预览
   ↓
10. 提交表单（signature一起提交）
```

### 再次进入流程

```
1. 进入页面
   ↓
2. 加载数据
   ↓
3. formData.signature有值（从后端加载）
   ↓
4. v-if判断，显示图片预览
   ↓
5. 用户看到之前的签名
```

### 重新签名流程

```
1. 当前显示图片预览
   ↓
2. 点击"重新签名"按钮
   ↓
3. formData.signature = ''
   ↓
4. v-if切换，显示Canvas
   ↓
5. $nextTick后清空Canvas
   ↓
6. 用户可以重新签名
```

## 方案优势

### 与之前方案对比

| 特性 | 旧方案（Canvas恢复） | 新方案（图片显示） |
|------|---------------------|-------------------|
| 显示方式 | 恢复到Canvas | 直接显示图片 |
| 复杂度 | ❌ 复杂 | ✅ 简单 |
| 可靠性 | ❌ 不稳定 | ✅ 稳定 |
| Canvas尺寸 | ❌ 需要匹配 | ✅ 无关 |
| 恢复逻辑 | ❌ 需要drawImage | ✅ 不需要 |
| 兼容性 | ❌ 可能有问题 | ✅ 完美 |

### 新方案的优势

1. **简单直观**
   - 签名完成后就是图片
   - 不需要复杂的Canvas恢复逻辑

2. **可靠稳定**
   - 图片显示不依赖Canvas
   - 不会有尺寸匹配问题

3. **性能更好**
   - 不需要每次进入都绘制Canvas
   - 图片显示更快

4. **代码更少**
   - 删除了`restoreSignatureToCanvas`方法
   - 简化了`loadSavedData`逻辑

5. **符合直觉**
   - 用户写完签名看到的就是图片
   - 就像拍照后预览照片一样自然

## 核心代码变更

### 删除的代码

```javascript
// ❌ 不再需要
restoreSignatureToCanvas(signatureBase64) {
    this.signatureCtx.drawImage(signatureBase64, 0, 0, width, height)
    this.signatureCtx.draw(false)
}
```

### 简化的代码

```javascript
// 之前
if (data.signature) {
    this.hasSignature = true
    this.$nextTick(() => {
        this.restoreSignatureToCanvas(data.signature)  // ❌ 复杂
    })
}

// 现在
if (data.signature) {
    this.hasSignature = true  // ✅ 简单，图片自动显示
}
```
