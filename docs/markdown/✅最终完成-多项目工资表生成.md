# ✅ 多项目工资表生成 - 最终完成

## 🎯 功能说明

选择多个项目生成工资表时，所有员工合并到**一条工资表记录**中。

## 🔧 修复的所有问题

### 1. ❌ `Column not found: 'project_id'`
**原因**：`employees` 表没有 `project_id` 字段，而是 `project_ids`（JSON 数组）

**解决**：使用 `whereJsonContains` 查询

### 2. ❌ `Column not found: 'status'`
**原因**：`employees` 表没有 `status` 字段（`status` 在中间表 `employee_projects` 中）

**解决**：去掉 `status` 过滤条件

### 3. ❌ `Undefined variable $projectId`
**原因**：重构后变量名改为 `$projectIds`（数组），但某些地方还在使用旧的 `$projectId`

**解决**：使用 `$mainProjectId`（第一个选中的项目ID）

## 📝 最终代码

### 获取员工
```php
// 获取所有项目的员工（去重）
// project_ids 是 JSON 数组字段，需要使用 JSON 查询
$allEmployees = Employee::where('account_set_id', $accountSetId)
    ->where(function($query) use ($projectIds) {
        foreach ($projectIds as $projectId) {
            $query->orWhereJsonContains('project_ids', $projectId);
        }
    })
    ->get()
    ->unique('id');
```

### 计算考勤

// 使用主项目ID获取考勤数据
$attendanceData = $this->calculateGrossSalaryWithAttendance(

## 🎯 核心逻辑

```
用户选择：[项目A, 项目B, 项目C]
          ↓
主项目ID：项目A (第一个)
          ↓
查询员工：project_ids 包含 A 或 B 或 C
          ↓
去重：同一员工只处理一次
          ↓
考勤数据：使用主项目A的考勤表
          ↓
部门名称：员工属于 A、B → "项目A、项目B"
          ↓
创建记录：所有记录的 project_id = 项目A
          ↓
结果：一条工资表，包含所有项目的员工
```
