## 🐛 常见问题排查

### 问题1：字段添加失败
**现象**：执行SQL时报错

**排查步骤**：
1. 检查是否选择了正确的数据库
2. 检查字段是否已存在（可能之前已添加）
3. 检查数据库用户权限

**解决方案**：
```sql
-- 查看字段是否存在
SHOW COLUMNS FROM `projects` LIKE 'require_attendance';

-- 如果不存在，手动添加
ALTER TABLE `projects` 
ADD COLUMN `require_attendance` TINYINT(1) NOT NULL DEFAULT 1 
COMMENT '是否需要考勤：1-需要，0-不需要' 
AFTER `status`;
```

---

### 问题2：前端不显示新字段
**现象**：项目设置中看不到"需要考勤表"开关

**排查步骤**：
1. 检查前端代码是否有 `requires_attendance` 字段（注意是复数形式）
2. 清除浏览器缓存
3. 重启前端开发服务器

**解决方案**：
```bash
# 清除前端缓存
Ctrl + Shift + Delete（浏览器）

# 重启开发服务器
npm run dev
```

---

### 问题3：设置不生效
**现象**：修改了项目设置，但工资表生成逻辑没有变化

**排查步骤**：
1. 检查后端是否正确读取了 `require_attendance` 字段
2. 清除后端缓存
3. 检查浏览器控制台是否有错误

**解决方案**：
```bash
# 清除后端缓存
cd backend
php artisan config:clear
php artisan cache:clear
php artisan route:clear
```

---

### 问题4：提示信息不准确
**现象**：项目数量统计不正确

**排查步骤**：
1. 检查后端API返回的数据
2. 打开浏览器开发者工具 → Network
3. 查看 `/api/payroll/projects-with-approval-status` 接口返回

**解决方案**：
- 检查 `PayrollController.php` 中的 `getProjectsWithApprovalStatus` 方法
- 确认 `can_create_count` 字段计算正确
