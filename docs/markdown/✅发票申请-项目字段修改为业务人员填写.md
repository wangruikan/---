**添加更新方法**：

```javascript
// 更新项目名称
const handleUpdateProjectName = async () => {
  try {
    if (!currentApplication.value.project_name || !currentApplication.value.project_name.trim()) {
      ElMessage.warning('项目名称不能为空')
      await loadApplicationDetail(currentApplication.value.id)
      return
    }

    const response = await request({
      url: `/invoice-applications/${currentApplication.value.id}/update-project`,
      method: 'put',
      data: {
        project_name: currentApplication.value.project_name
      }
    })

    if (response.success) {
      ElMessage.success('项目名称更新成功')
    }
  } catch (error) {
    console.error('更新项目名称失败', error)
    ElMessage.error(error.response?.data?.message || '更新失败')
    await loadApplicationDetail(currentApplication.value.id)
  }
}
```

### 4. 后端 - 添加更新项目名称接口

#### 文件：`app/Http/Controllers/InvoiceApplicationController.php`

```php
/**
 * 更新项目名称
 */
public function updateProject($id, Request $request)
{
    $application = InvoiceApplication::find($id);

    if (!$application) {
        return response()->json([
            'success' => false,
            'message' => '申请不存在'
        ], 404);
    }

    // 只有草稿或已驳回状态可以修改
    if (!$application->canEdit()) {
        return response()->json([
            'success' => false,
            'message' => '当前状态不允许修改'
        ], 400);
    }

    $validator = Validator::make($request->all(), [
        'project_name' => 'required|string|max:100',
    ], [
        'project_name.required' => '项目名称不能为空',
        'project_name.max' => '项目名称不能超过100个字符',
    ]);

    if ($validator->fails()) {
        return response()->json([
            'success' => false,
            'message' => '验证失败',
            'errors' => $validator->errors()
        ], 422);
    }

    $application->update([
        'project_name' => $request->input('project_name')
    ]);

    return response()->json([
        'success' => true,
        'message' => '更新成功',
        'data' => $application
    ]);
}
```

## 📊 业务流程

### 原流程
1. ✅ **审批人**创建开票任务 → 填写项目名称
2. **业务人员**编辑发票申请 → 添加明细项和附件
3. **业务人员**提交审批

### 新流程
1. ✅ **审批人**创建开票任务 → 只填写任务名称、年份、月份
2. ✅ **业务人员**编辑发票申请 → **填写项目名称** + 添加明细项和附件
3. ✅ **业务人员**提交审批 → 系统验证项目名称必填

## 🎯 功能特点

### 1. 编辑权限控制

| 状态 | 可编辑项目名称 | 显示方式 |
|------|-------------|---------|
| 草稿 | ✅ 可以 | 输入框 |
| 已驳回 | ✅ 可以 | 输入框 |
| 审批中 | ❌ 不可以 | 只读文本 |
| 已通过 | ❌ 不可以 | 只读文本 |
| 红冲 | ❌ 不可以 | 只读文本 |

### 2. 实时保存

- 用户在输入框中输入项目名称
- 失去焦点（`@blur`）时自动保存
- 保存成功提示"项目名称更新成功"
- 保存失败自动恢复原值

### 3. 提交验证

提交审批时系统会验证：
- ✅ 明细项不为空
- ✅ 附件已上传
- ✅ **项目名称已填写**（新增）

如果项目名称为空，提示："请先填写项目名称"

## 💡 设计说明

### 为什么这样设计？

1. **职责分离**
   - **审批人**：创建任务框架（时间、名称）
   - **业务人员**：填充具体业务信息（项目、明细、附件）

2. **实时保存**
   - 失去焦点自动保存，避免数据丢失
   - 不需要额外的"保存"按钮，用户体验更流畅

3. **状态控制**
   - 只有草稿和驳回状态可编辑
   - 提交后锁定，保证数据一致性

4. **提交验证**
   - 确保项目名称必填
   - 生成Excel时保证有正确的项目信息
