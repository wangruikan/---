# 商业险退保功能改造说明

## 📋 改造内容

### 原来的逻辑（已移除）
- ❌ 员工离职 → 减少参保 → 自动创建退保记录
- ❌ 退保记录关联员工

### 新的逻辑
- ✅ 管理员/超级管理员手动创建退保记录
- ✅ 创建时选择：保单（商业险）+ 项目
- ✅ 后续流程不变：业务上传保单页 → 财务上传回单

## 📝 使用流程

### 管理员创建退保
1. 进入"商业险退保管理"页面
2. 点击"创建退保"按钮（只有管理员和超级管理员可见）
3. 选择要退保的商业险保单
4. 选择关联的项目
5. 点击"确定创建"

### 业务人员处理
1. 查看待业务处理的退保记录
2. 上传退保保单页
3. 填写退保金额
4. 提交给财务

### 财务人员处理
1. 查看待财务处理的退保记录
2. 上传收款回单
3. 提交完成

## ⚠️ 注意事项

1. **去重逻辑**：同一账套下，同一保单在同一项目中只能有一条未完成的退保记录
2. **保单类型限制**：只能为"商业险"类型的保单创建退保记录
3. **权限要求**：只有管理员（admin）和超级管理员（super_admin）可以创建退保
4. **数据兼容性**：旧的退保记录（有employee_id的）仍然可以正常查看和处理

## 🔄 数据迁移

如果有旧的退保记录需要处理，它们仍然可以正常使用。新创建的退保记录 `employee_id` 和 `insurance_change_id` 字段为 `NULL`。
