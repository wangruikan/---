# 人员汇总申请功能 - 完整部署说明

## 📋 功能概述

当导入应发工资Excel时，系统会自动对比人员差异：
- ✅ **智能项目识别**：多项目工资表自动识别Excel对应的项目
- ✅ **人员差异检测**：自动对比工资表和Excel的人员差异
- ✅ **自动记录变动**：将新增/减少的人员记录到人员汇总申请
- ✅ **流程保护**：已发起流程的记录不能被覆盖
- ✅ **跳过第一节点**：提交时自动跳过经办，直接进入第二节点审批

## 🔧 核心业务逻辑

### 1. Excel导入流程

```
导入Excel
  ↓
读取第2行标题（识别"身份证号码"和"应发工资"列）
  ↓
读取第3行起的数据
  ↓
智能识别项目（通过身份证号查找员工所属项目）
  ↓
对比人员差异
  ├─ Excel有但工资表没有 → 创建"新增"记录
  └─ 工资表有但Excel没有 → 创建"减少"记录
  ↓
更新匹配人员的应发工资
  ↓
返回导入结果
```

### 2. 智能项目识别逻辑

```php
// 遍历Excel中的每个身份证号
foreach (Excel中的身份证) {
    // 在系统中查找员工
    员工 = Employee::where('id_number', 身份证号)->first();
    
    if (员工存在 && 员工有项目) {
        // 找到第一个匹配的员工后，返回其第一个项目ID
        return 员工->project_ids[0];
    }
}
```

### 3. 人员差异对比

| 情况 | Excel人数 | 工资表人数 | 系统行为 |
|------|----------|-----------|---------|
| 正常 | 5人 | 5人 | 直接更新应发工资 |
| 需要新增 | 7人 | 5人 | 创建"新增"记录（2人），提示维护人员档案 |
| 需要减少 | 3人 | 5人 | 创建"减少"记录（2人），提示维护人员档案 |
| 混合变动 | 6人（a1,a2,a3,a4,a5,a6） | 5人（a1,a2,a3,b1,b2） | 新增记录（a4,a5,a6），减少记录（b1,b2） |

### 4. 记录覆盖规则

| 现有记录状态 | 是否可覆盖 | 行为 |
|-------------|-----------|------|
| 不存在 | ✅ | 创建新记录 |
| pending（待审批） | ✅ | 更新人员列表，重置状态 |
| in_approval（审批中，无流程ID） | ✅ | 更新人员列表，重置状态 |
| in_approval（审批中，有流程ID） | ❌ | 提示"已发起流程，请先驳回" |
| approved（已通过） | ❌ | 不处理 |
| rejected（已拒绝） | ❌ | 不处理 |

## 🚀 使用流程

### 场景1：Excel人员比工资表多（需要新增）

**示例：**
- 工资表有：张三、李四（2人）
- Excel有：张三、李四、王五（3人）

**系统行为：**
1. ✅ 更新张三、李四的应发工资
2. ⚠️ 发现王五不在工资表中
3. 📝 自动创建"人员新增"记录（包含王五的身份证和姓名）
4. 💬 提示：Excel中有1个人员在工资表中不存在，已记录到人员汇总申请

**后续操作：**
1. 到"人员汇总申请"查看记录
2. 上传相关附件（如入职通知、劳动合同等）
3. 点击"提交审批"
4. 审批通过后，到员工管理添加该员工

### 场景2：Excel人员比工资表少（需要减少）

**示例：**
- 工资表有：张三、李四、王五（3人）
- Excel有：张三、李四（2人）

**系统行为：**
1. ✅ 更新张三、李四的应发工资
2. ⚠️ 发现王五在Excel中不存在
3. 📝 自动创建"人员减少"记录（包含王五的身份证和姓名）
4. 💬 提示：工资表中有1个人员在Excel中不存在，已记录到人员汇总申请

**后续操作：**
1. 到"人员汇总申请"查看记录
2. 上传相关附件（如离职申请、解除劳动合同证明等）
3. 点击"提交审批"
4. 审批通过后，到员工管理删除或归档该员工

### 场景3：多项目工资表导入

**示例：**
- 生成工资表时选择了3个项目（公园项目、老年公寓、幼儿园）
- 现在导入一个Excel，里面的员工来自"公园项目"

**系统行为：**
1. 🔍 自动从Excel的身份证号识别项目
2. 📌 找到Excel中的第一个有效员工
3. 🔎 查询该员工所属项目 → 公园项目
4. ✅ 仅更新"公园项目"的员工工资
5. 📊 仅与"公园项目"的工资表人员进行对比

### 场景4：尝试覆盖已发起流程的记录

**情况：**
- 上次导入时创建了"人员新增"记录
- 该记录已提交审批（status = in_approval）

**系统行为：**
1. ❌ 检测到记录已发起流程
2. 🛑 拒绝覆盖
3. 💬 提示："该项目2025-11月份的人员新增记录已发起流程申请，请先驳回后再导入"

**解决方法：**
- 到"审批管理"中找到该审批
- 驳回审批
- 重新导入Excel
