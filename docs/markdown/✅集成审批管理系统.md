# ✅ 发票申请集成审批管理系统

## ❌ 原问题

1. **审批信息不显示**：提交后看不到审批信息
2. **待办列表无数据**：第二个审批人的待办里没有发票申请
3. **原因**：使用了错误的模型 `ProcessApproval`，没有集成到现有的审批管理系统

---

## ✅ 解决方案

### 修改内容

#### 1. 使用正确的审批系统模型

**修改前（错误）**：
```php
// 使用 ProcessApproval（这是独立的模型，不在审批系统中）
$approvalInstance = ProcessApproval::create([
    'account_set_id' => $application->account_set_id,
    'initiator_id' => $application->submitter_id,
    // ...
]);
```

**修改后（正确）**：
```php
// 使用 ApprovalInstance（现有审批系统的模型）
$instance = \App\Models\ApprovalInstance::create([
    'account_set_id' => $accountSetId,
    'business_type' => '发票申请',
    'business_id' => $application->id,
    'current_step' => 2,
    'total_steps' => $approvers->count() + 1,
    'status' => 'pending',
    'created_by' => $user->id,
]);
```

---

#### 2. 创建审批节点记录

**新增逻辑**：
```php
// 获取审批人配置（第2、3、4节点）
$approvers = \App\Models\User::where('account_set_id', $accountSetId)
    ->where('status', 'active')
    ->whereIn('approval_level', [2, 3, 4])
    ->orderBy('approval_level')
    ->get();

// 为每个审批人创建审批记录
foreach ($approvers as $index => $approver) {
    \App\Models\ApprovalRecord::create([
        'approval_instance_id' => $instance->id,
        'step' => $index + 2, // 从第2步开始
        'approver_id' => $approver->id,
        'status' => $index === 0 ? 'pending' : 'waiting', // 第一个人pending，其他waiting
    ]);
}
```

---

#### 3. 添加附件到审批系统

**新增逻辑**：
```php
// 将发票附件添加到审批系统
if ($application->attachments) {
    $attachments = is_array($application->attachments) 
        ? $application->attachments 
        : json_decode($application->attachments, true);
        
    if (is_array($attachments)) {
        foreach ($attachments as $attachment) {
            \App\Models\ApprovalAttachment::create([
                'approval_instance_id' => $instance->id,
                'file_name' => $attachment['filename'] ?? '',
                'file_path' => $attachment['path'] ?? '',
                'file_size' => $attachment['size'] ?? 0,
                'uploaded_by' => $user->id,
            ]);
        }
    }
}
```

## 📊 审批系统架构

### 模型关系
```
ApprovalInstance（审批实例）
├── business_type: "发票申请"
├── business_id: 发票申请ID
├── current_step: 当前审批步骤
├── total_steps: 总步骤数
└── status: pending/approved/rejected

ApprovalRecord（审批记录） x N
├── approval_instance_id: 关联审批实例
├── step: 审批步骤（2, 3, 4）
├── approver_id: 审批人ID
└── status: pending/approved/rejected/waiting

ApprovalAttachment（审批附件） x N
├── approval_instance_id: 关联审批实例
├── file_name: 文件名
├── file_path: 文件路径
└── file_size: 文件大小
```

## 🔄 审批流程

### 1. 提交审批
```
经办人提交
    ↓
创建 ApprovalInstance（审批实例）
    ↓
创建 ApprovalRecord（审批记录）
    - 第2节点审批人: status = 'pending'
    - 第3节点审批人: status = 'waiting'
    - 第4节点审批人: status = 'waiting'
    ↓
创建 ApprovalAttachment（审批附件）
    ↓
第2节点审批人的待办列表显示 ✅
```

### 2. 审批流转
```
第2节点审批人审批通过
    ↓
更新 ApprovalRecord
    - 第2节点: status = 'approved'
    - 第3节点: status = 'pending'  ← 变为待审批
    ↓
第3节点审批人的待办列表显示 ✅
    ↓
依次类推...
```

## 🔗 与现有系统的集成

### 集成点

1. **审批待办**：`ApprovalRecord` 中 `status = 'pending'` 的记录会显示在待办列表
2. **审批详情**：通过 `ApprovalInstance` 和 `ApprovalAttachment` 查看附件
3. **审批流转**：审批通过后自动流转到下一个节点
4. **审批历史**：通过 `ApprovalRecord` 查看审批历史

- ✅ 工资申请

## ⚠️ 注意事项

1. **审批人配置**：确保账套中有配置第2、3、4节点的审批人
2. **审批级别**：用户表的 `approval_level` 字段需要正确设置（2, 3, 4）
3. **用户状态**：审批人的 `status` 必须是 'active'
4. **跳过第1节点**：发票申请由经办人发起，直接从第2节点开始审批
