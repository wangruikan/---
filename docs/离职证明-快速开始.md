## 📋 功能说明

### 适用人员
- 离职员工（contract_status = 'terminated'）
- 退休员工（contract_status = 'retired'）

### 上传要求
- 文件格式：JPG、PNG、PDF
- 文件大小：单个文件最大 10MB
- 文件数量：支持多个文件

### 审批验证
创建工资表并提交审批时，系统会自动检查：
- 工资表中是否有离职/退休员工
- 这些员工是否已上传离职证明
- 如果未上传，将阻止提交并显示缺失员工列表

## 🔗 API 接口

所有接口都已实现并可以直接使用：

```
GET    /api/employees/{employeeId}/resignation-certificates
       获取员工的离职证明列表

POST   /api/employees/{employeeId}/resignation-certificates/upload
       上传离职证明（支持多文件）

DELETE /api/employees/resignation-certificates/{id}
       删除离职证明

GET    /api/employees/resignation-certificates/{id}/download
       下载离职证明
```

## ❓ 常见问题

### Q1: 为什么看不到"离职证明"标签页？
**A:** 只有离职（terminated）或退休（retired）状态的员工才会显示此标签页。

### Q2: 上传文件时提示"文件太大"？
**A:** 单个文件最大 10MB，请压缩文件或分多次上传。

### Q3: 支持哪些文件格式？
**A:** 支持 JPG、PNG、PDF 格式。

### Q4: 可以上传多个文件吗？
**A:** 可以，支持多文件上传。

### Q5: 工资表提交失败，提示"未上传离职证明"？
**A:** 工资表中有离职/退休员工未上传离职证明，请先为这些员工上传离职证明。

### Q6: 员工可以删除自己上传的离职证明吗？
**A:** 可以，员工和管理员都可以删除离职证明。

### Q7: 小程序端什么时候可以使用？
**A:** 小程序端需要前端开发人员实现，已提供完整代码示例，预计2-3小时可以完成开发。
