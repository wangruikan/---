# ✅ 保险数据调试指南

## ❌ 问题

工资明细中的保险金额显示为 `¥0.00` 或 `-`，无法正确显示保险明细。

---

## 🔍 可能原因

### 1. **员工没有参保记录**
- 员工未在"参保明细"中添加参保信息
- `insurance_personnel` 表中没有该员工的记录

### 2. **员工基数为0**
- 员工的 `social_security_base`（社保基数）为 0
- 员工的 `housing_fund_base`（公积金基数）为 0

### 3. **参保配置为空**
- 员工的 `social_security_types`（社保险种配置）为空
- 员工的 `housing_fund_config_id`（公积金配置ID）为空

### 4. **参保状态不是active**
- `insurance_personnel` 表中的 `status` 不是 'active'

---

**检查点**：
- ✅ 应该有记录（不为空）

- ✅ `social_security_types` 不为 NULL（是JSON格式）
- ✅ `housing_fund_config_id` 不为 NULL（是数字）


**检查点**：
- ✅ 应该有记录
- ✅ `company_ratio` 和 `personal_ratio` 应该 > 0

## ✅ 解决方案

### 方案1：添加参保记录

如果员工没有参保记录，需要在系统中添加：

```
1. 进入"保险管理" → "参保明细"
2. 找到该员工
3. 点击"新增参保"
4. 填写：
   - 社保险种（选择养老、医疗、失业等）
   - 公积金配置（选择配置）
   - 状态：选择"在职"
5. 保存
```

### 方案2：设置员工基数

如果员工基数为0，需要设置：

```sql
-- 更新员工基数
UPDATE employees 
SET social_security_base = 7947,
    housing_fund_base = 7947,
    large_medical_base = 0
WHERE name = 'wrk';
```

或在系统中设置：
```
1. 进入"人员档案管理"
2. 编辑该员工
3. 填写：
   - 社保基数：7947
   - 公积金基数：7947
4. 保存
```

## 💻 后端计算逻辑

### SalaryController::calculateMonthlyInsurance()

```php
private function calculateMonthlyInsurance($employee, $month)
{
    // 1. 查找参保记录
    $insurancePersonnel = InsurancePersonnel::where('employee_id', $employee->id)
        ->where('status', 'active')
        ->first();

    if (!$insurancePersonnel) {
        // ❌ 没有参保记录，返回空数据
        return ['details' => ['social_security' => [], 'housing_fund' => null, 'large_medical' => null]];
    }

    // 2. 计算社保
    if ($insurancePersonnel->social_security_types) {
        $socialSecurityTypes = json_decode($insurancePersonnel->social_security_types, true);
        foreach ($socialSecurityTypes as $typeConfig) {
            $base = $employee->social_security_base ?? 0; // ✅ 需要有基数
            $companyRatio = $typeConfig['company_ratio'] ?? 0; // ✅ 需要有比例
            $companyAmount = $base * $companyRatio;
            // ... 保存明细
        }
    }

    // 3. 计算公积金
    if ($insurancePersonnel->housing_fund_config_id) {
        $housingFundConfig = HousingFundConfig::find($insurancePersonnel->housing_fund_config_id);
        $base = $employee->housing_fund_base ?? 0; // ✅ 需要有基数
        $companyRatio = $housingFundConfig->company_ratio ?? 0; // ✅ 需要有比例
        $companyAmount = $base * $companyRatio;
        // ... 保存明细
    }
}
```

## 📝 常见问题

### Q1: 为什么保险金额全是0？
**A**: 检查：
1. 员工是否有参保记录
2. 员工基数是否 > 0
3. 参保配置的比例是否 > 0

### Q2: 为什么只有社保没有公积金？
**A**: 检查：
1. `insurance_personnel.housing_fund_config_id` 是否为NULL
2. `housing_fund_configs` 表是否有对应记录

### Q3: 如何快速添加测试数据？
