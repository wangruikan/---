# 🔍 补差记录检查步骤

## 📋 问题：工资表没有显示补差记录

### 可能的原因

1. ✅ **数据库中没有补差记录**（最可能）
2. 前端页面缓存
3. 后端代码没有正确调用

---

## 🔍 第一步：检查数据库是否有补差记录

执行 `check_compensation_data.sql`：

### SQL 1：查看所有补差记录

```sql
SELECT 
    id,
    employee_id,
    employee_name,
    compensation_type,
    old_base,
    new_base,
    compensation_start_month,
    compensation_end_month,
    company_total,
    personal_total,
    total_amount
FROM insurance_compensation_records
LIMIT 10;
```

**如果返回 0 条记录：**
说明数据库中**根本没有补差记录**！这就是为什么工资表显示不出来。

**如果有记录，继续下一步：**

### SQL 2：查看 2025-10 月的补差记录

```sql
SELECT 
    id,
    employee_id,
    employee_name,
    compensation_type,
    compensation_start_month,
    compensation_end_month,
    company_total,
    personal_total,
    total_amount
FROM insurance_compensation_records
WHERE compensation_start_month <= '2025-10'
AND compensation_end_month >= '2025-10';
```

**如果返回 0 条记录：**
说明 2025-10 月**没有补差记录**。

## 📝 如果数据库中没有补差记录

### 原因：补差记录需要手动创建

补差记录不是自动生成的，需要通过以下方式创建：

#### 方式 1：通过"基数调整"功能创建

1. 打开"参保管理" → "基数调整"
2. 为员工调整社保或公积金基数
3. 系统会自动计算补差并创建 `insurance_compensation_records` 记录

#### 方式 2：手动在数据库中插入测试数据

```sql

    updated_at
) VALUES (
    1,  -- 账套ID
```

## 🔍 第三步：检查前端显示

### 当前前端只有"补差合计"列

```vue
<el-table-column prop="compensation_total" label="补差合计">
  <template #default="{ row }">
    <span v-if="row.compensation_total > 0">
      {{ formatMoney(row.compensation_total) }}
    </span>
    <span v-else>-</span>
  </template>
</el-table-column>
```

### 需要添加"补差明细"列

补差明细数据在 `row.compensation_details` 中：

```javascript
row.compensation_details = {
  social_security: [...],  // 社保补差明细
  housing_fund: [...]      // 公积金补差明细
}
```
