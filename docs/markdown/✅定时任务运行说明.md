## 📋 定时任务详细说明

### 1️⃣ 生成交付记录任务 (`delivery:generate`)

**执行时间：** 每月1日早上8点

**Cron表达式：** `0 8 1 * *`

**命令：** `php artisan delivery:generate`

**功能：**
生成新的资料交付记录，包括月度和季度交付。

#### 🔄 执行逻辑：

##### **月度交付（Monthly）：**
1. 查询所有 `delivery_cycle = 'monthly'` 且 `is_active = 1` 的配置
2. 为每个配置生成当月的交付记录
   - `delivery_period` = 当前月份（如：`2025-10`）
   - `status` = `'pending'`（待交付）
   - `handler_id` = 第一个审批节点账号（或配置创建人）
   - `config_id` = 配置ID
3. 向经办人发送"新周期提醒"（记录到 `document_delivery_reminders` 表）
4. 记录日志到 Laravel 日志文件

**示例：**
- 10月1日早上8点执行
- 生成 `delivery_period = '2025-10'` 的交付记录
- 提醒经办人：需要在本月内完成10月份的资料交付

##### **季度交付（Quarterly）：**
1. 查询所有 `delivery_cycle = 'quarterly'` 且 `is_active = 1` 的配置
2. 检查每个项目是否需要生成新的季度记录：
   - 如果从未生成过记录 → 立即生成
   - 如果距离上次生成已满3个月 → 生成新季度记录
3. 生成季度交付记录
   - `delivery_period` = 当前季度（如：`2025-Q4`）
   - `status` = `'pending'`
   - 其他字段同月度
4. 发送"新周期提醒"

**示例：**
- 10月1日：属于2025年第4季度（Q4）
- 如果上次生成是7月1日（Q3），距今已3个月 → 生成 `2025-Q4` 记录
- 如果上次生成是8月1日，距今才2个月 → **不生成**，等到11月1日

---

### 2️⃣ 检查未交付任务 (`delivery:check-pending`)

**执行时间：** 每月最后一天晚上20点

**Cron表达式：** `0 20 31 * *`

**命令：** `php artisan delivery:check-pending`

**功能：**
检查当月/当季度是否有未完成的交付记录，发送提醒通知。

#### 🔄 执行逻辑：

##### **月度交付检查：**
1. 查询当月 `delivery_cycle = 'monthly'` 且 `status = 'pending'` 的记录
2. 对每条未交付记录：
   - 向经办人发送"未交付提醒"
   - 防重复：同一天只发送一次提醒
3. 记录日志

**示例：**
- 10月31日晚上20点执行
- 检查 `delivery_period = '2025-10'` 的记录
- 如果状态仍是 `'pending'`（未提交）→ 发送提醒

##### **季度交付检查：**
1. 查询所有 `delivery_cycle = 'quarterly'` 的启用配置
2. 对每个项目：
   - 获取最近的季度交付记录
   - 检查距离生成时间是否已满3个月
   - 如果已满3个月且状态仍是 `'pending'` → 发送提醒
3. 记录日志

**示例：**
- 10月31日晚上20点执行
- 某项目的 `2025-Q4` 记录是10月1日生成的
- 现在距离生成不到3个月 → **暂不提醒**
- 如果是7月1日生成的 `2025-Q3` 记录，距今已4个月 → **发送提醒**

## 📊 数据库表说明

### `project_delivery_configs` - 配置表
- 存储每个项目的交付配置
- 字段：项目、周期、方式、资料清单、是否启用

### `document_deliveries` - 交付记录表
- 存储每月/每季度的交付记录
- 字段：
  - `config_id`：关联的配置ID
  - `handler_id`：经办人ID（负责交付的人）
  - `delivery_period`：交付期间（2025-10 或 2025-Q4）
  - `status`：pending（待交付）/ submitted（已提交）/ completed（已完成）

### `document_delivery_reminders` - 提醒记录表
- 存储发送给经办人的提醒
- 字段：
  - `reminder_type`：new_period（新周期提醒）/ not_submitted（未交付提醒）
  - `recipient_id`：接收人ID
  - `is_read`：是否已读

## ⚠️ 注意事项

### 1. 定时任务需要配置 Cron
Laravel 定时任务需要在服务器上配置 Cron：

**Linux/Mac：**
```bash
crontab -e

# 添加以下行
* * * * * cd /path/to/your/project && php artisan schedule:run >> /dev/null 2>&1
```

**Windows：**
使用"任务计划程序"，每分钟执行一次：
```
php artisan schedule:run
```

### 2. 月底任务的特殊处理
由于不是每个月都有31日，代码中使用了 `when()` 回调：
```php
->when(function () {
    return Carbon::now()->day === Carbon::now()->endOfMonth()->day;
})
```
确保只在真正的月末最后一天执行。

### 3. 时区设置
所有定时任务使用 `Asia/Shanghai` 时区（北京时间）。

### 4. 日志记录
所有定时任务执行情况都会记录到 Laravel 日志：
- 位置：`storage/logs/laravel.log`
- 包括：开始时间、结束时间、生成记录数、错误信息

## 🎯 实际运行示例

### 场景1：月度交付项目
- **10月1日 08:00** - 自动生成 `2025-10` 的交付记录
- **10月31日 20:00** - 检查 `2025-10` 是否已交付，未交付则提醒
- **11月1日 08:00** - 自动生成 `2025-11` 的新记录

### 场景2：季度交付项目
- **10月1日 08:00** - 检查距上次生成是否满3个月
  - 如果是，生成 `2025-Q4` 记录
  - 如果否，跳过
- **10月31日 20:00** - 检查 `2025-Q4` 是否已交付（如果已满3个月）
- **1月1日 08:00** - 生成 `2026-Q1` 记录
