# ✅ 前端计算合计 - 实现说明

## 🎯 需求

合计不从数据库字段读取，而是在**前端直接把左边几个保险的金额相加**。

## ✅ 实现方式

### 修改 1：单位合计

**修改前（从数据库字段读取）：**
```vue
<el-table-column prop="company_insurance_total" label="单位合计">
  <template #default="{ row }">
    {{ formatMoney(row.company_insurance_total) }}
  </template>
</el-table-column>
```

**修改后（前端计算）：**
```vue
<el-table-column label="单位合计">
  <template #default="{ row }">
    {{ formatMoney(calculateCompanyTotal(row)) }}
  </template>
</el-table-column>
```

### 新增方法：计算单位合计

```javascript
// 计算单位合计（所有保险的单位部分相加）
const calculateCompanyTotal = (row) => {
  let total = 0
  
  // 1. 社保单位部分（包括医保）- 遍历所有险种
  if (row.insurance_details && row.insurance_details.social_security) {
    row.insurance_details.social_security.forEach(item => {
      total += parseFloat(item.company_amount || 0)
    })
  }
  
  // 2. 公积金单位部分
  if (row.insurance_details && row.insurance_details.housing_fund) {
    total += parseFloat(row.insurance_details.housing_fund.company_amount || 0)
  }
  
  // 3. 大额医疗单位部分
  if (row.insurance_details && row.insurance_details.large_medical) {
    total += parseFloat(row.insurance_details.large_medical.company_amount || 0)
  }
  
  return total
}
```

// 计算个人合计（所有保险的个人部分相加）

## 📊 计算逻辑

### 单位合计计算公式

```
单位合计 = 
  Σ(所有社保险种的单位金额) +
  Σ(所有医保险种的单位金额) +  // 医保在 social_security 数组中
  公积金单位金额 +
  大额医疗单位金额
```

### 个人合计计算公式

```
个人合计 = 
  Σ(所有社保险种的个人金额) +
  Σ(所有医保险种的个人金额) +  // 医保在 social_security 数组中
  公积金个人金额 +
  大额医疗个人金额
```

### 数据来源

所有金额都来自 `insurance_details` 对象：

```javascript
row.insurance_details = {
  social_security: [
    { name: '养老保险', company_amount: 1009.00, personal_amount: 1009.00 },
    { name: '医保1', company_amount: 599.50, personal_amount: 3501.00 },
    // ... 其他险种
  ],
  housing_fund: {
    name: '住房公积金',
    company_amount: 1200.00,
    personal_amount: 1200.00
  },
  large_medical: {
    name: '大额医疗',
    company_amount: 121.00,
    personal_amount: 232.00
  }
}
```

## ✅ 优势

1. **实时计算**：每次数据更新，合计自动重新计算
2. **无需数据库字段**：不依赖 `company_insurance_total` 和 `personal_insurance_total` 字段
3. **灵活性高**：前端可以随时调整计算逻辑
4. **准确性好**：直接从明细金额相加，不会出错
5. **无需重新生成**：修改代码后刷新页面即可看到效果
