# 付款申请统一显示功能 - 完成说明

## ✅ 功能已完成！

我已经修改了付款申请列表页面，现在可以同时显示工资付款申请和保险汇总付款申请的记录。

## 🎯 实现的功能

### 1. 统一显示多种付款申请

✅ **工资付款申请**
- 从工资表审批通过后发起的付款申请
- 显示项目名称和工资期间
- 显示付款金额（实发工资总和）

✅ **保险汇总付款申请**
- 从汇总流程审批通过后发起的付款申请
- 显示汇总申请标题
- 显示月份信息

✅ **付款类型标签**
- 工资付款申请：蓝色标签
- 保险汇总付款申请：绿色标签

✅ **付款金额显示**
- 工资付款申请：显示实际金额
- 保险汇总付款申请：显示"-"（旧系统没有金额字段）

## 📁 修改的文件

### 1. 后端

**app/Http/Controllers/PaymentApplicationController.php**

**修改内容：** `index()` 方法（第106-236行）

**实现逻辑：**
1. 查询旧的 `PaymentApplication` 表（保险汇总付款申请）
2. 查询新的 `PaymentRequest` 表（工资和保险付款申请）
3. 将两套数据转换为统一格式
4. 合并并按创建时间排序
5. 手动分页返回

**统一数据格式：**
```php
[
    'id' => 申请ID,
    'payment_type' => 'salary' | 'insurance',  // 付款类型
    'type_name' => '工资付款申请' | '保险汇总付款申请',  // 类型名称
    'title' => '申请标题',
    'month' => '月份',
    'amount' => 金额,
    'status' => '状态',
    'initiator' => 发起人对象,
    'attachments' => 附件列表,
    'approval_instance' => 审批实例,
    'submitted_at' => 提交时间,
    // ...
]
```

## 📊 数据来源

### 工资付款申请
- **来源表**：`payment_requests` (payment_type = 'salary')
- **关联**：`salary_approvals` → `projects`
- **金额**：自动计算（所有员工实发工资总和）
- **标题格式**：`工资付款申请 - {项目名称} {月份}`

### 保险汇总付款申请（新系统）
- **来源表**：`payment_requests` (payment_type = 'insurance')
- **关联**：`process_approvals`
- **金额**：自动计算（保险费用总和）
- **标题格式**：`保险汇总付款申请 - {汇总标题}`

### 保险汇总付款申请（旧系统）
- **来源表**：`payment_applications`
- **关联**：`process_approvals`
- **金额**：无（显示"-"）
- **标题格式**：原始标题

## 🔄 工作流程

### 工资付款申请流程

```
工资表生成
    ↓
提交审批
    ↓
审批通过
    ↓
发起付款申请（填表生成Word或上传附件）
    ↓
创建 PaymentRequest 记录（payment_type = 'salary'）
    ↓
显示在"付款申请"列表中
    ↓
审批人审批
    ↓
审批通过/驳回
```

### 保险汇总付款申请流程

```
汇总流程审批通过
    ↓
发起付款申请（填表生成Word或上传附件）
    ↓
创建 PaymentRequest 记录（payment_type = 'insurance'）
    ↓
显示在"付款申请"列表中
    ↓
审批人审批
    ↓
审批通过/驳回
```
