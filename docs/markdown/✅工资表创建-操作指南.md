# ✅ 工资表创建 - 操作指南

## 🔍 问题诊断

您遇到的错误：
```
1054 - Unknown column 'housing_fund' in 'salaries'
```

**说明**：`salaries` 表可能：
1. 不存在
2. 存在但缺少 `housing_fund` 字段

---

## 📋 操作步骤

### 步骤1：检查表是否存在

执行 `check_salaries_table.sql`：

```sql
SHOW TABLES LIKE 'salaries';
```

---

### 步骤2：根据结果选择方案

#### 🆕 方案A：表不存在（推荐）

**执行**：`create_salaries_table.sql`

这个文件会创建完整的表，包含所有字段：
- ✅ 所有基础字段
- ✅ `seq_number`（序号）
- ✅ `department`（部门）
- ✅ `position`（岗位）
- ✅ `company_insurance_total`（单位保险合计）
- ✅ `personal_insurance_total`（个人保险合计）
- ✅ `compensation_total`（补差合计）

**优点**：
- 一次性创建完整表结构
- 字段顺序正确
- 不会有字段引用错误

---

#### 🔧 方案B：表已存在但字段不全

**执行**：`update_salaries_compatible.sql`

这个文件会添加缺少的字段，**不使用 AFTER 子句**，避免引用不存在的字段。

**注意**：
- 如果某个字段已存在，会报错（可以忽略）
- 字段会添加到表的末尾（不影响功能）

---


## 🔄 如果出错怎么办？

### 错误1：字段已存在

```
Duplicate column name 'xxx'
```

**解决**：忽略这个错误，继续执行下一条语句

---

### 错误2：表已存在

```
Table 'salaries' already exists
```

**解决**：使用方案B（`update_salaries_compatible.sql`）

---

### 错误3：引用的字段不存在

```
Unknown column 'xxx' in 'salaries'
```

**解决**：使用 `update_salaries_compatible.sql`（不使用 AFTER 子句）
