# ✅ 找到根本原因了！

## 🎯 问题根源

**Salary 模型的 `$fillable` 数组中没有 `department` 和 `position`！**

即使代码中写了：
```php
$salary = Salary::create([
    'department' => $project->name,
    'position' => $employee->position,
    // ...
]);
```

Laravel 也会**自动忽略这两个字段**，因为它们不在 `$fillable` 白名单中！

## 📊 原理说明

Laravel 的 **Mass Assignment Protection**（批量赋值保护）机制：

1. 只有在 `$fillable` 数组中的字段才能被 `create()` 或 `fill()` 方法批量赋值
2. 如果字段不在 `$fillable` 中，Laravel 会**静默忽略**，不报错也不保存
3. 这是为了防止恶意用户通过表单提交修改敏感字段

**所以必须把 `department` 和 `position` 加到 `$fillable` 中！**
