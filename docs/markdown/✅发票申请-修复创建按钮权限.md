# ✅ 发票申请 - 修复创建按钮权限显示

## 📋 问题

创建开票任务按钮没有显示，权限检查接口调用错误。

### 问题原因

前端代码中使用了错误的权限检查接口 `/check-delivery-config-access`（这是资料交付管理模块的接口），导致权限检查失败，按钮无法显示。

---

## ✅ 解决方案

### 1. 后端 - 添加权限检查接口

#### 文件：`app/Http/Controllers/InvoiceApplicationController.php`

添加权限检查方法：

```php
/**
 * 检查创建发票申请的权限
 * 只有审批级别2、3、4的审批人可以创建
 */
public function checkCreatePermission(Request $request)
{
    $user = Auth::user();
    $accountSetId = $request->input('account_set_id', $user->account_set_id);

    if (!$accountSetId) {
        return response()->json([
            'success' => true,
            'has_access' => false,
            'message' => '未找到账套信息'
        ]);
    }

    // 检查用户是否是该账套的审批人（审批级别 2、3、4）
    $approver = DB::table('account_set_users')
        ->where('account_set_id', $accountSetId)
        ->where('user_id', $user->id)
        ->whereIn('approval_level', [2, 3, 4])
        ->first();

    return response()->json([
        'success' => true,
        'has_access' => $approver !== null,
        'approval_level' => $approver ? $approver->approval_level : null,
        'approval_level_name' => $approver ? $approver->approval_level_name : null
    ]);
}
```

---

### 2. 路由 - 添加权限检查路由

#### 文件：`routes/api.php`

在发票申请管理路由组中添加（必须在 `{id}` 路由之前）：

```php
// 发票申请管理
Route::prefix('invoice-applications')->group(function () {
    // 权限检查（必须在 {id} 路由之前）
    Route::get('/check-permission/create', [InvoiceApplicationController::class, 'checkCreatePermission']); // 检查创建权限
    
    Route::get('/', [InvoiceApplicationController::class, 'index']);
    // ... 其他路由
});
```

**重要说明**：权限检查路由必须放在 `Route::get('/{id}', ...)` 之前，否则 `/check-permission/create` 会被当作 `/{id}` 路由处理。

---

### 3. 前端 - 修改权限检查接口调用

#### 文件：`src/views/InvoiceApplications/index.vue`

修改前：
```javascript
const response = await request({
  url: '/check-delivery-config-access',  // ❌ 错误：这是资料交付的接口
  method: 'get',
  params: { account_set_id: accountSetId }
})
```

修改后：
```javascript
const response = await request({
  url: '/invoice-applications/check-permission/create',  // ✅ 正确：发票申请的权限检查接口
  method: 'get',
  params: { account_set_id: accountSetId }
})
```

## 📊 权限规则

### 发票申请权限体系

| 角色 | 审批级别 | 权限说明 |
|------|---------|---------|
| 经办 | 1 | ❌ 不能创建开票任务 |
| 初审 | 2 | ✅ 可以创建开票任务 |
| 复审 | 3 | ✅ 可以创建开票任务 |
| 终审 | 4 | ✅ 可以创建开票任务 |

### 权限检查逻辑

1. 获取当前用户信息和账套ID
2. 查询 `account_set_users` 表
3. 检查条件：
   - `account_set_id` = 当前账套
   - `user_id` = 当前用户
   - `approval_level` IN (2, 3, 4)
4. 如果存在记录，则有权限创建

  `approval_level_name` varchar(50) COMMENT '审批级别名称',

## 🎯 功能流程

### 页面加载时

1. **前端**：页面加载时调用 `checkCreatePermission()`
2. **前端**：发送请求到 `/invoice-applications/check-permission/create?account_set_id=xxx`
3. **后端**：查询用户在该账套中的审批级别
4. **后端**：返回 `has_access: true/false`
5. **前端**：根据返回结果设置 `canCreateTask` 变量
6. **前端**：通过 `v-if="canCreateTask"` 控制按钮显示

### 创建按钮显示逻辑

```vue
<el-button v-if="canCreateTask" type="primary" @click="handleCreate">
  <el-icon><Plus /></el-icon>
  创建开票任务
</el-button>
```
