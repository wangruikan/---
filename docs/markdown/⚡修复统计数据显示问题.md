# ⚡ 修复统计数据显示问题

## 🐛 问题描述

**用户反馈：**
> 明明有已中标，可是上面已中标还是显示为0，即将到期和准备中好像也是

**截图显示：**
- 项目列表中有"已中标"的项目（绿色标签）
- 但统计卡片显示：已中标 0

## 🔍 问题原因

### 原因1：状态字段混淆

数据库中有两个相关字段：
- `status` - 项目状态（如：preparing, submitted, won, lost等）
- `bid_result` - 投标结果（如：won, lost, abandoned）

**原来的统计逻辑：**
```php
'won' => BidProject::where('account_set_id', $accountSetId)
    ->where('bid_result', BidProject::RESULT_WON)  // 只查 bid_result
    ->count(),
```

**实际情况：**
- 创建项目时，`status` 初始为 `preparing`
- 用户更新状态为 "已中标"时，`status` 变成 `won`
- 但 `bid_result` 可能还是 `NULL`（如果没有通过"设置投标结果"功能）

所以统计时只查 `bid_result`，就查不到那些通过"更新状态"设置为已中标的项目。

### 原因2：时间计算可能有问题

`isDeadlineApproaching()` 和 `isOverdue()` 方法的时间计算可能有偏差。

## ✅ 解决方案

### 1. 修改统计逻辑 - 双重检查

**修改文件：** `app/Http/Controllers/BidProjectController.php`

**修改后的逻辑：**
```php
// 中标：检查 status='won' 或 bid_result='won'
'won' => BidProject::where('account_set_id', $accountSetId)
    ->where(function($query) {
        $query->where('status', BidProject::STATUS_WON)
              ->orWhere('bid_result', BidProject::RESULT_WON);
    })
    ->count(),

// 未中标：同样双重检查
'lost' => BidProject::where('account_set_id', $accountSetId)
    ->where(function($query) {
        $query->where('status', BidProject::STATUS_LOST)
              ->orWhere('bid_result', BidProject::RESULT_LOST);
    })
    ->count(),
```

### 2. 添加详细日志

在统计方法中添加了详细的日志记录：
- 记录账套ID和用户ID
- 记录有截止时间的项目数量
- 记录每个项目的截止时间检查结果
- 记录最终统计结果

## 📊 统计逻辑说明

### 各个统计项的计算方式

| 统计项 | 查询条件 | 说明 |
|--------|---------|------|
| 项目总数 | `account_set_id = X` | 当前账套的所有项目 |
| 已中标 | `status='won'` OR `bid_result='won'` | 状态或结果为中标 |
| 未中标 | `status='lost'` OR `bid_result='lost'` | 状态或结果为未中标 |
| 即将到期 | `bid_deadline` 在3天内 | 动态计算 |
| 已过期 | `bid_deadline` < 当前时间 | 动态计算 |
| 准备中 | `status='preparing'` | 初始状态 |
| 已提交 | `status='submitted'` | 已提交投标文件 |

### 状态 vs 投标结果

**两种方式设置中标：**

1. **方式1：更新项目状态**
   - 点击列表中的"更新状态"按钮
   - 选择"已中标"
   - 结果：`status='won'`, `bid_result=NULL`

2. **方式2：设置投标结果**
   - 在项目详情中点击"设置结果"
   - 选择"中标"并填写中标金额
   - 结果：`status='won'`, `bid_result='won'`

**修复后：** 无论用哪种方式，统计都能正确计算！
