# ✅ 完成 - 重新发起后隐藏按钮

## 📋 需求

红冲状态的申请重新发起后，原申请的"重新发起"按钮应该隐藏，避免重复发起。

---

## ✅ 实现方案

### 思路

添加一个标记字段 `has_resubmitted`，记录该申请是否已经重新发起过：
- `false` - 未重新发起，显示"重新发起"按钮
- `true` - 已重新发起，隐藏"重新发起"按钮

---

## 🔧 实现内容

### 1. 数据库 - 添加标记字段

#### 新增字段

| 字段名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `has_resubmitted` | `TINYINT(1)` | `0` (false) | 是否已重新发起 |
| `new_application_id` | `BIGINT` | `NULL` | 重新发起后的新申请ID（可选） |

#### SQL脚本

```sql
-- 添加 has_resubmitted 字段
ALTER TABLE `invoice_applications` 
ADD COLUMN `has_resubmitted` TINYINT(1) NOT NULL DEFAULT 0 
COMMENT '是否已重新发起（红冲后）' 
AFTER `status`;

-- 添加 new_application_id 字段
ALTER TABLE `invoice_applications` 
ADD COLUMN `new_application_id` BIGINT NULL 
COMMENT '重新发起后的新申请ID' 
AFTER `has_resubmitted`;
```

**执行方法**：
```bash

# 方法2：手动复制SQL到数据库管理工具执行
```

**修改权限判断**：
```php
public function canResubmit()
{
    // 只有红冲状态且未重新发起过的申请可以重新发起
    return $this->status === self::STATUS_RED_FLUSHED && !$this->has_resubmitted;
}
```

**添加访问器（用于API返回）**：
```php
protected $appends = [
    'status_text',
    'can_resubmit',  // 新增
];

public function getCanResubmitAttribute()
{
    return $this->canResubmit();
}
```

```
                ↓
        has_resubmitted = false

        点击"重新发起"
                ↓
        创建新申请（审批中）

                ↓
        ❌ 隐藏"重新发起"按钮
```
