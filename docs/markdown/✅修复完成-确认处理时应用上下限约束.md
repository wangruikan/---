# ✅ 修复完成：确认处理时应用上下限约束

## 问题描述

用户反馈：**从增减表导入到参保人员表时，基数和金额都不对**。

### 问题根源

在 `InsurancePersonnel::getOrCreateFromInsuranceChange()` 方法中，代码直接将 `insurance_changes` 表中的基数原样复制到 `insurance_personnel` 表，**没有应用上下限约束**。

```php
// ❌ 修复前的代码
'employee_social_security_base' => $change->employee_social_security_base,  // 直接复制，如 100.00
'employee_medical_insurance_base' => $change->employee_medical_insurance_base,
'employee_housing_fund_base' => $change->employee_housing_fund_base,
```

这导致：
- 参保人员表中存储的基数是**原始基数**（如 100.00）
- 但上下限要求最低是 101.00
- 后续计算明细时，使用了错误的基数

---

## 解决方案

### 修改文件：`app/Models/InsurancePersonnel.php`

#### 1. 在导入时应用上下限约束

```php
// ✅ 修复后的代码
// 应用上下限约束到基数
$constrainedBases = self::applyBaseConstraintsFromChange($change);

// 更新/创建时使用约束后的基数
'employee_social_security_base' => $constrainedBases['social_security'],  // 约束后的基数，如 101.00
'employee_medical_insurance_base' => $constrainedBases['medical_insurance'],
'employee_housing_fund_base' => $constrainedBases['housing_fund'],
```

#### 2. 新增方法：`applyBaseConstraintsFromChange()`

此方法负责：
1. 从 `insurance_changes` 的 JSON 配置中提取上下限
2. 使用**第一个保险类型的上下限**来约束基数
3. 返回约束后的三个基数

```php
private static function applyBaseConstraintsFromChange($change)
{
    // 获取原始基数
    $socialSecurityBase = $change->employee_social_security_base ?? 0;
    $medicalInsuranceBase = $change->employee_medical_insurance_base ?? 0;
    $housingFundBase = $change->employee_housing_fund_base ?? 0;

    // 解析保险配置获取上下限
    $socialSecurityTypes = json_decode($change->social_security_types, true) ?: [];
    $medicalInsuranceTypes = json_decode($change->medical_insurance_types, true) ?: [];
    $housingFundParams = json_decode($change->housing_fund_params, true) ?: [];

    // 应用社保上下限约束（使用第一个类型的上下限）
    if (!empty($socialSecurityTypes) && is_array($socialSecurityTypes)) {
        $firstType = reset($socialSecurityTypes);
        $minBase = $firstType['min_base_amount'] ?? null;
        $maxBase = $firstType['max_base_amount'] ?? null;
        $socialSecurityBase = self::applyConstraint($socialSecurityBase, $minBase, $maxBase);
    }

    // 医保和公积金同理...

    return [
        'social_security' => $socialSecurityBase,
        'medical_insurance' => $medicalInsuranceBase,
        'housing_fund' => $housingFundBase
    ];
}
```

#### 3. 新增方法：`applyConstraint()`

通用的单个基数约束方法：

```php
private static function applyConstraint($base, $minBase, $maxBase)
{
    $constrainedBase = $base;

    // 应用下限约束：低于下限就用下限
    if ($minBase !== null && $constrainedBase < $minBase) {
        $constrainedBase = $minBase;
    }

    // 应用上限约束：高于上限就用上限
    if ($maxBase !== null && $constrainedBase > $maxBase) {
        $constrainedBase = $maxBase;
    }

    return $constrainedBase;
}
```

```
1. insurance_changes 表：
   - employee_social_security_base = 100.00 （原始基数）
   - social_security_types = JSON 包含 min_base_amount = 101.00

2. 确认处理时，调用 getOrCreateFromInsuranceChange()

3. 直接复制到 insurance_personnel 表：
   - employee_social_security_base = 100.00 ❌（没有约束）

4. 生成明细时：
   - 使用 100.00 计算金额 ❌（基数错误）
   - 公司金额 = 100.00 × 比例
   - 个人金额 = 100.00 × 比例
```

### 修复后的流程（✅ 正确）

```
1. insurance_changes 表：
   - employee_social_security_base = 100.00 （原始基数）
   - social_security_types = JSON 包含 min_base_amount = 101.00

2. 确认处理时，调用 getOrCreateFromInsuranceChange()

3. 应用上下限约束：
   - 原始基数 100.00 < 下限 101.00
   - 约束后基数 = 101.00 ✅

4. 存储到 insurance_personnel 表：
   - employee_social_security_base = 101.00 ✅（约束后的基数）

5. 生成明细时：
   - 使用 101.00 计算金额 ✅（基数正确）
   - 公司金额 = 101.00 × 比例
   - 个人金额 = 101.00 × 比例
```

## 重要说明

### 为什么在确认处理时约束？

1. **保留原始数据**：`insurance_changes` 表中保留员工的原始设定基数
2. **确认时约束**：确认处理时应用上下限，存储约束后的基数到 `insurance_personnel`
3. **明细正确**：生成明细时使用的是约束后的基数，确保金额正确

### 为什么使用第一个类型的上下限？

在同一个地区内，所有保险类型的上下限通常是统一的：
- 社保：所有社保类型（养老、失业、工伤等）共用同一个上下限
- 医保：医疗保险的上下限
- 公积金：公积金配置中的上下限

因此，使用第一个类型的上下限即可代表整个地区的约束规则。
