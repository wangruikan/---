**错误信息**：
```

SQL: ... where `approval_level` in (2, 3, 4) ...
```

**错误**：从 `users` 表查询审批人

## ✅ 修复方案

### 1. 查询审批人配置

#### 修改前（错误）
```php
$approvers = \App\Models\User::where('account_set_id', $accountSetId)
    ->where('status', 'active')  // ❌ users 表没有此字段
    ->whereIn('approval_level', [2, 3, 4])  // ❌ users 表没有此字段
    ->orderBy('approval_level')
    ->get();
```

#### 修改后（正确）
```php
$approvers = DB::table('account_set_users')
    ->where('account_set_id', $accountSetId)
    ->where('approval_level', '>', 1) // ✅ 跳过经办（级别1）
    ->orderBy('approval_level')
    ->get();
```

### 3. 创建审批记录

#### 正确的字段名和数据结构

```php
// 1. 创建经办节点（自动通过）
\App\Models\ApprovalRecord::create([
    'instance_id' => $instance->id,  // ✅ 使用 instance_id
    'step_order' => 1,
    'step_name' => '经办',
    'approver_id' => $user->id,
    'approver_name' => $user->name,
    'status' => 'approved',
    'comment' => '经办提交，自动通过',
    'approved_at' => now(),

$firstApprover = $approvers->first();
$approverUser = \App\Models\User::find($firstApprover->user_id);  // ✅ 使用 user_id

\App\Models\ApprovalRecord::create([
    'instance_id' => $instance->id,
    'step_order' => 2,
    'step_name' => $firstApprover->approval_level_name,  // ✅ 使用 approval_level_name
    'approver_id' => $firstApprover->user_id,
    'approver_name' => $approverUser->name,
    'status' => 'pending',
    'comment' => null,
    'approved_at' => null,
]);

// 3. 创建后续审批人记录（等待中）
$stepOrder = 3;
foreach ($approvers->skip(1) as $approver) {
    $approverUser = \App\Models\User::find($approver->user_id);
    \App\Models\ApprovalRecord::create([
        'instance_id' => $instance->id,
        'step_order' => $stepOrder,
        'step_name' => $approver->approval_level_name,
        'approver_id' => $approver->user_id,
        'approver_name' => $approverUser->name,
        'status' => 'waiting',
        'comment' => null,
        'approved_at' => null,
    ]);
    $stepOrder++;
}
```

## 📊 数据库表结构说明

```
字段名                说明
------------------   ------------------
```

### 3. approval_records（审批记录表）
```
字段名                说明

```

| 3 | 第二个审批人 | 3 | waiting | 等待步骤2完成 |

## ⚠️ 注意事项

1. **审批人配置必填**：`account_set_users` 表必须有配置，否则会提示"未找到审批人员配置"
2. **审批级别顺序**：审批级别从 1 开始，1 表示经办，2+ 表示审批人
3. **字段名称**：
   - ✅ `instance_id`（不是 `approval_instance_id`）
   - ✅ `user_id`（不是 `id`）
   - ✅ `approval_level_name`（不是 `level_name`）
4. **审批状态**：
   - `approved`：已通过
   - `pending`：待审批（出现在待办列表）
   - `waiting`：等待中
   - `rejected`：已驳回
