## 🎯 测试步骤

### 步骤1：第一次设置上下限

```php
// 获取第一个社保类型
$type = App\Models\SocialSecurityType::where('region_id', $employee->social_security_region_id)->first();

// 使用服务设置上下限
$service = app(App\Services\BaseLimitUpdateService::class);
$result = $service->updateLimits($type, 2360, 26421);

// 查看结果
print_r($result);

// 刷新数据
$type->refresh();
echo "old上下限: " . $type->old_min_base_amount . " - " . $type->old_max_base_amount . PHP_EOL;
echo "new上下限: " . $type->new_min_base_amount . " - " . $type->new_max_base_amount . PHP_EOL;
```

**预期结果**：
```
should_compensate => false
reason => 第一次设置上下限，没有旧值，不需要补差
old上下限: NULL - NULL
new上下限: 2360 - 26421
```

### 步骤2：第二次修改上下限（触发补差）

```php
// 修改上下限
$result = $service->updateLimits($type, 3000, 30000);
print_r($result);

// 如果需要补差，调用补差服务
if ($result['should_compensate']) {
    $compensationService = app(App\Services\BaseLimitCompensationService::class);
    $compResult = $compensationService->calculateSocialSecurityCompensation(
        $type->region_id,
        $result['old_min'],
        $result['old_max'],
        $result['new_min'],
        $result['new_max']
    );
    echo "补差结果: " . PHP_EOL;
    print_r($compResult);
}

// 查看上下限
$type->refresh();
echo "old上下限: " . $type->old_min_base_amount . " - " . $type->old_max_base_amount . PHP_EOL;
echo "new上下限: " . $type->new_min_base_amount . " - " . $type->new_max_base_amount . PHP_EOL;
```

**预期结果**：
```
should_compensate => true
reason => 修改上下限，需要生成/更新补差明细
old上下限: 2360 - 26421
new上下限: 3000 - 30000
补差结果: 
  affected_count => 1
  compensation_count => 1
```

### 步骤4：同月第三次修改（更新补差）

```php
// 再次修改上下限
$result = $service->updateLimits($type, 3500, 35000);

// 调用补差服务
if ($result['should_compensate']) {
    $compResult = $compensationService->calculateSocialSecurityCompensation(
        $type->region_id,
        $result['old_min'],
        $result['old_max'],
        $result['new_min'],
        $result['new_max']
    );
    print_r($compResult);
}

// 查看上下限
$type->refresh();
echo "old上下限: " . $type->old_min_base_amount . " - " . $type->old_max_base_amount . PHP_EOL;
echo "new上下限: " . $type->new_min_base_amount . " - " . $type->new_max_base_amount . PHP_EOL;

// 查看补差记录的更新时间
$comp = App\Models\InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->where('compensation_type', 'social_security')
    ->orderBy('updated_at', 'desc')
    ->first();
```

**预期结果**：
```

new上下限: 3500 - 35000
补差记录被更新（updated_at是最新的）
```

## 📊 SQL查询验证

### 查看上下限字段
```sql
SELECT 
    id,
    name,
    old_min_base_amount AS '旧下限',
    old_max_base_amount AS '旧上限',
    old_limits_updated_at AS '旧修改时间',
    new_min_base_amount AS '新下限',
    new_max_base_amount AS '新上限',
    new_limits_updated_at AS '新修改时间'
FROM social_security_types
WHERE region_id = (
    SELECT social_security_region_id 
    FROM employees 
    WHERE name = 'wrk'
);
```

### 查看补差记录
```sql
SELECT 
    employee_name AS '员工',
    old_base AS '旧基数',
    new_base AS '新基数',
    compensation_start_month AS '起始',
    compensation_end_month AS '结束',
    compensation_months AS '月数',
    total_amount AS '总额',
    created_at AS '创建时间',
    updated_at AS '更新时间'
FROM insurance_compensation_records
WHERE employee_name = 'wrk'
  AND compensation_type = 'social_security'
ORDER BY updated_at DESC;
```

## ✅ 完整测试脚本（一次性执行）

```php
// 在tinker中一次性执行
$employee = App\Models\Employee::where('name', 'wrk')->first();
$type = App\Models\SocialSecurityType::where('region_id', $employee->social_security_region_id)->first();
$service = app(App\Services\BaseLimitUpdateService::class);
$compensationService = app(App\Services\BaseLimitCompensationService::class);

// 第1次
echo "=== 第1次设置上下限 ===" . PHP_EOL;
$result = $service->updateLimits($type, 2360, 26421);
echo "需要补差: " . ($result['should_compensate'] ? '是' : '否') . PHP_EOL;

// 第2次
echo "\n=== 第2次修改上下限 ===" . PHP_EOL;
$result = $service->updateLimits($type, 3000, 30000);
if ($result['should_compensate']) {
    $compResult = $compensationService->calculateSocialSecurityCompensation(
        $type->region_id, $result['old_min'], $result['old_max'], 
        $result['new_min'], $result['new_max']
    );
    echo "生成补差: " . $compResult['compensation_count'] . " 条" . PHP_EOL;
}

// 第3次
echo "\n=== 第3次修改上下限 ===" . PHP_EOL;
$result = $service->updateLimits($type, 3500, 35000);
if ($result['should_compensate']) {
    $compResult = $compensationService->calculateSocialSecurityCompensation(
        $type->region_id, $result['old_min'], $result['old_max'], 
        $result['new_min'], $result['new_max']
    );
    echo "更新补差: " . $compResult['affected_count'] . " 个员工" . PHP_EOL;
}

// 查看结果
$comp = App\Models\InsuranceCompensationRecord::where('employee_id', $employee->id)->first();
echo "\n=== 补差记录 ===" . PHP_EOL;
echo "期间: " . $comp->compensation_start_month . " ~ " . $comp->compensation_end_month . PHP_EOL;
echo "旧基数: " . $comp->old_base . ", 新基数: " . $comp->new_base . PHP_EOL;
echo "总额: " . $comp->total_amount . PHP_EOL;
```

## 🎯 测试要点

1. ✅ 第一次设置不生成补差
2. ✅ 第二次修改生成补差
3. ✅ 第三次修改更新补差（不是新增）
4. ✅ 补差期间正确（今年1月到修改前一个月）
5. ✅ 旧基数读取正确（本月之前最近一次）
6. ✅ 新基数读取正确（本月有就用本月的）
