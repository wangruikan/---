# 累计专项附加扣除字段说明

## 📊 两个字段的区别

工资表中有**2个**与专项附加扣除相关的字段，它们有不同的作用：

| 字段名 | 数据库字段 | 作用 | 计算方式 |
|-------|-----------|------|---------|
| **累计专项附加扣除（6项扣除）** | `special_deduction_monthly` | **当月的**6项扣除合计 | 当月6项扣除之和 |
| **累计专项附加扣除** | `special_deduction` | **1月到当前月的**累计值 | 1月到当前月的`special_deduction_monthly`累加 |

---

## 🎯 字段详解

### 1️⃣ 累计专项附加扣除（6项扣除）
**字段名**：`special_deduction_monthly`

**作用**：记录**当月**的个人所得税专项附加扣除合计金额。

**包含的6项扣除**：
1. 子女教育
2. 继续教育
3. 大病医疗
4. 住房贷款利息
5. 住房租金
6. 赡养老人

**计算公式**：
```
累计专项附加扣除（6项扣除）= 当月的6项扣除金额之和
```

**示例**（2025年9月）：
```
子女教育：1000元/月
住房贷款利息：1000元/月
赡养老人：2000元/月
其他3项：0元/月

累计专项附加扣除（6项扣除）= 1000 + 1000 + 2000 = 4000元
```

---

### 2️⃣ 累计专项附加扣除
**字段名**：`special_deduction`

**作用**：记录**1月到当前月**的专项附加扣除累计总额，用于计算个人所得税。

**计算公式**：
```
累计专项附加扣除 = ∑(1月到当前月的special_deduction_monthly)
```

**示例**（2025年9月）：
```
假设每月的专项附加扣除都是4000元：

1月：4000元
2月：4000元
3月：4000元
4月：4000元
5月：4000元
6月：4000元
7月：4000元
8月：4000元
9月：4000元

累计专项附加扣除 = 4000 × 9 = 36000元
```

---

## 💡 为什么需要两个字段？

### 场景1：专项扣除金额每月相同
```
1月：子女教育1000 + 住房贷款1000 = 2000元
2月：子女教育1000 + 住房贷款1000 = 2000元
...
9月：子女教育1000 + 住房贷款1000 = 2000元

9月工资表：
- 累计专项附加扣除（6项扣除）= 2000元（仅当月）
- 累计专项附加扣除 = 2000 × 9 = 18000元（1-9月累计）
```

### 场景2：专项扣除金额中途变化
```
1-6月：子女教育1000元/月
7月：新增赡养老人2000元/月
7-9月：子女教育1000 + 赡养老人2000 = 3000元/月

9月工资表：
- 累计专项附加扣除（6项扣除）= 3000元（9月当月）
- 累计专项附加扣除 = (1000×6) + (3000×3) = 6000 + 9000 = 15000元（1-9月累计）
```

## 📊 在个税计算中的应用

### 个税计算公式
```
累计应纳税所得额 = 累计收入 
                - 累计减除费用（5000×月数）
                - 累计专项扣除（社保公积金）
                - 累计专项附加扣除 ← 使用这个字段！
                - 累计其他应纳税项
```

**示例**（2025年9月）：
```
累计收入：90,000元（1-9月应发工资）
累计减除费用：45,000元（5000×9个月）
累计专项扣除：12,600元（社保公积金1-9月累计）
累计专项附加扣除：36,000元（6项扣除1-9月累计）← 用于计算个税

累计应纳税所得额 = 90,000 - 45,000 - 12,600 - 36,000 = -3,600元
（应纳税所得额为负，不需要缴税）
```

## 🔄 字段之间的关系

### 数据流程图
```
每月工资生成时：

1. 查询员工的6项专项附加扣除配置
   ↓
2. 计算当月的6项扣除合计 
   → special_deduction_monthly（当月）
   ↓
3. 查询1月到上月的special_deduction_monthly
   ↓
4. 累加计算
   → special_deduction = ∑(1月到当月的special_deduction_monthly)
```

### 代码实现（后端）
```php
// 1. 计算当月专项扣除（6项扣除）
$specialDeductionData = $this->calculateSpecialDeduction($employee, $month);
$specialDeductionMonthly = $specialDeductionData['total']; // 当月6项合计

// 2. 查询之前月份的数据
$previousSalaries = Salary::where('employee_id', $employee->id)
    ->where('month', '>=', $year . '-01')
    ->where('month', '<', $month)
    ->get();

// 3. 累计专项附加扣除 = 之前月份的累计 + 当月的
$previousCumulativeSpecialDeduction = $previousSalaries->sum('special_deduction_monthly');
$cumulativeSpecialDeduction = $previousCumulativeSpecialDeduction + $specialDeductionMonthly;

// 4. 保存到数据库
Salary::create([
    'special_deduction_monthly' => $specialDeductionMonthly,    // 当月
    'special_deduction' => $cumulativeSpecialDeduction,         // 累计
    // ...
]);
```

## 🎯 实际用途

### 1. 数据核对
通过查看"累计专项附加扣除（6项扣除）"，可以快速核对：
- ✅ 当月的6项扣除配置是否正确
- ✅ 各项扣除金额之和是否匹配

### 2. 变化追踪
对比不同月份的"累计专项附加扣除（6项扣除）"，可以发现：
- 📊 哪个月增加了新的扣除项
- 📊 哪个月的扣除金额发生了变化

**示例**：
```
6月：累计专项附加扣除（6项扣除）= ¥2,000（子女教育1000 + 住房贷款1000）
7月：累计专项附加扣除（6项扣除）= ¥4,000（子女教育1000 + 住房贷款1000 + 赡养老人2000）
→ 发现：7月新增了"赡养老人"扣除项
```

### 3. 个税计算验证
检查"累计专项附加扣除"的计算是否正确：
```
累计专项附加扣除（9月）应该等于：
= 1月的special_deduction_monthly
+ 2月的special_deduction_monthly
+ ...
+ 9月的special_deduction_monthly
```

## ✨ 总结

| 对比项 | 累计专项附加扣除（6项扣除） | 累计专项附加扣除 |
|-------|------------------------|---------------|
| **时间范围** | 仅当月 | 1月到当前月 |
| **用途** | 数据核对、变化追踪 | **个税计算** |
| **计算方式** | 6项扣除直接相加 | 历史月份累计相加 |
| **显示颜色** | 蓝色 | 绿色加粗 |
| **是否参与计税** | ❌ 不直接参与 | ✅ **直接用于个税计算** |

**关键点**：
- 📌 `special_deduction_monthly` 是中间计算值，记录每月的扣除金额
- 📌 `special_deduction` 是最终用于个税计算的累计值
- 📌 两个字段配合使用，既能看到当月明细，又能追踪累计总额
