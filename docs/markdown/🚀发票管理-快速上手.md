# 🚀 发票管理模块 - 快速上手指南

## ⚡ 3步快速启动

### 第1步：创建数据库表（在Navicat中执行）

```sql
-- 复制以下SQL到Navicat中执行
-- ========================================

-- 1. 发票项目配置表
CREATE TABLE IF NOT EXISTS `invoice_projects` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `account_set_id` BIGINT UNSIGNED NOT NULL,
  `project_name` VARCHAR(255) NOT NULL,
  `remark` TEXT NULL,
  `created_by` BIGINT UNSIGNED NOT NULL,
  `created_at` TIMESTAMP NULL DEFAULT NULL,
  `updated_at` TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  INDEX `idx_account_set` (`account_set_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 2. 发票申请表
CREATE TABLE IF NOT EXISTS `invoice_applications` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `account_set_id` BIGINT UNSIGNED NOT NULL,
  `application_no` VARCHAR(50) NOT NULL,
  `year` INT NOT NULL,
  `month` INT NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'draft',
  `approval_instance_id` BIGINT UNSIGNED NULL,
  `submitter_id` BIGINT UNSIGNED NOT NULL,
  `submitted_at` TIMESTAMP NULL DEFAULT NULL,
  `rejection_reason` TEXT NULL,
  `attachments` JSON NULL,
  `created_by` BIGINT UNSIGNED NOT NULL,
  `created_at` TIMESTAMP NULL DEFAULT NULL,
  `updated_at` TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_application_no` (`application_no`),
  INDEX `idx_account_set` (`account_set_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 3. 发票明细表
CREATE TABLE IF NOT EXISTS `invoice_items` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `application_id` BIGINT UNSIGNED NOT NULL,
  `invoice_project_id` BIGINT UNSIGNED NOT NULL,
  `project_name` VARCHAR(255) NOT NULL,
  `sequence` INT NOT NULL DEFAULT 1,
  `item_name` VARCHAR(255) NOT NULL,
  `amount` DECIMAL(15, 2) NOT NULL DEFAULT 0.00,
  `remark` VARCHAR(500) NULL,
  `created_at` TIMESTAMP NULL DEFAULT NULL,
  `updated_at` TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  INDEX `idx_application` (`application_id`),
  CONSTRAINT `fk_invoice_items_application` FOREIGN KEY (`application_id`) REFERENCES `invoice_applications` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 🎯 使用流程

### 1. 配置发票项目（首次使用）

1. 登录系统（使用审批人账号，不能是经办人）
2. 左侧菜单 → **开票管理** → **发票项目配置**
3. 点击 **[+ 新建项目]**
4. 填写：
   - 项目名称：如"应发工资"
   - 备注：项目说明（可选）
5. 点击"确定"

**建议创建的项目：**
- 应发工资
- 劳保费
- 商业险

### 2. 创建开票任务

1. 左侧菜单 → **开票管理** → **发票申请**
2. 点击 **[+ 创建开票任务]**
3. 选择年份和月份（如：2025年11月）
4. 点击"确定"
5. 系统自动跳转到编辑页面

### 3. 添加扣除明细

1. 在"扣除明细"标签页中
2. 点击 **[+ 添加明细]**
3. 填写：
   - 项目：选择"应发工资"
   - 名称：输入"应发工资"
   - 金额：输入金额（如：100000）
   - 备注：可选
4. 点击"确定"
5. 重复步骤2-4，添加其他明细（劳保费、商业险等）

**示例明细：**
| 序号 | 项目     | 名称     | 金额      |
|------|----------|----------|-----------|
| 1    | 应发工资 | 应发工资 | 100,000   |
| 11   | 劳保费   | 劳保费   | 5,000     |
| 12   | 商业险   | 商业险   | 3,000     |

**合计：¥108,000.00**

### 4. 生成Excel扣除明细表

1. 添加完所有明细后
2. 点击 **[📄 生成扣除明细表（Excel）]** 按钮
3. 等待生成完成（1-2秒）
4. 生成的Excel文件会自动添加到"附件上传"中

### 5. 上传附件

1. 切换到 **[附件上传]** 标签页
2. 点击 **[上传附件]** 按钮
3. 选择文件（支持最大10MB）
4. 上传成功

**注意：** 必须至少有1个附件才能提交审批

### 6. 提交审批

1. 确认明细和附件都已添加
2. 点击 **[提交审批]** 按钮
3. 确认提交
4. 等待审批

**审批流程：**
- ❌ 第1节点（经办）：自动跳过
- ✅ 第2节点：等待审批
- ✅ 第3节点：等待审批
- ✅ 第4节点：等待审批

### 7. 如果被驳回，重新提交

1. 在列表中找到"已驳回"的申请
2. 点击 **[重新提交]** 按钮
3. 确认重新提交
4. 系统会：
   - 原申请变为"红冲"状态
   - 创建新申请（复制所有数据）
   - 新申请自动提交审批

## 🔧 常见问题

### Q1: 看不到"开票管理"菜单？

**A:** 当前用户不是审批人（节点2-4）

**解决：**
1. 联系管理员
2. 在"审批管理"中配置用户为审批人
3. 重新登录

### Q2: 生成Excel失败？

**A:** 缺少 PhpSpreadsheet 库

**解决：**
```bash
cd E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
composer require phpoffice/phpspreadsheet
```

### Q3: 上传附件失败？

**A:** 文件太大或权限不足

**解决：**
1. 确认文件大小 < 10MB
2. 检查 storage 目录权限
3. 执行：`php artisan storage:link`

### Q4: 提交后没有审批记录？

**A:** 审批节点配置不正确

**解决：**
1. 进入"审批管理"
2. 检查节点2、3、4是否配置了审批人
3. 确认审批流程类型为"invoice"

### Q5: 重新提交后原申请没变红冲？

**A:** 缓存问题

**解决：**
```bash
php artisan cache:clear
```
然后刷新浏览器（Ctrl + F5）
