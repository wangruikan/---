# ✅ 投标项目管理模块 - 完整部署说明

## 📋 模块概述

投标项目管理模块是一个专为甲方投标业务设计的完整管理系统，提供从项目登记、文档管理、进度跟踪到结果管理的全流程功能。

### 核心功能
1. **项目登记管理** - 项目基本信息、招标单位信息、投标时间等
2. **状态流转管理** - 10种状态（准备中→已提交→已开标→评标中→已中标/未中标/已放弃→已签约→已完成/已取消）
3. **文档管理** - 支持招标文件、投标文件、技术方案、报价单等多种文档类型
4. **进度记录** - 时间线展示项目进度，支持添加各类记录
5. **投标结果管理** - 中标/未中标/放弃，中标金额管理
6. **统计分析** - 项目总数、中标数、即将到期提醒等
7. **搜索筛选** - 支持关键词、类别、状态、结果等多维度搜索

## 📊 数据库表结构说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |

| bid_result | VARCHAR(20) | 投标结果（won/lost/abandoned） |
| win_amount | DECIMAL(15,2) | 中标金额 |
| ... | ... | 其他字段详见SQL文件 |

### 2. 项目状态流转

```
准备中 (preparing)
    ↓
已提交 (submitted)
    ↓
已开标 (opened)
    ↓
评标中 (evaluating)
    ↓
    ├─→ 已中标 (won) → 已签约 (contracted) → 已完成 (completed)
    ├─→ 未中标 (lost)
    ├─→ 已放弃 (abandoned)
    └─→ 已取消 (cancelled)
```

## 🎯 功能特性详解

## 🔧 配置说明

### 文件上传配置

默认文件保存路径：`storage/app/public/bid_documents/{项目ID}/`

如需修改上传限制，请编辑：
- **后端**：`app/Http/Controllers/BidProjectController.php` 第485行
  ```php
  'file' => 'required|file|max:51200', // 最大50MB
  ```
- **前端**：暂无限制，由后端控制

### 权限配置

当前模块使用账套ID进行数据隔离，确保：
1. 用户只能看到自己账套的项目
2. 管理员可以看到所有账套的项目

## 📝 使用指南

### 创建投标项目
1. 点击"新建项目"按钮
2. 填写基本信息：
   - 项目名称（必填）
   - 项目类别（可选或新增）
   - 招标单位信息
   - 项目预算
3. 填写投标信息：
   - 投标截止时间
   - 开标时间
   - 招标方式
   - 投标保证金
   - 负责人信息
4. 点击"确定"保存

### 管理项目文档
1. 在项目列表点击"查看详情"
2. 切换到"文档管理"标签
3. 点击"上传文档"按钮
4. 选择文档类型、填写文档名称、选择文件
5. 上传成功后可随时下载或删除

### 跟踪项目进度
1. 在项目详情中切换到"进度记录"标签
2. 点击"添加记录"按钮
3. 选择记录类型、填写标题和内容
4. 保存后在时间线中显示

### 更新项目状态
1. 在项目列表点击"更新状态"按钮
2. 选择新状态
3. 填写变更原因（可选）
4. 确认后系统自动记录状态变更日志

### 设置投标结果
1. 在项目详情的基本信息中
2. 点击"设置结果"按钮
3. 选择结果类型（中标/未中标/放弃）
4. 如果中标，填写中标金额和日期
5. 确认后系统自动更新项目状态

## ⚠️ 注意事项

1. **数据安全**
   - 删除项目会同时删除所有关联的文档和记录
   - 删除操作不可恢复，请谨慎操作

2. **文件存储**
   - 上传的文件保存在 `storage/app/public/` 目录
   - 需要确保该目录有写权限
   - 定期备份重要文档

3. **性能优化**
   - 大量项目时建议使用搜索筛选
   - 文件上传限制为50MB
   - 建议定期清理已完成的历史项目

4. **状态管理**
   - 状态变更会自动记录日志
   - 建议按照标准流程进行状态流转
   - 关键节点及时添加进度记录

5. **截止时间提醒**
   - 系统会自动检测即将到期的项目
   - 3天内会显示黄色警告标签
   - 已过期会显示红色标签

## 🐛 常见问题

### 1. 菜单中看不到"投标项目管理"
**解决方案：**
- 清理浏览器缓存（Ctrl+Shift+Delete）
- 确认 `src/layouts/MainLayout.vue` 已更新
- 检查路由配置 `src/router/index.js`

### 2. 点击菜单显示404
**解决方案：**
- 确认前端已重启
- 检查 `src/views/BidProjects/index.vue` 文件是否存在
- 清理 `node_modules/.vite` 缓存并重启

### 3. API请求失败
**解决方案：**
- 检查数据库表是否创建成功
- 执行 `php artisan cache:clear`
- 检查 `routes/api.php` 是否包含投标项目路由
- 查看Laravel日志：`storage/logs/laravel.log`

### 4. 文件上传失败
**解决方案：**
- 检查 `storage/app/public/` 目录权限
- 确认文件大小不超过50MB
- 检查PHP配置：`upload_max_filesize` 和 `post_max_size`

### 5. 统计数据不更新
**解决方案：**
- 刷新页面
- 检查API接口 `/api/bid-projects/statistics` 是否正常
- 查看浏览器控制台是否有错误
