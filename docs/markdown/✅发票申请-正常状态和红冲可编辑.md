# ✅ 发票申请 - 初始状态为"正常"，红冲可编辑

## 📋 需求调整

1. **初始状态**：创建开票任务后，状态为"正常"（而不是"草稿"）
2. **被驳回**：状态改为"红冲"
3. **红冲状态**：可以编辑、上传附件，然后重新发起流程

### 1. 模型 - 添加"正常"状态

#### 文件：`app/Models/InvoiceApplication.php`

**添加状态常量**：
```php
// 状态常量
const STATUS_DRAFT = 'draft';              // 草稿（保留用于兼容）
const STATUS_NORMAL = 'normal';            // 正常（初始状态）
const STATUS_PENDING = 'pending';          // 审批中
const STATUS_APPROVED = 'approved';        // 已通过
const STATUS_REJECTED = 'rejected';        // 已驳回（保留用于兼容）
const STATUS_RED_FLUSHED = 'red_flushed';  // 红冲
```

**更新状态文本映射**：
```php
public function getStatusTextAttribute()
{
    $statusMap = [
        self::STATUS_DRAFT => '草稿',
        self::STATUS_NORMAL => '正常',
        self::STATUS_PENDING => '审批中',
        self::STATUS_APPROVED => '已通过',
        self::STATUS_REJECTED => '已驳回',
        self::STATUS_RED_FLUSHED => '红冲',
    ];

    return $statusMap[$this->status] ?? $this->status;
}
```

**修改权限方法**：
```php
/**
 * 是否可以编辑
 */
public function canEdit()
{
    // 正常、草稿、红冲状态都可以编辑
    return in_array($this->status, [self::STATUS_DRAFT, self::STATUS_NORMAL, self::STATUS_RED_FLUSHED]);
}

/**
 * 是否可以提交审批
 */
public function canSubmit()
{
    // 正常、草稿、红冲状态都可以提交（需满足条件）
    return in_array($this->status, [self::STATUS_DRAFT, self::STATUS_NORMAL, self::STATUS_RED_FLUSHED]) && 
           $this->items()->count() > 0 &&
           !empty($this->attachments);
}

/**
 * 是否可以重新提交（重新发起）
 */
public function canResubmit()
{
    // 只有红冲状态可以重新发起
    return $this->status === self::STATUS_RED_FLUSHED;
}
```

## 💡 业务逻辑

### 1. 创建开票任务
- **操作人**：审批人（审批级别2、3、4）
- **填写内容**：任务名称、年份、月份
- **初始状态**：`normal`（正常）
- **可见按钮**：编辑、删除

### 2. 业务人员编辑（正常状态）
- **可以做**：
  - ✅ 填写项目名称
  - ✅ 添加扣除明细项
  - ✅ 上传附件
  - ✅ 生成扣除明细表
- **完成后**：点击"提交审批"按钮

### 3. 审批流程
- **状态变化**：正常 → 审批中
- **审批人操作**：逐级审批
- **结果**：
  - ✅ **全部通过** → 状态变为"已通过"
  - ❌ **有人驳回** → 状态变为"红冲"

### 4. 被驳回后（红冲状态）
- **状态**：`red_flushed`（红冲）
- **可见按钮**：编辑、重新发起
- **可以做**：
  - ✅ 修改项目名称
  - ✅ 修改扣除明细
  - ✅ 重新上传附件
  - ✅ 点击"重新发起"

### 5. 重新发起
- **触发**：点击"重新发起"按钮
- **系统行为**：
  1. 创建新的发票申请（复制所有数据）
  2. 自动提交审批
  3. 原红冲申请保留（不可再编辑）
- **状态变化**：新申请状态为"审批中"

| 方面 | 旧逻辑 | 新逻辑 |
