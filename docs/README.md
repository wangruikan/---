## 📖 使用说明

### SQL 脚本使用
1. **执行前必须备份数据库**
2. 仔细阅读脚本内容，了解其功能
3. 按照文件名中的顺序执行（如有编号）
4. 注意脚本的依赖关系

### 文档查阅
- 带 ⚡ 前缀：需要立即执行或重要文件
- 带 ✅ 前缀：已完成的功能或修复
- 带 🚀 前缀：快速部署指南
- 带 🔧 前缀：问题修复说明
- 带 📋 前缀：操作指南或清单

### 脚本执行
- Windows 系统：在项目根目录下运行 `.bat` 文件
- Linux 系统：在项目根目录下运行 `.sh` 文件
- Python 脚本：确保已安装 Python 环境

## ⚠️ 注意事项

1. **数据安全**：执行任何 SQL 脚本前务必备份数据库
2. **环境检查**：确认当前环境与脚本要求一致
3. **顺序执行**：某些脚本有依赖关系，需按顺序执行
4. **测试优先**：建议先在测试环境验证后再在生产环境执行
