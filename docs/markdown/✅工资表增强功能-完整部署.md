# ✅ 工资表增强功能 - 完整部署指南

## 🎯 新增功能

### 1. **序号字段**
- 自动编号，从1开始递增
- 用于前端排序显示

### 2. **部门和岗位字段**
- 从员工档案自动导入
- 在员工档案中新增"岗位"字段

### 3. **基础工资自动导入**
- 从员工档案的 `basic_salary` 字段读取
- 作为应发工资的基础

### 4. **保险金额自动计算**
- **单位保险合计**：社保单位部分 + 公积金单位部分
- **个人保险合计**：社保个人部分 + 公积金个人部分
- 从参保人员表自动计算

### 5. **补差金额自动导入**
- 从补差记录表自动读取
- 按补差月数平均分配

## 📋 部署步骤

### 步骤1：更新数据库表结构

#### 1.1 添加岗位字段到员工档案表

执行 `add_position_to_employees.sql`：

```sql
-- 添加岗位字段
ALTER TABLE employees 
ADD COLUMN position VARCHAR(100) NULL COMMENT '岗位' AFTER name;
```

#### 1.2 更新工资表结构

执行 `update_salaries_table_structure.sql`：

```sql
-- 添加序号字段
ALTER TABLE salaries 
ADD COLUMN row_number INT NULL COMMENT '序号' AFTER id;

-- 添加部门字段
ALTER TABLE salaries 
ADD COLUMN department VARCHAR(100) NULL COMMENT '所在部门' AFTER employee_id;

-- 添加岗位字段
ALTER TABLE salaries 
ADD COLUMN position VARCHAR(100) NULL COMMENT '岗位' AFTER department;

-- 添加单位保险合计字段
ALTER TABLE salaries 
ADD COLUMN company_insurance_total DECIMAL(10, 2) DEFAULT 0.00 COMMENT '单位保险合计' AFTER housing_fund;

-- 添加个人保险合计字段
ALTER TABLE salaries 
ADD COLUMN personal_insurance_total DECIMAL(10, 2) DEFAULT 0.00 COMMENT '个人保险合计' AFTER company_insurance_total;

-- 添加补差合计字段
ALTER TABLE salaries 
ADD COLUMN compensation_total DECIMAL(10, 2) DEFAULT 0.00 COMMENT '补差合计' AFTER personal_insurance_total;
```

## 🔄 工资表生成逻辑

### 自动计算流程

```
1. 读取员工基本信息
   ↓
2. 从员工档案导入：
   - 姓名
   - 部门
   - 岗位
   - 基础工资
   ↓
3. 计算本月保险金额：
   - 查询参保人员表
   - 计算社保（单位+个人）
   - 计算公积金（单位+个人）
   - 汇总单位保险合计
   - 汇总个人保险合计
   ↓
4. 计算补差金额：
   - 查询补差记录表
   - 按月份平均分配
   - 汇总补差合计
   ↓
5. 计算应发和实发：
   - 应发工资 = 基础工资
   - 实发工资 = 应发工资 - 个人保险合计
```

## 🔍 计算示例

### 示例：张三的工资计算

**员工档案**：
- 姓名：张三
- 部门：技术部
- 岗位：Java开发工程师
- 基础工资：10,000元
- 社保基数：7,947元
- 公积金基数：7,947元

**参保配置**：
- 社保单位：1,271.52元
- 社保个人：556.36元
- 公积金单位：954.00元
- 公积金个人：954.00元

**补差记录**：
- 本月补差：100元

**计算结果**：
```
应发工资 = 10,000
单位保险合计 = 1,271.52 + 954.00 = 2,225.52
个人保险合计 = 556.36 + 954.00 = 1,510.36
补差合计 = 100.00
实发工资 = 10,000 - 1,510.36 = 8,489.64
```

## ⚠️ 注意事项

### 1. 数据准确性

- **员工基础工资**：必须在员工档案中填写
- **参保配置**：必须正确配置比例
- **补差记录**：确保状态为 `pending`

### 2. 计算逻辑

- 保险金额按基数×比例计算
- 补差金额按月平均分配
- 实发工资 = 应发 - 个人保险（暂不扣税）

### 3. 字段映射

- `department`：从 `employees.department` 读取
- `position`：从 `employees.position` 读取（新字段）
- `basic_salary`：从 `employees.basic_salary` 读取
