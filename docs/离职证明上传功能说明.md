# 离职证明上传功能说明

## 功能概述

为了规范离职/退休员工的管理，系统要求离职、退休、已终止状态的员工必须上传离职证明。在提交工资表审批时，系统会自动检查这些员工是否已上传离职证明，未上传则无法提交审批。

## 功能特性

### 1. 多端上传
- ✅ **小程序端**：员工可以在小程序"我的"页面上传离职证明
- ✅ **PC端**：管理员可以在人员档案中为员工上传离职证明

### 2. 多文件支持
- ✅ 支持上传多个文件
- ✅ 支持常见文件格式（PDF、图片等）
- ✅ 单个文件最大10MB

### 3. 工资表审批验证
- ✅ 提交审批前自动检查
- ✅ 离职/退休/已终止员工必须上传离职证明
- ✅ 未上传则无法提交，显示缺失员工列表

## 数据库设计

### resignation_certificates 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| employee_id | bigint | 员工ID |
| file_name | string | 文件名 |
| file_path | string | 文件路径 |
| file_type | string | 文件类型 |
| file_size | bigint | 文件大小（字节） |

| upload_source | enum | 上传来源（miniprogram/pc） |
| remark | text | 备注 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

## API 接口

### 1. 获取离职证明列表

```http
GET /api/employees/{employeeId}/resignation-certificates
Headers:
  Authorization: Bearer {token}
```

**响应：**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "employee_id": 10,
      "file_name": "离职证明.pdf",
      "file_path": "resignation_certificates/10/xxx.pdf",
      "file_type": "application/pdf",
      "file_size": 102400,
      "uploaded_by": 1,
      "upload_source": "pc",
      "remark": null,
      "created_at": "2025-12-26 10:00:00",
      "uploader": {
        "id": 1,
        "name": "管理员"
      }
    }
  ]
}
```

### 2. 上传离职证明

```http
POST /api/employees/{employeeId}/resignation-certificates/upload
Headers:
  Authorization: Bearer {token}
  Content-Type: multipart/form-data
Body:
  file: [文件]
  source: "pc" 或 "miniprogram"
  remark: "备注信息"（可选）
```

**响应：**
```json
{
  "success": true,
  "message": "上传成功",
  "data": {
    "id": 1,
    "employee_id": 10,
    "file_name": "离职证明.pdf",
    ...
  }
}
```

### 3. 删除离职证明

```http
DELETE /api/employees/resignation-certificates/{id}
Headers:
  Authorization: Bearer {token}
```

### 4. 下载离职证明

```http
GET /api/employees/resignation-certificates/{id}/download
Headers:
  Authorization: Bearer {token}
```

## 工资表审批验证

### 验证逻辑

```php
// 在提交工资表审批时
1. 获取该工资表的所有员工
2. 检查每个员工的合同状态
3. 如果状态为 terminated 或 retired
   - 检查是否已上传离职证明
   - 如果未上传，添加到缺失列表
4. 如果有缺失，返回错误，无法提交
5. 如果都已上传，允许提交审批
```

## 注意事项

### 1. 文件存储

- 文件存储在 `storage/app/public/resignation_certificates/{employee_id}/` 目录
- 文件名格式：`时间戳_随机字符串.扩展名`
- 需要运行 `php artisan storage:link` 创建符号链接

### 2. 权限控制

- 员工只能上传自己的离职证明
- 管理员可以为任何员工上传离职证明
- 删除操作需要权限验证

### 3. 文件大小限制

- 单个文件最大 10MB
- 可以在配置中调整：`'file' => 'required|file|max:10240'`

### 4. 合同状态判断

需要上传离职证明的状态：
- `terminated`：已终止
- `retired`：退休

不需要上传的状态：
- `active`：在职
- `expired`：已过期
- `unsigned`：未签署

### 5. 工资表验证时机

- 只在提交审批时验证
- 创建工资表时不验证
- 允许先创建工资表，后上传离职证明

## 常见问题

### Q1: 在职员工需要上传离职证明吗？
A: 不需要。只有离职、退休、已终止状态的员工才需要。

### Q2: 可以上传多个文件吗？
A: 可以。系统支持为同一员工上传多个离职证明文件。

### Q3: 如果员工没有小程序怎么办？
A: 管理员可以在PC端人员档案中为员工上传。

### Q4: 上传后可以删除吗？
A: 可以。但删除后如果员工在工资表中，将无法提交审批。

### Q5: 工资表中有多个离职员工怎么办？
A: 系统会列出所有未上传离职证明的员工，需要全部上传后才能提交。
