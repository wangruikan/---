# 发工资表生成逻辑说明

## 概述

发工资表是在**工资表审批通过**后自动生成的，用于记录每个员工的银行账户信息和实发工资金额，方便财务人员进行工资发放。

## 触发时机

### 自动生成
发工资表在以下情况下自动生成：

1. **工资表审批通过时**
   - 审批流程状态变为 `approved`
   - 系统自动调用生成方法
   - 为该工资表的所有员工创建发工资记录

2. **触发位置**
   - `ApprovalService.php` 的 `updateBusinessStatus()` 方法
   - 当业务类型为 `工资表审批` 且状态为 `completed` 时触发

## 生成逻辑

### 流程图

```
工资表审批通过
    ↓
获取该项目该月份的所有工资记录
    ↓
遍历每条工资记录
    ↓
检查是否已存在发工资记录（避免重复）
    ↓
获取员工信息
    ↓
创建发工资记录
    ↓
记录日志
```

### 详细步骤

#### 1. 获取工资记录

```php
$salaries = Salary::where('project_id', $salaryApproval->project_id)
                 ->where('month', $salaryApproval->month)
                 ->get();
```

**条件：**
- 项目ID：`$salaryApproval->project_id`
- 月份：`$salaryApproval->month`

#### 2. 检查是否有记录

```php
if ($salaries->isEmpty()) {
    Log::warning('工资表中没有员工记录');
    return;
}
```

**如果没有记录：**
- 记录警告日志
- 不生成发工资表
- 返回

#### 3. 遍历工资记录

```php
foreach ($salaries as $salary) {
    $employee = $salary->employee;
    if (!$employee) {
        continue; // 跳过没有员工信息的记录
    }
    
    // ... 生成逻辑
}
```

#### 4. 检查是否已存在

```php
$exists = SalaryPaymentRecord::where('salary_id', $salary->id)
                            ->exists();
if ($exists) {
    continue; // 跳过已存在的记录
}
```

**避免重复生成：**
- 检查是否已有该工资记录的发工资记录
- 如果存在，跳过
- 如果不存在，创建新记录

#### 5. 创建发工资记录

```php
SalaryPaymentRecord::create([
    'salary_id' => $salary->id,              // 工资记录ID
    'employee_id' => $employee->id,          // 员工ID
    'project_id' => $approval->project_id,   // 项目ID
    'month' => $approval->month,             // 月份
    'bank_account' => $employee->bank_account ?? '',           // 银行账号
    'bank_account_holder' => $employee->bank_account_holder ?? '', // 开户人
    'amount' => $salary->net_salary ?? 0,    // 实发工资
    'bank_name' => $employee->bank_name ?? '',               // 开户行
    'bank_province' => $employee->bank_province ?? '',       // 开户省份
    'remittance_remark' => $employee->remittance_remark ?? '', // 汇款备注
    'account_set_id' => $approval->account_set_id,           // 账套ID
]);
```

## 数据来源

### 从工资记录获取
- `salary_id`：工资记录ID
- `amount`：实发工资（`net_salary`）

### 从员工信息获取
- `employee_id`：员工ID
- `bank_account`：银行账号
- `bank_account_holder`：开户人姓名
- `bank_name`：开户行名称
- `bank_province`：开户省份
- `remittance_remark`：汇款备注

### 从审批信息获取
- `project_id`：项目ID
- `month`：工资月份
- `account_set_id`：账套ID

## 注意事项

### 1. 员工信息完整性

**必须确保员工档案中有以下信息：**
- ✅ 银行账号（`bank_account`）
- ✅ 开户人姓名（`bank_account_holder`）
- ✅ 开户行名称（`bank_name`）

**如果缺少信息：**
- 系统会使用空字符串（''）
- 不会阻止发工资记录的生成
- 但可能影响工资发放

### 2. 重复生成防护

**系统会自动检查：**
- 是否已存在该工资记录的发工资记录
- 如果存在，跳过生成
- 避免重复数据

### 3. 实发工资计算

**金额来源：**
- 使用工资表中的 `net_salary`（实发工资）
- 已扣除社保、公积金、个税等
- 是员工实际到手的金额

### 4. 账套隔离

**数据隔离：**
- 每个账套的发工资表独立
- 通过 `account_set_id` 区分
- 不同账套的数据互不影响

## 常见问题

### Q1: 发工资表什么时候生成？
A: 工资表审批通过后自动生成。

### Q2: 如果员工没有银行账号怎么办？
A: 系统会使用空字符串，不会阻止生成，但需要手动补充信息。

### Q3: 可以手动生成发工资表吗？
A: 目前只支持自动生成，不支持手动生成。

### Q4: 发工资表可以修改吗？
A: 生成后可以在前端查看和导出，但不支持修改。如需修改，需要修改员工档案后重新审批。

### Q5: 如果审批被驳回，发工资表会删除吗？
A: 不会。已生成的发工资表不会因为审批驳回而删除。

### Q6: 一个月可以生成多次发工资表吗？
A: 系统会检查是否已存在，避免重复生成。如果已存在，会跳过。
