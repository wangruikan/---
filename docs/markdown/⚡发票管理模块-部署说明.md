# 📋 发票管理模块 - 部署说明

## 🎯 功能概述

发票管理模块包含两个主要页面：

### 1. 发票项目配置
- **功能**：管理发票项目（如：应发工资、劳保费、商业险等）
- **权限**：只有审批管理中第2、3、4节点的审批人可见（经办人不可见）
- **操作**：创建、编辑、删除项目

### 2. 发票申请管理
- **功能**：创建开票任务、填写扣除明细、生成Excel表格、上传附件、提交审批
- **权限**：
  - 创建任务：只有第2、3、4节点审批人可创建
  - 业务人员：填写明细、上传附件、提交审批
- **流程**：
  1. 创建开票任务（选择年月）
  2. 添加扣除明细（选择项目、填写金额）
  3. 生成Excel扣除明细表
  4. 上传附件（必须）
  5. 提交审批（跳过第1节点，直接从第2节点开始）
  6. 如果被驳回，可以重新提交（原申请变为"红冲"状态）

```sql
-- 1. 发票项目配置表
CREATE TABLE IF NOT EXISTS `invoice_projects` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `account_set_id` BIGINT UNSIGNED NOT NULL COMMENT '账套ID',
  `project_name` VARCHAR(255) NOT NULL COMMENT '项目名称',
  `remark` TEXT NULL COMMENT '备注',
  `created_by` BIGINT UNSIGNED NOT NULL COMMENT '创建人ID',
  `created_at` TIMESTAMP NULL DEFAULT NULL COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_account_set` (`account_set_id`),
  INDEX `idx_created_by` (`created_by`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='发票项目配置表';

-- 2. 发票申请表
CREATE TABLE IF NOT EXISTS `invoice_applications` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `account_set_id` BIGINT UNSIGNED NOT NULL COMMENT '账套ID',
  `application_no` VARCHAR(50) NOT NULL COMMENT '申请单号',
  `year` INT NOT NULL COMMENT '年份',
  `month` INT NOT NULL COMMENT '月份',
  `status` VARCHAR(20) NOT NULL DEFAULT 'draft' COMMENT '状态：draft-草稿，pending-审批中，approved-已通过，rejected-已驳回，red_flushed-红冲',
  `approval_instance_id` BIGINT UNSIGNED NULL COMMENT '审批实例ID',
  `submitter_id` BIGINT UNSIGNED NOT NULL COMMENT '提交人ID（业务人员）',
  `submitted_at` TIMESTAMP NULL DEFAULT NULL COMMENT '提交时间',
  `rejection_reason` TEXT NULL COMMENT '驳回原因',
  `attachments` JSON NULL COMMENT '附件列表',
  `created_by` BIGINT UNSIGNED NOT NULL COMMENT '创建人ID',
  `created_at` TIMESTAMP NULL DEFAULT NULL COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_application_no` (`application_no`),
  INDEX `idx_account_set` (`account_set_id`),
  INDEX `idx_year_month` (`year`, `month`),
  INDEX `idx_status` (`status`),
  INDEX `idx_submitter` (`submitter_id`),
  INDEX `idx_approval` (`approval_instance_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='发票申请表';

-- 3. 发票申请明细表（扣除明细）
CREATE TABLE IF NOT EXISTS `invoice_items` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `application_id` BIGINT UNSIGNED NOT NULL COMMENT '发票申请ID',
  `invoice_project_id` BIGINT UNSIGNED NOT NULL COMMENT '项目配置ID',
  `project_name` VARCHAR(255) NOT NULL COMMENT '项目名称（冗余，用于生成Excel）',
  `sequence` INT NOT NULL DEFAULT 1 COMMENT '序号',
  `item_name` VARCHAR(255) NOT NULL COMMENT '名称（应发工资/劳保费/商业险）',
  `amount` DECIMAL(15, 2) NOT NULL DEFAULT 0.00 COMMENT '金额',
  `remark` VARCHAR(500) NULL COMMENT '备注',
  `created_at` TIMESTAMP NULL DEFAULT NULL COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_application` (`application_id`),
  INDEX `idx_invoice_project` (`invoice_project_id`),
  CONSTRAINT `fk_invoice_items_application` FOREIGN KEY (`application_id`) REFERENCES `invoice_applications` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='发票申请明细表';
```

## 🔧 功能详解

### 1. 发票项目配置

**用途：** 定义可以用于发票申请的项目类型

**字段：**
- 项目名称：如"应发工资"、"劳保费"、"商业险"
- 备注：项目说明
- 创建时间：自动记录

**操作：**
- ✅ 新建项目
- ✅ 编辑项目
- ✅ 删除项目（如果已被使用，不能删除）
- ✅ 搜索项目

### 2. 发票申请管理

#### 2.1 创建开票任务

**操作流程：**
1. 点击"创建开票任务"按钮
2. 选择年份和月份
3. 点击"确定"
4. 系统自动生成申请单号（格式：INV+日期+随机数）
5. 自动跳转到编辑页面

#### 2.2 添加扣除明细

**操作流程：**
1. 在"扣除明细"标签页中，点击"添加明细"
2. 选择项目（从项目配置中选择）
3. 填写名称（如：应发工资）
4. 填写金额
5. 填写备注（可选）
6. 点击"确定"

可以添加多条明细，系统会自动计算合计金额。

#### 2.3 生成Excel扣除明细表

**操作流程：**
1. 添加完所有明细后，点击"生成扣除明细表（Excel）"按钮
2. 系统自动生成Excel文件
3. 生成的Excel文件会自动添加到附件列表中

**Excel表格格式：**
```
┌─────────────────────────────────────────────────────────┐
│                      扣除明细表                           │
├────────┬─────────────────┬───────────┬──────────────────┤
│ 序号   │ 名称             │ 金额      │ 备注              │
├────────┼─────────────────┼───────────┼──────────────────┤
│ 1      │ 应发工资         │ 100,000.00│                  │
│ 11     │ 劳保费           │ 5,000.00  │                  │
│ 12     │ 商业险           │ 3,000.00  │                  │
├────────┼─────────────────┼───────────┼──────────────────┤
│        │ 合计             │ 108,000.00│                  │
└────────┴─────────────────┴───────────┴──────────────────┘
```

#### 2.4 上传附件

**操作流程：**
1. 切换到"附件上传"标签页
2. 点击"上传附件"按钮
3. 选择文件（支持最大10MB）
4. 上传成功后，文件会显示在附件列表中

**注意：** 必须至少上传一个附件才能提交审批

#### 2.5 提交审批

**操作流程：**
1. 确认明细和附件都已添加
2. 点击"提交审批"按钮
3. 确认提交
4. 系统创建审批实例，**直接从第2节点开始审批**（跳过第1节点/经办人）
5. 申请状态变为"审批中"

#### 2.6 审批流程

**审批节点：**
- ❌ 节点1（经办）：跳过
- ✅ 节点2（审批人）：第一级审批
- ✅ 节点3（审批人）：第二级审批
- ✅ 节点4（审批人）：第三级审批

**审批结果：**
- **通过**：申请状态变为"已通过"
- **驳回**：申请状态变为"已驳回"，可以重新提交

#### 2.7 重新提交（驳回后）

**操作流程：**
1. 在列表中找到"已驳回"的申请
2. 点击"重新提交"按钮
3. 确认重新提交
4. 系统执行以下操作：
   - 原申请状态变为"红冲"
   - 创建一个新的申请（复制所有数据）
   - 新申请自动提交审批（跳过第1节点）
   - 生成新的申请单号

**红冲说明：**
- 红冲状态的申请不能再编辑或删除
- 红冲申请会在列表中显示，用于追踪历史记录

## 🐛 常见问题

### 1. 看不到"开票管理"菜单

**原因：** 当前用户没有权限（不是第2、3、4节点审批人）

**解决：**
1. 确认当前账套是否正确
2. 确认当前用户是否被配置为审批人
3. 在审批管理中检查用户的审批节点级别

### 2. 生成Excel失败

**原因：** 缺少 PhpSpreadsheet 库

**解决：**
```bash
cd E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
composer require phpoffice/phpspreadsheet
```

### 3. 上传附件失败

**原因：** storage目录权限不足

**解决：**
```bash
# Windows系统
cd E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
php artisan storage:link
```

确保 `storage/app` 目录可写。

### 4. 提交审批后没有审批记录

**原因：** 审批节点配置不正确

**解决：**
1. 检查审批管理中的节点配置
2. 确认至少有第2、3、4节点配置了审批人
3. 查看后端日志 `storage/logs/laravel.log`

### 5. 重新提交后原申请没有变红冲

**原因：** 后端逻辑错误

**解决：**
1. 检查数据库中的 `status` 字段是否更新为 `red_flushed`
2. 清理缓存后重试

```bash
php artisan cache:clear
```

## 📊 数据库查询

### 查看所有发票项目
```sql
SELECT * FROM invoice_projects ORDER BY created_at DESC;
```

### 查看所有发票申请
```sql
SELECT 
  ia.*,
  u.name as submitter_name,
  (SELECT SUM(amount) FROM invoice_items WHERE application_id = ia.id) as total_amount
FROM invoice_applications ia
LEFT JOIN users u ON ia.submitter_id = u.id
ORDER BY ia.created_at DESC;
```

### 查看某个申请的明细
```sql
SELECT 
  ii.*,
  ip.project_name as config_project_name
FROM invoice_items ii
LEFT JOIN invoice_projects ip ON ii.invoice_project_id = ip.id
WHERE ii.application_id = 1
ORDER BY ii.sequence;
```

### 按月份统计发票金额
```sql
SELECT 
  year,
  month,
  COUNT(*) as application_count,
  SUM((SELECT SUM(amount) FROM invoice_items WHERE application_id = ia.id)) as total_amount
FROM invoice_applications ia
WHERE status IN ('approved', 'pending')
GROUP BY year, month
ORDER BY year DESC, month DESC;
```
