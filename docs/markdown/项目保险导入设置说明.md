# 项目保险导入设置说明

## 功能概述

在项目设置中新增"保险导入"字段，用于控制生成工资表时使用哪个月的保险数据。

## 设置选项

### 1. 当月（current）- 默认
- **说明**：使用当前月的保险数据
- **示例**：生成2024-10月工资表时，使用2024-10月的保险数据
- **适用场景**：保险数据与工资同步，当月参保当月扣款

### 2. 次月（next）
- **说明**：使用上个月的保险数据
- **示例**：生成2024-10月工资表时，使用2024-09月的保险数据
- **适用场景**：保险数据延后一个月，比如10月工资扣的是9月的社保

### 3. 不导入（none）
- **说明**：不导入保险数据，所有保险金额为0
- **示例**：生成工资表时，社保、公积金、大额医疗等所有保险数据都为0
- **适用场景**：项目不需要缴纳保险，或保险由其他渠道缴纳

## 使用方法

### 1. 设置项目保险导入方式

1. 进入"项目管理"页面
2. 点击项目的"编辑"按钮
3. 找到"保险导入"字段
4. 选择合适的选项：
   - 当月
   - 次月
   - 不导入
5. 点击"保存"

### 2. 生成工资表

设置完成后，生成工资表时会自动根据项目设置读取相应月份的保险数据。

## 数据逻辑

### 当月设置（current）
```
工资月份：2024-10
保险数据月份：2024-10
```

### 次月设置（next）
```
工资月份：2024-10
保险数据月份：2024-09（上个月）
```

### 不导入设置（none）
```
工资月份：2024-10
社保单位部分：0
社保个人部分：0
公积金单位部分：0
公积金个人部分：0
大额医疗单位部分：0
大额医疗个人部分：0
```

## 技术实现

### 后端逻辑（SalaryController.php）

```php
// 1. 读取项目的保险导入设置
$insuranceImportMonth = $project->insurance_import_month;

// 2. 根据设置确定使用哪个月的数据
if ($insuranceImportMonth === 'next') {
    // 使用上个月数据
    $insuranceMonth = Carbon::parse($month)->subMonth()->format('Y-m');
} elseif ($insuranceImportMonth === 'none') {
    // 不导入，设为null
    $insuranceMonth = null;
} else {
    // 使用当月数据
    $insuranceMonth = $month;
}

// 3. 根据月份获取保险数据或设为0
if ($insuranceMonth === null) {
    // 所有保险数据为0
    $insuranceData = [...全部为0...];
} else {
    // 获取指定月份的保险数据
    $insuranceData = $this->calculateMonthlyInsurance($employee, $insuranceMonth);
}
```

### 数据库字段

**projects表新增字段：**
- `insurance_import_month` ENUM('current', 'next', 'none')
- 默认值：'current'

## 注意事项

1. ⚠️ **数据一致性**：设置为"次月"时，需要确保上个月的保险数据已经存在
2. ⚠️ **首月数据**：生成1月工资表时，如果设置为"次月"，会尝试读取上年12月的数据
3. ⚠️ **不导入场景**：设置为"不导入"后，工资表中所有保险相关金额都为0，但保险明细栏目仍会显示
4. ✅ **实时生效**：修改项目设置后，下次生成工资表时立即生效
5. ✅ **已生成数据**：修改设置不影响已经生成的工资表数据

## 示例场景

### 场景1：标准缴纳（当月）
- 项目：某科技公司
- 设置：当月
- 说明：10月工资扣10月社保，数据同步

### 场景2：延后缴纳（次月）
- 项目：某外包项目
- 设置：次月
- 说明：10月工资扣9月社保，保险数据延后一个月

### 场景3：不缴纳保险（不导入）
- 项目：某短期临时项目
- 设置：不导入
- 说明：项目不需要缴纳社保公积金，保险金额全部为0
