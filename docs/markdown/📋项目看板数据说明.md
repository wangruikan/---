# 📋 项目看板数据说明

## 🎯 各项数据的含义

### 📊 统计卡片数据

1. **员工总数** - 当前账套下的员工总数
2. **项目数量** - 当前账套下的项目总数  
3. **待审批数** - 状态为"pending"的考核记录数量
4. **合同总数** - 当前账套下员工的合同总数

### 📈 项目看板数据

#### 🏗️ 项目进度含义
项目进度直接基于合同审批完成比例计算，表示项目的实际完成程度：

**计算逻辑：**
1. **合同审批完成率**（主要计算方式）
   - 进度 = (已审批通过的合同数 / 总合同数) × 100%
   - 合同状态为"approved"表示审批完成

2. **无合同时的固定值**（备用方案）
   - `completed`（已完成）：100%
   - `active`（进行中）：50%
   - `pending`（待开始）：10%
   - `paused`（已暂停）：30%

#### 👥 员工数量含义
显示分配到该项目的员工数量：

**计算逻辑：**
1. **优先级1**：如果有`employee_projects`中间表
   - 统计状态为"active"的员工数

2. **优先级2**：如果员工表有`project_id`字段
   - 统计`project_id`等于该项目ID的员工数

3. **优先级3**：智能分配算法
   - 基础分配 = 账套总员工数 ÷ 账套总项目数
   - 项目类型调整：
     - 大型/重点项目：+5人
     - 小型/试点项目：-2人
   - 状态调整：
     - `active`：+2人
     - `completed`：-3人
     - `pending`：-1人

#### 📅 项目状态含义
- **进行中** (`active`) - 项目正在执行
- **已完成** (`completed`) - 项目已完成
- **待开始** (`pending`) - 项目尚未开始
- **已暂停** (`paused`) - 项目暂时停止

## 🔄 数据更新机制

### 自动刷新
- **页面加载时**：自动加载所有数据
- **账套切换时**：重新加载对应账套的数据
- **手动刷新**：点击"刷新"按钮更新数据

### 实时性
- 统计数据：每次API调用实时计算
- 项目进度：基于时间和状态实时计算
- 员工数量：基于当前数据库状态计算

## 📍 数据来源

### 数据库表
- `employees` - 员工信息
- `projects` - 项目信息
- `assessment_records` - 考核记录
- `employee_contracts` - 员工合同
- `employee_projects` - 员工项目关联（如果存在）

### API接口
- `GET /api/dashboard/stats` - 获取统计数据
- `GET /api/dashboard/projects` - 获取项目列表
- `GET /api/dashboard/reminders` - 获取提醒事项

## 🎨 视觉说明

### 进度条颜色
- 🟢 **绿色** (80%+) - 进度良好
- 🟡 **黄色** (60-79%) - 进度正常
- 🔵 **蓝色** (40-59%) - 进度一般
- 🔴 **红色** (0-39%) - 进度较慢

### 状态标签颜色
- 🟢 **绿色** - 已完成
- 🔵 **蓝色** - 进行中
- 🟡 **黄色** - 待开始
- ⚪ **灰色** - 已暂停

## 💡 使用建议

1. **定期更新项目状态** - 确保进度计算准确
2. **维护员工项目关联** - 获得精确的员工数量
3. **及时处理待审批事项** - 减少待审批数量
4. **关注进度较慢的项目** - 红色进度条需要重点关注
