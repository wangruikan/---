# 付款申请单历史记忆功能 - 部署说明

## 📋 功能说明

实现了付款申请单的历史记忆功能，以**银行账号为唯一标识**：
- ✅ 点击银行账号输入框时，显示历史填写过的银行账号列表
- ✅ 支持按银行账号或支付对象搜索
- ✅ 选择历史记录后，自动填充所有相关字段（部门、支付对象、金额、付款方式、开户行、用途、开票情况等）
- ✅ 相同银行账号的数据会覆盖旧数据（保持最新）
- ✅ 数据存储在数据库中，跨设备、跨浏览器可用

## 🗄️ 数据库部署

### 步骤1：执行SQL创建表

在数据库中执行以下SQL（或直接执行 `database/migrations/2024_01_create_payment_form_history_table.sql`）：

```sql
CREATE TABLE IF NOT EXISTS `payment_form_history` (
  `id` bigint(20) UNSIGNED NOT NULL AUTO_INCREMENT,
  `bank_account` varchar(100) NOT NULL COMMENT '银行账号（唯一标识）',
  `department` varchar(100) DEFAULT NULL COMMENT '所在部门',
  `payee` varchar(100) DEFAULT NULL COMMENT '支付对象',
  `amount_small` varchar(50) DEFAULT NULL COMMENT '小写金额',
  `amount_large` varchar(100) DEFAULT NULL COMMENT '大写金额',
  `payment_method` json DEFAULT NULL COMMENT '付款方式（数组）',
  `bank` varchar(100) DEFAULT NULL COMMENT '开户行',
  `purpose` text DEFAULT NULL COMMENT '付款用途',
  `invoice_status` json DEFAULT NULL COMMENT '开票情况（数组）',
  `user_id` bigint(20) UNSIGNED DEFAULT NULL COMMENT '创建用户ID',
  `created_at` timestamp NULL DEFAULT NULL,
  `updated_at` timestamp NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `unique_bank_account` (`bank_account`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='付款申请单历史记忆表';
```

### 步骤2：验证表创建成功

```sql

DESC payment_form_history;
```

## 🎯 使用流程

### 用户操作流程

1. **首次填写付款申请单**
   - 用户在"付款申请单"表单中填写所有信息
   - 点击"生成PDF并添加为附件"
   - ✅ 系统自动保存该银行账号的所有数据到数据库

2. **再次填写相同银行账号**
   - 用户点击"银行账号"输入框
   - 系统显示历史银行账号列表（包含支付对象信息）
   - 用户可以：
     - 直接选择历史记录 → 自动填充所有字段
     - 输入关键词搜索 → 按银行账号或支付对象过滤
     - 手动输入新的银行账号

3. **覆盖历史数据**
   - 如果再次提交相同银行账号但数据不同
   - 系统会自动更新该银行账号的历史记录
   - 保持每个银行账号只有一条最新记录

## 🔍 故障排查

### 问题1：下拉列表不显示
**检查：**
- 浏览器控制台是否有错误
- 数据库中是否有数据：`SELECT * FROM payment_form_history;`
- API 是否返回正确：访问 `/api/payment-form-history`
- Token 是否有效

### 问题2：自动填充不工作
**检查：**
- 浏览器控制台是否有错误
- `handleSelectHistoryAccount` 函数是否被调用
- 数据格式是否正确（特别是 JSON 字段）

### 问题3：数据保存失败
**检查：**
- 数据库表是否创建成功
- `bank_account` 字段是否有值（必填）
- 后端日志：`storage/logs/laravel.log`

## 📝 注意事项

1. **银行账号唯一性**
   - 每个银行账号只保存一条记录
   - 新提交会覆盖旧数据

2. **数据安全**
   - 所有 API 都需要认证
   - 建议定期备份 `payment_form_history` 表

3. **性能优化**
   - 历史记录限制返回最近 20 条
   - 已创建索引优化查询性能

4. **兼容性**
   - 前端使用 Element Plus 的 `el-autocomplete` 组件
   - 后端使用 Laravel 的 `updateOrCreate` 方法
