## 一、入职流程

### 业务流程

```
管理员录入员工 → 上传合同 → 提交合同 → 员工短信 
→ 员工小程序 → 签署合同 → 入职登记表 → 入职
```

### 管理操作

1.**录入员工信息**：在人员档案"中，必须手机（用于小程序登录）2.**上传合同**：选择，此时为"草稿"3.**提交**：点击"提交签署"，系统自动发短信通知员工，
员工操作（）
1.**登**：手机+ 身份证后6位2.**签署**：合同→手写→ 输入身份证后4位→ 提交

## 三、审批流程

### 业务逻辑

系统4级审批：提交 → 二级审批 → 三级审批 → 四级审批 → 完成
### 

**适用场景**：人员变动需要多级审批

**流程**：
1. 经办提交2.二级审批（通过/）3.三级审批（通过/驳回）
4.四级审批（通过/驳回）
5.任何一级驳回

**查看记录**：在"流程记录管理"中查看所有审批历史

## 四、档案管理

### 业务逻辑

**核心功能**：-录入和管理员工基本信息
-管理员工合同
-查看入职登记表
导出工数据

**合同状态流转**：- 草稿 → 待签署 → 已签署 →已归档

**关键点**：
-手机号用于小程序登录，必须准确
-身份证号用于生成密码和签署验证
合同提交后系统自动发短信知员工
