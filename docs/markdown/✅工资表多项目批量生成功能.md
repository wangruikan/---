# ✅ 工资表多项目批量生成功能

## 🎯 功能说明

现在支持一次性选择多个项目，批量生成工资表。系统会自动获取所有选中项目下的员工，为他们生成工资记录。

---

## ✨ 主要变化

### 1. 项目选择支持多选
- 使用 `el-select` 的 `multiple` 属性
- 支持 `collapse-tags` 折叠标签
- 显示已选择的项目数量

### 2. 批量生成逻辑
- 依次为每个选中的项目生成工资表
- 显示成功和失败的统计
- 如果部分失败，会显示具体错误信息

### 3. 用户体验优化
- 实时显示已选择的项目数量
- 多项目选择时显示警告提示
- 生成结果清晰显示成功/失败数量

### 修改3：批量生成逻辑

**文件**：`src/views/Salaries/index.vue` 第1102-1166行

```javascript
const handleConfirmCreate = async () => {
  if (!createForm.month) {
    ElMessage.warning('请选择工资期间')
    return
  }
  if (!createForm.project_ids || createForm.project_ids.length === 0) {
    ElMessage.warning('请至少选择一个项目')
    return
  }

  try {
    loading.value = true
    
    // 依次为每个项目生成工资表
    let successCount = 0
    let failedProjects = []
    
    for (const projectId of createForm.project_ids) {
      try {
        await generateSalarySheet({
          month: createForm.month,
          project_id: projectId
        })
        successCount++
      } catch (error) {
        const projectName = availableProjects.value.find(p => p.id === projectId)?.name || `项目ID: ${projectId}`
        failedProjects.push({
          name: projectName,
          error: error.response?.data?.message || '生成失败'
        })
      }
    }
    
    // 显示结果
    if (successCount === createForm.project_ids.length) {
      ElMessage.success(`成功为 ${successCount} 个项目生成工资表`)
    } else if (successCount > 0) {
      ElMessage.warning({
        message: `成功: ${successCount} 个，失败: ${failedProjects.length} 个`,
        duration: 5000
      })
      console.error('生成失败的项目:', failedProjects)
    } else {
      ElMessage.error('所有项目工资表生成失败')
      if (failedProjects.length > 0) {
        ElMessage.error(failedProjects[0].error)
      }
    }
    
    if (successCount > 0) {
      createDialogVisible.value = false
      handleSearch()
    }
  } catch (error) {
    console.error('Generate salary sheet error:', error)
    ElMessage.error('生成工资表失败')
  } finally {
    loading.value = false
  }
}
```

## ⚠️ 注意事项

### 1. 顺序执行
项目是按顺序依次生成的，不是并发执行。这样可以：
- 避免数据库并发问题
- 准确记录每个项目的生成结果
- 更容易排查错误

### 2. 部分失败处理
如果某些项目生成失败：
- 已成功的项目工资表会保留
- 失败的项目不会生成工资表
- 可以再次尝试为失败的项目生成

### 3. 重复生成检查
后端会检查工资表是否已存在：
- 如果已存在，会提示"该项目该月的工资表已存在"
- 不会重复生成
- 需要先删除旧工资表才能重新生成

## 🔍 技术细节

### 1. 为什么使用顺序执行而不是并发？
```javascript
// 顺序执行（采用）
for (const projectId of createForm.project_ids) {
  await generateSalarySheet({ ... })
}

// 并发执行（不采用）
await Promise.all(
  createForm.project_ids.map(id => generateSalarySheet({ ... }))
)
```

**原因**：
- ✅ 避免数据库锁竞争
- ✅ 更容易追踪哪个项目失败
- ✅ 后端日志更清晰
- ✅ 资源占用更可控

### 2. 错误处理
每个项目都有独立的 try-catch，确保：
- 一个项目失败不影响其他项目
- 记录每个项目的失败原因
- 最后统一显示结果
