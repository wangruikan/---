# ✅ 工资表审批附件路径修复

## 🐛 问题描述

### 现象
在审批管理的待办列表中查看工资表审批时，点击附件无法正常查看或下载。

### 原因
前端代码在判断附件路径时，只检查了以下类型：
- `attendance-attachments` - 考勤表附件
- `process_approvals` - 流程管理附件

但没有检查 `salary_approvals`（工资表审批附件），导致工资表审批的附件路径被错误地加上了 `storage/` 前缀。

### 错误的路径
```
实际路径: salary_approvals/8/1761664920_6900df9879318.docx
错误的URL: /storage/salary_approvals/8/1761664920_6900df9879318.docx  ❌
正确的URL: /salary_approvals/8/1761664920_6900df9879318.docx  ✅
```

---

## 🔧 修复内容

### 文件位置
`src/views/Approvals/index.vue`

### 修改1：查看附件（handleViewAttachment）

**修改前：**
```javascript
const handleViewAttachment = (attachment) => {
  // ...
  
  // 判断文件存储位置：
  // - 考勤表附件：attendance-attachments（直接访问）
  // - 流程管理附件：process_approvals（直接访问）
  // - 其他附件：storage 目录
  const needsStoragePrefix = !attachment.file_path.includes('attendance-attachments') 
    && !attachment.file_path.includes('process_approvals')  // ❌ 缺少 salary_approvals
  
  const fileUrl = needsStoragePrefix
    ? `${baseURL}/storage/${attachment.file_path}`
    : `${baseURL}/${attachment.file_path}`
  
  window.open(fileUrl, '_blank')
}
```

**修改后：**
```javascript

```

## 📊 附件存储说明

### 各类型附件的存储位置

| 业务类型 | 存储目录 | 完整路径示例 | URL访问路径 |
|---------|---------|-------------|-----------|
| 考勤表 | `public/attendance-attachments/` | `attendance-attachments/123/file.pdf` | `/attendance-attachments/123/file.pdf` |
| 流程管理 | `public/process_approvals/` | `process_approvals/456/file.pdf` | `/process_approvals/456/file.pdf` |
| **工资表审批** | `public/salary_approvals/` | `salary_approvals/8/file.docx` | `/salary_approvals/8/file.docx` ✅ |
| 其他（合同等） | `storage/app/public/` | `contracts/789/file.pdf` | `/storage/contracts/789/file.pdf` |

### 路径判断逻辑

```javascript
// 判断是否需要添加 /storage/ 前缀
const needsStoragePrefix = 
  !file_path.includes('attendance-attachments') &&  // 考勤表附件直接访问
  !file_path.includes('process_approvals') &&       // 流程管理附件直接访问
  !file_path.includes('salary_approvals')           // 工资表审批附件直接访问

// 如果不包含以上路径，说明在 storage 目录，需要加前缀
const fullUrl = needsStoragePrefix 
  ? `/storage/${file_path}`   // storage目录的文件
  : `/${file_path}`            // public目录的文件
```

## 🔍 调试方法

### 如果附件还是无法访问

#### 1. 检查文件是否存在
```bash
# Windows
dir public\salary_approvals\8\

# 应该看到文件：1761664920_6900df9879318.docx
```

#### 2. 检查浏览器控制台
打开开发者工具（F12）→ Console 标签
- 查看是否有404错误
- 查看请求的URL是否正确

#### 3. 检查Network标签
1. 打开开发者工具（F12）→ Network 标签
2. 点击查看或下载附件
3. 找到对应的请求
4. 查看：
   - **Request URL**: 应该是 `/salary_approvals/8/...`
   - **Status Code**: 应该是 200
   - 如果是 404，说明文件不存在或路径错误

#### 4. 检查数据库
```sql
-- 查看附件记录
SELECT * FROM approval_attachments WHERE instance_id = 103;

-- 查看工资表审批附件
SELECT * FROM salary_approval_attachments WHERE salary_approval_id = 8;
```

两个表的 `file_path` 应该一致：
```
salary_approvals/8/1761664920_6900df9879318.docx
```

## 📝 后续优化建议

### 1. 统一附件管理
可以考虑将所有附件都存储在一个统一的目录，例如 `public/approvals/`，然后通过业务类型和ID区分：
```
public/approvals/
  ├─ salary/8/file.docx
  ├─ attendance/123/file.pdf
  ├─ process/456/file.pdf
  └─ contract/789/file.pdf
```

### 2. 使用统一的附件下载接口
可以创建一个统一的附件下载API：
```
GET /api/approvals/attachments/{id}/download
```
这样前端就不需要判断路径，后端根据附件ID自动返回文件。

### 3. 添加附件预览功能
对于常见文件类型（PDF、图片），可以在详情对话框中直接预览，而不是打开新窗口。
