# 🔍 排查 - 基数调差记录未创建问题

## 问题描述
劳动合同审批通过后，基数调差管理中没有自动创建记录，显示"暂无数据"。

## 排查步骤

### 步骤 1：检查员工基数是否已填写

#### 查询 SQL
```sql
SELECT id, name, 
       social_security_base, 
       medical_insurance_base, 
       housing_fund_base, 
       large_medical_base,
       contract_status
FROM employees
WHERE name = '张三'  -- 替换为您的员工姓名
ORDER BY created_at DESC
LIMIT 1;
```

#### 预期结果
- 基数字段应该有值（例如 5000）
- `contract_status` 应该为 `active`

#### 如果基数为空或 0
**原因**：员工创建时没有填写基数
**解决方案**：
1. 编辑员工信息
2. 填写基数（社保、医保、公积金、大额医疗）
3. 保存后重新走审批流程

---

### 步骤 2：检查合同审批是否真正通过

#### 查询 SQL
```sql
SELECT ec.id, ec.employee_id, ec.contract_type, ec.status, 
       e.name as employee_name,
       ec.created_at, ec.updated_at
FROM employee_contracts ec
LEFT JOIN employees e ON ec.employee_id = e.id
WHERE e.name = '张三'  -- 替换为您的员工姓名
ORDER BY ec.created_at DESC
LIMIT 1;
```

#### 预期结果
- `contract_type` 应该为 `labor`（劳动合同）
- `status` 应该为 `completed`（已完成）

#### 如果状态不对
**可能原因**：
- 合同类型不是劳动合同
- 审批未通过或被驳回
- 审批流程卡住了

### 步骤 4：检查 base_adjustments 表

#### 查询 SQL
```sql
SELECT * FROM base_adjustments
WHERE employee_id = (SELECT id FROM employees WHERE name = '张三' LIMIT 1)
ORDER BY created_at DESC;
```

#### 预期结果
应该有一条记录，包含：
- `status` = `effective`
- `old_social_security_base` = `0`
- `new_social_security_base` = `5000`（或您填写的值）
- `adjustment_reason` = `劳动合同审批通过，自动导入员工基数`

#### 如果表是空的
说明自动创建功能没有执行，继续下一步排查。

### 步骤 5：手动测试代码

#### 创建测试脚本
在项目根目录创建 `test_base_adjustment.php`：

```php
<?php

require __DIR__.'/vendor/autoload.php';

$app = require_once __DIR__.'/bootstrap/app.php';
$app->make('Illuminate\Contracts\Console\Kernel')->bootstrap();

// 查找员工
$employee = \App\Models\Employee::where('name', '张三')->first();

if (!$employee) {
    echo "员工不存在\n";
    exit;
}

echo "员工信息：\n";
echo "ID: {$employee->id}\n";
echo "姓名: {$employee->name}\n";
echo "社保基数: {$employee->social_security_base}\n";
echo "医保基数: {$employee->medical_insurance_base}\n";
echo "公积金基数: {$employee->housing_fund_base}\n";
echo "大额医疗基数: {$employee->large_medical_base}\n";
echo "\n";

// 检查是否有基数
if (empty($employee->social_security_base) && 
    empty($employee->medical_insurance_base) && 
    empty($employee->housing_fund_base) && 
    empty($employee->large_medical_base)) {
    echo "❌ 员工没有填写任何基数！\n";
    exit;
}

// 手动创建基数调差记录
$createData = [
    'employee_id' => $employee->id,
    'account_set_id' => $employee->account_set_id,
    'status' => 'effective',
    'adjustment_reason' => '手动测试：劳动合同审批通过，自动导入员工基数',
    'created_by' => 1,
    'applied_at' => now()
];

$effectiveDate = now()->format('Y-m-d');

if (!empty($employee->social_security_base)) {
    $createData['old_social_security_base'] = 0;
    $createData['new_social_security_base'] = $employee->social_security_base;
    $createData['social_security_effective_date'] = $effectiveDate;
}

if (!empty($employee->medical_insurance_base)) {
    $createData['old_medical_insurance_base'] = 0;
    $createData['new_medical_insurance_base'] = $employee->medical_insurance_base;
    $createData['medical_insurance_effective_date'] = $effectiveDate;
}

if (!empty($employee->housing_fund_base)) {
    $createData['old_housing_fund_base'] = 0;
    $createData['new_housing_fund_base'] = $employee->housing_fund_base;
    $createData['housing_fund_effective_date'] = $effectiveDate;
}

if (!empty($employee->large_medical_base)) {
    $createData['old_large_medical_base'] = 0;
    $createData['new_large_medical_base'] = $employee->large_medical_base;
    $createData['large_medical_effective_date'] = $effectiveDate;
}

try {
    $adjustment = \App\Models\BaseAdjustment::create($createData);
    echo "✅ 基数调差记录创建成功！\n";
    echo "记录ID: {$adjustment->id}\n";
    echo "状态: {$adjustment->status}\n";
    echo "生效日期: {$effectiveDate}\n";
} catch (\Exception $e) {
    echo "❌ 创建失败：{$e->getMessage()}\n";
    echo "详细错误：\n";
    echo $e->getTraceAsString();
}
```

#### 运行测试
```bash

```

## 常见问题

### 问题 1：员工基数为空
**原因**：创建员工时没有填写基数
**解决**：编辑员工，填写基数后重新走审批流程

### 问题 2：合同类型不对
**原因**：添加的不是劳动合同
**解决**：只有劳动合同审批通过才会自动创建基数调差记录

### 问题 3：代码未生效
**原因**：代码未部署或缓存未清除
**解决**：
1. 确认 `ApprovalService.php` 文件已更新
2. 清除所有缓存
3. 重启 PHP 服务（如果使用 PHP-FPM）

### 问题 4：权限问题
**原因**：数据库写入权限不足
**解决**：检查数据库用户权限

### 问题 5：字段不存在
**原因**：数据库表结构不完整
**解决**：运行迁移文件或手动添加字段
