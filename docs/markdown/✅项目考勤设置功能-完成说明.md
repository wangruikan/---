# ✅ 项目考勤设置功能 - 完成说明

## 📋 功能概述

实现了项目级别的考勤开关功能，允许灵活控制哪些项目需要考勤审批才能生成工资表。

---

## 🎯 核心功能

### 1. 项目设置
- **位置**：项目管理 → 编辑项目
- **字段**：`需要考勤表` 开关
- **默认值**：开启（需要考勤）
- **说明**：
  - ✅ **开启**：生成工资表前必须先审批考勤表
  - ❌ **关闭**：可直接生成工资表，无需考勤审批

### 2. 工资表生成逻辑
根据项目的考勤设置，智能判断是否需要考勤审批：

| 项目设置 | 考勤状态 | 是否可生成工资表 | 显示标签 |
|---------|---------|----------------|---------|
| 需要考勤 | 已审批 | ✅ 可以 | 项目名称 |
| 需要考勤 | 未审批 | ❌ 不可以 | 项目名称 (考勤未审批) |
| 无需考勤 | - | ✅ 可以 | 项目名称 (无需考勤) |

### 3. 用户提示优化
- 选择工资期间后，自动显示可生成工资表的项目数量
- 无可用项目时，提供明确的操作指引
- 项目下拉列表中显示项目状态标签

**方法**：`getProjectsWithApprovalStatus()`

**核心逻辑**：
```php
// 判断项目是否需要考勤（兼容两个字段名）
$requireAttendance = $project->require_attendance ?? $project->requires_attendance ?? true;

// 如果不需要考勤，则直接可以创建工资表
// 如果需要考勤，则必须考勤已审批才能创建
$isApproved = in_array($project->id, $approvedProjectIds);
$canCreate = !$requireAttendance || $isApproved;

// 生成提示标签
$label = $project->name;
if ($requireAttendance && !$isApproved) {
    $label .= ' (考勤未审批)';
} elseif (!$requireAttendance) {
    $label .= ' (无需考勤)';
}
```

**返回数据**：
```php
return [
    'id' => $project->id,
    'name' => $project->name,
    'code' => $project->code,
    'status' => $project->status,
    'period' => $period,
    'require_attendance' => $requireAttendance,  // 是否需要考勤
    'attendance_approved' => $isApproved,        // 考勤是否已审批
    'can_create_payroll' => $canCreate,          // 是否可以生成工资表
    'disabled' => !$canCreate,                   // 是否禁用（前端使用）
    'label' => $label                            // 带提示的标签
];
```

## 🔍 常见问题

### Q1: 如何修改项目的考勤设置？
**A**: 进入"项目管理" → 点击项目的"编辑"按钮 → 找到"需要考勤表"开关 → 开启或关闭 → 保存

### Q2: 修改考勤设置后，已有的工资表会受影响吗？
**A**: 不会。考勤设置只影响新生成的工资表，不影响已存在的工资表。

### Q3: 如果项目设置为"无需考勤"，还能创建考勤表吗？
**A**: 可以。考勤设置只影响工资表生成的前置条件，不影响考勤表的创建。

### Q4: 默认新建的项目需要考勤吗？
**A**: 是的。默认值为"需要考勤"，保持系统原有逻辑。
