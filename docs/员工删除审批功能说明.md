# 员工删除审批功能说明

## 功能概述

为了保护在职员工数据，系统对员工删除功能进行了权限控制和审批流程管理。

## 删除规则

### 1. 在职员工（contract_status = 'active'）

#### 管理员
- ✅ **可以直接删除**，无需审批
- 删除时会有确认提示："您是管理员，可以直接删除在职员工。确定要删除吗？"

#### 普通用户
- ❌ **不能直接删除**
- ✅ **需要提交删除审批**
- 审批通过后系统自动删除员工

### 2. 非在职员工（离职、退休等）

- ✅ **所有人都可以直接删除**，无需审批
- 删除时会有确认提示："确定要删除该员工吗？"

### 3. 有待审批的删除申请

- ❌ **任何人都不能删除**
- 🚫 **删除按钮会被隐藏**
- 需要等待审批完成后才能再次操作

## 使用流程

### 普通用户删除在职员工

1. 在员工列表中找到要删除的在职员工
2. 点击"删除"按钮
3. 系统提示："该员工为在职状态，删除需要提交审批。是否提交删除审批？"
4. 点击"提交审批"
5. 填写删除原因（必填）
6. 提交审批申请
7. 等待审批通过
8. 审批通过后，员工自动被删除

### 管理员删除在职员工

1. 在员工列表中找到要删除的在职员工
2. 点击"删除"按钮
3. 系统提示："您是管理员，可以直接删除在职员工。确定要删除吗？"
4. 点击"确定删除"
5. 员工立即被删除

### 删除非在职员工

1. 在员工列表中找到要删除的非在职员工
2. 点击"删除"按钮
3. 系统提示："确定要删除该员工吗？"
4. 点击"确定删除"
5. 员工立即被删除

## 审批流程

### 提交删除审批

**API 接口**：
```http
POST /api/employees/delete-approval
Headers:
  Authorization: Bearer {token}
  X-Account-Set-Id: {accountSetId}
Body:
{
  "employee_id": 123,
  "reason": "员工已离职，需要清理档案"
}
```

### 审批通过后

- 系统自动执行删除操作
- 员工数据从数据库中删除
- 记录操作日志

### 审批驳回后

- 员工数据保留
- 可以重新提交删除审批

显示内容：
- 警告提示：该员工为在职状态

- 删除原因输入框（必填，最多500字）
- 操作按钮：取消、提交审批

## 技术实现

### 后端

#### EmployeeController.php

**destroy 方法**：
- 检查是否有待审批的删除申请
- 检查员工是否在职
- 检查当前用户是否是管理员
- 根据规则执行删除或返回错误

**submitDeleteApproval 方法**：
- 验证员工是否在职
- 检查是否已有待审批的删除申请
- 创建审批流程
- 保存删除原因

#### ApprovalService.php

**updateBusinessStatus 方法**：
- 处理 `employee_deletion` 业务类型
- 审批通过时执行删除
- 审批驳回时记录日志

### 前端

#### src/views/Employees/index.vue

**canShowDeleteButton 方法**：
- 检查是否有待审批的删除申请
- 返回是否显示删除按钮

**handleDelete 方法**：
- 检查员工是否在职
- 检查当前用户是否是管理员
- 根据规则执行删除或打开审批对话框

**handleSubmitDeleteApproval 方法**：
- 验证删除原因
- 调用 API 提交删除审批

## 权限说明

### 管理员角色

- `admin`
- `super_admin`

管理员可以：
- 直接删除在职员工
- 直接删除非在职员工
- 查看所有删除审批

### 普通用户

普通用户可以：
- 直接删除非在职员工
- 提交在职员工的删除审批
- 查看自己提交的删除审批

## 数据库

### approval_instances 表

删除审批记录：
- `business_type`: 'employee_deletion'
- `business_id`: 员工ID
- `status`: 'pending' | 'approved' | 'rejected' | 'withdrawn'
- `remark`: 删除原因

### 查询待审批的删除申请

```sql
SELECT * FROM approval_instances
WHERE business_type = 'employee_deletion'
  AND business_id = {employee_id}
  AND status = 'pending'
```

## 注意事项

1. **删除是不可逆的**：员工删除后无法恢复，请谨慎操作
2. **关联数据**：删除员工时，相关的合同、资料等数据也会被删除
3. **审批流程**：删除审批遵循账套配置的审批流程
4. **权限控制**：确保正确配置管理员角色
5. **待审批状态**：有待审批的删除申请时，删除按钮会被隐藏

## 常见问题

### Q1: 为什么我看不到删除按钮？
A: 可能的原因：
- 该员工有待审批的删除申请
- 刷新页面后重试

### Q2: 管理员可以删除所有员工吗？
A: 是的，管理员可以直接删除任何状态的员工，无需审批。

### Q3: 删除审批被驳回后怎么办？
A: 可以重新提交删除审批，或者联系管理员直接删除。

### Q4: 删除员工后数据能恢复吗？
A: 不能。删除是永久性的，请在删除前确认。

### Q5: 如何查看删除审批的进度？
A: 在"审批管理" > "我发起的"中查看删除审批的状态。
