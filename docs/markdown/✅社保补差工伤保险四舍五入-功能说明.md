# 社保补差 - 工伤保险四舍五入处理

## 📝 功能说明

在社保补差明细生成时，对不同险种的补差金额采用不同的小数处理方式：

- **工伤保险**：保留2位小数，**四舍五入**
- **其他保险**（养老、医疗、失业等）：保留2位小数，**直接截断**（不四舍五入）

---

## 🎯 处理逻辑

### 工伤保险
```php
if (保险名称包含"工伤" 或 "工伤保险") {
    金额 = round(原始金额, 2);  // 四舍五入
}
```

**示例**：
- 原始金额：`123.456` → 保留：`123.46` ✅（向上舍入）
- 原始金额：`123.454` → 保留：`123.45` ✅（向下舍入）
- 原始金额：`123.455` → 保留：`123.46` ✅（四舍五入）

### 其他保险
```php
else {
    金额 = floor(原始金额 * 100) / 100;  // 直接截断
}
```

**示例**：
- 原始金额：`123.456` → 保留：`123.45` ✅（截断）
- 原始金额：`123.459` → 保留：`123.45` ✅（截断）
- 原始金额：`123.499` → 保留：`123.49` ✅（截断）

## 🔧 实现细节

### 修改文件
**`app/Console/Commands/ProcessBaseCompensation.php`**

### 社保补差计算（第149-183行）

```php
foreach ($socialSecurityTypes as $type) {
    // 计算补差金额
    $companyCompensation = ($newBase - $oldBase) * $companyRate * $compensationMonths;
    $personalCompensation = ($newBase - $oldBase) * $personalRate * $compensationMonths;
    
    // 获取保险名称
    $insuranceName = $type['name'] ?? $type['type_name'] ?? '未知险种';
    
    // ✅ 工伤保险：四舍五入保留2位小数
    // 其他保险：直接截断保留2位小数（不四舍五入）
    if (strpos($insuranceName, '工伤') !== false || strpos($insuranceName, '工伤保险') !== false) {
        // 工伤保险：四舍五入
        $companyAmount = round($companyCompensation, 2);
        $personalAmount = round($personalCompensation, 2);
    } else {
        // 其他保险：直接截断（向下取整到2位小数）
        $companyAmount = floor($companyCompensation * 100) / 100;
        $personalAmount = floor($personalCompensation * 100) / 100;
    }
    
    $compensatedTypes[] = [
        'name' => $insuranceName,
        'company_base' => $newBase,
        'personal_base' => $newBase,
        'company_rate' => $companyRate * 100,
        'personal_rate' => $personalRate * 100,
        'company_amount' => $companyAmount,      // ✅ 根据险种类型处理
        'personal_amount' => $personalAmount      // ✅ 根据险种类型处理
    ];
}
```

## 💡 具体案例

### 案例1：工伤保险补差

**数据**：
- 旧基数：10,000元
- 新基数：12,000元
- 补差月数：3个月
- 工伤保险费率：0.5%（单位缴纳，个人不缴）

**计算**：
```
单位补差 = (12000 - 10000) × 0.005 × 3 = 30元（正好）
个人补差 = (12000 - 10000) × 0 × 3 = 0元
```

**如果费率不同导致小数**：
- 费率：0.48%
- 单位补差 = (12000 - 10000) × 0.0048 × 3 = 28.8元
- **结果**：`28.80`元 ✅（四舍五入）

**如果是28.855**：
- **结果**：`28.86`元 ✅（四舍五入）

### 案例2：养老保险补差（其他保险）

**数据**：
- 旧基数：10,000元
- 新基数：12,000元
- 补差月数：3个月
- 养老保险费率：16%（单位）

**计算**：
```
单位补差 = (12000 - 10000) × 0.16 × 3 = 960元（正好）
```

**如果计算结果有小数**：
- 假设计算结果：`960.456`元
- **结果**：`960.45`元 ✅（直接截断，不是960.46）

**如果是960.499**：
- **结果**：`960.49`元 ✅（直接截断，不是960.50）

## 📝 注意事项

### 1. 险种识别
通过保险名称判断是否为工伤保险：
```php
if (strpos($insuranceName, '工伤') !== false || strpos($insuranceName, '工伤保险') !== false)
```

**可识别的名称**：
- "工伤保险" ✅
- "工伤" ✅
- "工伤险" ✅
- "单位工伤保险" ✅

### 2. 大小写敏感
目前使用 `strpos()` 进行判断，**大小写敏感**。

如果保险名称是"WORK_INJURY"或其他英文形式，不会被识别为工伤保险。

建议保险名称统一使用中文"工伤保险"或包含"工伤"关键字。

### 3. 仅影响新生成的补差记录
- ✅ 修改后重新生成的补差记录会使用新的处理方式
- ❌ 已经生成的历史补差记录不会自动更新

如果需要更新历史数据，需要：
1. 删除旧的补差记录
2. 重新触发补差计算
