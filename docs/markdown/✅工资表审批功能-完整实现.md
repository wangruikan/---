# 🎯 工资表审批功能 - 完整实现方案

## 📋 功能概述

将工资表集成到审批管理系统中，实现完整的审批流程，并支持发起付款申请。

###关键功能

1. **工资表审批流程**
   - 工资表可以提交审批（类似考勤审批）
   - 选择线上/线下审批（盖章方式）
   - 审批完成后可以发起付款审批

2. **付款申请分类**
   - 工资付款审批（新增）
   - 保险汇总付款审批（原有的改名）

3. **审批管理集成**
   - 在审批管理中显示工资表审批
   - 统一的审批流程

## 🗄️ 数据库变更

### 1. 执行 SQL 脚本

请执行 `create_salary_approvals_table.sql`：

```sql
-- 创建工资表审批表
CREATE TABLE `salary_approvals` (
  `id` bigint(20) UNSIGNED NOT NULL AUTO_INCREMENT,
  `account_set_id` bigint(20) UNSIGNED NOT NULL COMMENT '账套ID',
  `project_id` bigint(20) UNSIGNED NOT NULL COMMENT '项目ID',
  `month` varchar(7) NOT NULL COMMENT '工资期间（YYYY-MM）',
  `approval_type` enum('online', 'offline') NOT NULL DEFAULT 'online' COMMENT '审批方式：线上/线下',
  `status` enum('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending' COMMENT '审批状态',
  `submitted_by` bigint(20) UNSIGNED DEFAULT NULL COMMENT '提交人ID',
  `submitted_at` datetime DEFAULT NULL COMMENT '提交时间',
  `approved_by` bigint(20) UNSIGNED DEFAULT NULL COMMENT '审批人ID',
  `approved_at` datetime DEFAULT NULL COMMENT '审批时间',
  `rejection_reason` text DEFAULT NULL COMMENT '拒绝原因',
  `remarks` text DEFAULT NULL COMMENT '备注',
  `created_at` timestamp NULL DEFAULT NULL,
  `updated_at` timestamp NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_account_set_id` (`account_set_id`),
  KEY `idx_project_id` (`project_id`),
  KEY `idx_month` (`month`),
  KEY `idx_status` (`status`),
  UNIQUE KEY `unique_project_month` (`project_id`, `month`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='工资表审批记录';

-- 为工资表（salaries）添加审批实例ID字段
ALTER TABLE `salaries` 
ADD COLUMN `salary_approval_id` bigint(20) UNSIGNED DEFAULT NULL COMMENT '工资表审批ID' AFTER `insurance_import_setting`,
ADD KEY `idx_salary_approval_id` (`salary_approval_id`);

-- 修改付款申请表，添加类型字段
ALTER TABLE `payment_requests` 
ADD COLUMN `payment_type` enum('salary', 'insurance') NOT NULL DEFAULT 'insurance' COMMENT '付款类型：工资/保险' AFTER `id`,
ADD COLUMN `salary_approval_id` bigint(20) UNSIGNED DEFAULT NULL COMMENT '工资表审批ID（当payment_type=salary时使用）' AFTER `insurance_summary_id`;
```

## 🔄 审批流程

### 1. 提交工资表审批

```
工资表列表 → 点击"提交审批" → 选择审批方式（线上/线下） → 提交
```

**后端逻辑**：
1. 检查工资表是否存在
2. 检查是否已提交过审批
3. 创建审批记录
4. 更新所有相关工资记录的 `salary_approval_id`

### 2. 审批处理

```
审批管理 → 工资表审批列表 → 审批通过/拒绝
```

**后端逻辑**：
1. 检查审批状态（只能处理待审批状态）
2. 更新审批记录状态
3. 记录审批人和审批时间

### 3. 发起付款申请（待实现）

```
已审批的工资表 → 发起付款审批 → 创建付款申请记录
```

## 📝 使用说明

### 提交工资表审批

1. 进入工资管理页面
2. 找到需要审批的工资表（状态为"草稿"）
3. 点击"提交审批"按钮
4. 在弹出的对话框中：
   - 确认项目名称和工资期间
   - 选择审批方式（线上/线下）
   - 填写备注（可选）
5. 点击"提交审批"

### 审批工资表

1. 进入审批管理页面
2. 在工资表审批列表中找到待审批的记录
3. 点击"审批"按钮
4. 选择"通过"或"拒绝"
5. 填写审批意见
6. 确认审批

### 发起付款申请（待实现）

1. 在工资表列表中找到已审批的工资表
2. 点击"发起付款"按钮
3. 确认付款信息
4. 提交付款申请

---

## ⚠️ 注意事项

1. **唯一性约束**：同一个项目的同一个月份只能提交一次审批
2. **撤回限制**：只有待审批状态才能撤回，且只有提交人可以撤回
3. **审批权限**：需要确保用户具有审批权限
4. **数据一致性**：删除审批记录时会清除相关工资记录的审批ID
