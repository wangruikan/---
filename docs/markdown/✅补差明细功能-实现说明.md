# ✅ 补差明细功能 - 实现说明

## 🎯 功能需求

在工资表中显示：
1. **社保补差明细**
2. **公积金补差明细**

## ✅ 已实现的功能

### 1. 后端数据结构

补差记录存储在 `insurance_compensation_records` 表中：

```php
字段说明：
- compensation_type: 补差类型（'social_security' | 'housing_fund' | 'medical_insurance'）
- old_base: 旧基数
- new_base: 新基数
- compensation_start_month: 补差开始月份
- compensation_end_month: 补差结束月份
- compensation_months: 补差月数
- compensation_details: 补差详细明细（JSON）
- company_total: 单位补差合计
- personal_total: 个人补差合计
- total_amount: 总补差金额
```

### 2. 后端计算逻辑

#### `calculateCompensation($employee, $month)` 方法

**功能：** 计算员工在指定月份的补差金额和明细

**查询条件：**
```php
WHERE employee_id = {员工ID}
AND compensation_start_month <= {月份}
AND compensation_end_month >= {月份}
```

**返回数据结构：**
```php
[
    'total' => 补差合计金额,
    'details' => [
        'social_security' => [
            [
                'id' => 补差记录ID,
                'old_base' => 旧基数,
                'new_base' => 新基数,
                'compensation_months' => 补差月数,
                'company_total' => 单位合计,
                'personal_total' => 个人合计,
                'total_amount' => 总金额,
                'details' => [...补差详细明细]
            ]
        ],
        'housing_fund' => [
            // 同上结构
        ]
    ]
]
```

## 📊 数据流程

```
1. 前端请求 /api/salaries/details
        ↓
2. 后端查询 salaries 表
        ↓
3. 对每个员工调用 calculateCompensation()
        ↓
4. 查询 insurance_compensation_records 表
        ↓
5. 筛选该员工在该月的补差记录
        ↓
6. 按类型分类（社保、公积金）
        ↓
7. 返回详细明细到前端
```
