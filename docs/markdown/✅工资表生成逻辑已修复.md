# ✅ 工资表生成逻辑已修复

## 🎯 问题描述

用户反馈：
> 虽然在选择项目时显示"(无需考勤)"，但是点击"确定生成"按钮后，还是提示"该项目该月的考勤表未审批，无法生成工资表"。

## 🔍 问题原因

**前端选择逻辑** ✅ 已正确处理：
- `PayrollController::getProjectsWithApprovalStatus()` 方法
- 根据 `require_attendance` 判断是否需要考勤
- 正确显示"(无需考勤)"标签

**后端生成逻辑** ❌ 有问题：
- `SalaryController::generate()` 方法
- **强制检查考勤审批状态**
- **没有判断项目是否需要考勤**

导致：前端可以选择，但后端生成时被拦截。

---

## ✅ 修复方案

修改 `app/Http/Controllers/SalaryController.php` 的 `generate()` 方法：

### 修复前的逻辑：
```php
// 强制检查考勤审批（所有项目都检查）
$attendanceSheet = AttendanceSheet::where('project_id', $projectId)
    ->where('month', $month)
    ->first();

if (!$attendanceSheet || $attendanceSheet->status !== AttendanceSheet::STATUS_APPROVED) {
    return response()->json([
        'success' => false,
        'message' => '该项目该月的考勤表未审批，无法生成工资表'
    ], 422);
}
```

### 修复后的逻辑：
```php
// 1. 先获取项目信息
$project = Project::with('employees')->findOrFail($projectId);

// 2. 判断项目是否需要考勤
$requireAttendance = true; // 默认需要考勤
if (isset($project->require_attendance)) {
    $requireAttendance = (bool) $project->require_attendance;
} elseif (isset($project->requires_attendance)) {
    $requireAttendance = (bool) $project->requires_attendance;
}

// 3. 只有需要考勤的项目才检查考勤审批状态
if ($requireAttendance) {
    $attendanceSheet = AttendanceSheet::where('project_id', $projectId)
        ->where('month', $month)
        ->first();

    if (!$attendanceSheet || $attendanceSheet->status !== AttendanceSheet::STATUS_APPROVED) {
        return response()->json([
            'success' => false,
            'message' => '该项目该月的考勤表未审批，无法生成工资表'
        ], 422);
    }
}
// 4. 如果不需要考勤，直接跳过检查，继续生成工资表
```
