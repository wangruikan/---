# 人员汇总申请 - 最新业务逻辑说明

## 🔄 核心逻辑变更

### ❌ 旧逻辑（已废弃）
导入Excel时，即使人员不匹配也会继续导入，然后记录差异。

### ✅ 新逻辑（当前）
**如果发现人员不匹配，直接拒绝导入！**

---

## 📊 业务流程

### 完整流程图：

```
导入应发工资Excel
  ↓
读取Excel数据（第2行标题，第3行起数据）
  ↓
智能识别项目（通过身份证查找员工所属项目）
  ↓
获取该项目该月的工资表人员
  ↓
对比人员差异
  ├─ 完全匹配 ✅
  │   ↓
  │ 更新所有人员的应发工资
  │   ↓
  │ 返回成功："导入成功：已更新 X 条工资记录"
  │
  └─ 有差异 ❌
      ↓
    创建人员变动记录到"人员汇总申请"
      ├─ Excel多 → 创建"新增"记录
      └─ Excel少 → 创建"减少"记录
      ↓
    拒绝导入
      ↓
    返回错误提示："人员信息不匹配，请先维护人员档案！"
      ↓
    显示差异详情和变动记录已保存的提示
```

### 场景5：记录已发起流程 ⚠️

**情况：**
- 上次导入时创建了"人员新增"记录
- 该记录已提交审批（status = in_approval）
- 再次导入相同的Excel

**系统行为：**
```
❌ 人员信息不匹配，请先维护人员档案！

该项目2025-11月份的人员新增记录已发起流程申请，请先驳回后再导入

人员变动记录已保存到【人员汇总申请】模块，请前往查看并提交审批。
```

**解决方法：**
1. 到"审批管理"找到该审批
2. 驳回审批
3. 重新导入Excel → 会覆盖原记录

## 🔧 技术实现

### 关键代码片段：

```php
// 1. 对比人员差异
$missingInSalary = array_diff($excelIdCards, $salaryIdCards);
$missingInExcel = array_diff($salaryIdCards, $excelIdCards);

// 2. 如果有差异，直接拒绝导入
if (!empty($missingInSalary) || !empty($missingInExcel)) {
    // 创建人员变动记录
    createOrUpdatePersonnelChangeRequest(...);
    
    // 返回错误，拒绝导入
    return response()->json([
        'success' => false,
        'message' => '人员信息不匹配，请先维护人员档案！'
    ], 422);
}

// 3. 人员完全匹配，才执行导入
foreach ($excelData as $row) {
    $salary->update(['gross_salary' => $grossSalary]);
}
```

## 📞 常见问题

### Q: 为什么不能部分导入？
**A:** 为了保证工资表数据的完整性和一致性，必须所有人员都匹配才能导入。

### Q: 如何快速解决人员不匹配？
**A:** 
1. 查看错误提示中的人员列表
2. 到"人员汇总申请"查看详细记录
3. 提交审批并维护人员档案
4. 重新导入

### Q: 可以忽略人员差异强制导入吗？
**A:** 不可以。必须先解决人员差异，确保数据准确。
