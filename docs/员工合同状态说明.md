# 员工合同状态说明

## 概述

员工档案中的合同状态用于标识员工当前的劳动合同状态，系统支持多种状态，并且有特殊的处理逻辑。

## 合同状态类型

### 数据库支持的状态（ENUM）

数据库 `employees` 表的 `contract_status` 字段是 ENUM 类型，只支持以下3种值：

| 状态值 | 中文名称 | 说明 |
|--------|---------|------|
| `active` | 在职 | 员工正在工作，合同有效 |
| `expired` | 已过期 | 合同已到期但未续签 |
| `terminated` | 已终止 | 合同已终止（包括离职和退休） |

### 前端显示的状态

前端为了更好的用户体验，显示更多的状态类型：

| 状态值 | 中文名称 | 实际存储 | 说明 |
|--------|---------|---------|------|
| `active` | 在职 | `active` | 正常在职 |
| `expired` | 已过期 | `expired` | 合同已过期 |
| `terminated` | 已终止 | `terminated` | 一般离职 |
| `retired` | 退休 | `terminated` + `is_retired=true` | 退休离职 |
| `unsigned` | 未签署 | `null` 或空 | 未签署合同 |

## 特殊处理逻辑

### 1. 退休状态（retired）

**问题：**
- 数据库 ENUM 字段不支持 `retired` 值
- 但前端需要区分"离职"和"退休"

**解决方案：**
- 使用 `contract_status = 'terminated'` 存储
- 同时设置 `is_retired = true` 标识退休
- 通过访问器（Accessor）返回 `retired` 状态

**代码实现：**

```php
// Employee.php - 访问器
public function getContractStatusAttribute($value)
{
    // 如果是退休状态，返回 'retired'
    if (isset($this->attributes['is_retired']) && $this->attributes['is_retired']) {
        return 'retired';
    }
    
    // 检查合同是否过期
    if (isset($this->attributes['contract_end_date']) && 
        $this->contract_end_date && 
        Carbon::now()->gt($this->contract_end_date)) {
        return 'expired';
    }
    
    return $value;
}

// Employee.php - 修改器
public function setContractStatusAttribute($value)
{
    if ($value === 'retired') {
        // 设置为 terminated
        $this->attributes['contract_status'] = 'terminated';
        // 同时设置 is_retired = true
        if (Schema::hasColumn('employees', 'is_retired')) {
            $this->attributes['is_retired'] = true;
        }
    } else {
        $this->attributes['contract_status'] = $value;
    }
}
```

### 2. 合同过期自动检测

**逻辑：**
- 如果 `contract_end_date` 已过期
- 访问器自动返回 `expired` 状态
- 即使数据库中存储的是 `active`

**示例：**
```php
// 数据库存储
contract_status: 'active'
contract_end_date: '2024-12-31'

// 当前日期：2025-12-26
// 访问器返回：'expired'
```

## 状态转换流程

### 1. 新员工入职

```
未签署 (unsigned)
    ↓
签订劳动合同
    ↓
在职 (active)
```

### 2. 合同到期

```
在职 (active)
    ↓
合同到期日过期
    ↓
已过期 (expired)
    ↓
续签合同
    ↓
在职 (active)
```

### 3. 员工离职

```
在职 (active)
    ↓
签订解除协议
    ↓
已终止 (terminated)
```

### 4. 员工退休

```
在职 (active)
    ↓
签订退休解除协议
    ↓
退休 (retired)
[实际存储: terminated + is_retired=true]
```

### SQL 查询示例

```sql
-- 查询所有在职员工
SELECT * FROM employees WHERE contract_status = 'active';

-- 查询所有退休员工
SELECT * FROM employees 
WHERE contract_status = 'terminated' 
AND is_retired = 1;

-- 查询所有离职员工（不包括退休）
SELECT * FROM employees 
WHERE contract_status = 'terminated' 
AND (is_retired = 0 OR is_retired IS NULL);

-- 按状态统计
SELECT 
    CASE 
        WHEN contract_status = 'terminated' AND is_retired = 1 THEN 'retired'
        ELSE contract_status 
    END as status,
    COUNT(*) as count
FROM employees
GROUP BY status;
```

## 注意事项

### 1. 退休状态的特殊性

- ⚠️ 数据库中不存储 `retired` 值
- ✅ 使用 `terminated` + `is_retired=true` 组合
- ✅ 通过访问器返回 `retired` 状态

### 2. 合同过期自动检测

- ⚠️ 过期状态是动态计算的
- ✅ 基于 `contract_end_date` 和当前日期
- ✅ 不需要手动更新状态

### 3. 状态查询

- ⚠️ 查询退休员工时需要同时检查 `is_retired` 字段
- ⚠️ 不能直接查询 `contract_status = 'retired'`

### 4. 状态设置

- ✅ 可以直接设置 `$employee->contract_status = 'retired'`
- ✅ 修改器会自动转换为正确的存储格式

## 常见问题

### Q1: 为什么数据库中没有 retired 状态？
A: 因为使用了 ENUM 类型，后期添加新值需要修改表结构。使用 `is_retired` 字段更灵活。

### Q2: 如何区分离职和退休？
A: 通过 `is_retired` 字段区分。`is_retired=true` 表示退休，`false` 表示普通离职。

### Q3: 合同过期后需要手动更新状态吗？
A: 不需要。系统会自动检测合同是否过期，动态返回 `expired` 状态。

### Q4: 如何查询所有非在职员工？
A: 
```php
Employee::whereIn('contract_status', ['expired', 'terminated'])->get();
```

### Q5: 员工删除功能中的"在职"判断是什么？
A: 判断 `contract_status === 'active'`，在职员工需要审批才能删除。
