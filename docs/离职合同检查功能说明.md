# 离职合同检查功能说明

## 功能概述

系统会在每月最后一天自动检查所有填写了离职日期的员工，如果员工的离职合同未完成审批，则将第一个审批节点的审批人记录到考核系统中。

## 实现细节

### 1. 数据库字段

- **员工表（employees）**
  - `resignation_date`：离职日期（可为空）
  
- **合同表（employee_contracts）**
  - `contract_type`：合同类型，`'termination'` 表示解除协议合同
  - `approval_instance_id`：关联的审批实例ID
  
- **考核记录表（assessment_records）**
  - `business_type`：业务类型，新增 `'resignation_contract'`（离职合同）
  - `handler_id`：责任人ID（第一个审批人）
  - `deadline_date`：截止日期（离职日期当月的最后一天）

### 2. 检查逻辑

1. 查找所有 `resignation_date` 不为空且 `<= 当月最后一天` 的员工
2. 对于每个员工：
   - 查找其 `contract_type = 'termination'` 的合同
   - 通过 `approval_instance_id` 获取审批实例
   - 检查审批实例的 `status` 是否为 `'approved'`（已完成）
   - 如果未完成，获取第一个审批节点（`step_order = 1`）的审批人
   - 创建考核记录

### 3. 定时任务

- **执行时间**：每月最后一天晚上23:00
- **命令**：`php artisan check:resignation-contracts`
- **配置位置**：`app/Console/Kernel.php`

```php
$schedule->command('check:resignation-contracts')
         ->lastDayOfMonth('23:00')
         ->timezone('Asia/Shanghai');
```

### 4. 考核记录字段说明

| 字段 | 说明 | 示例值 |
|------|------|--------|
| business_type | 业务类型 | `'resignation_contract'` |
| business_id | 员工ID | `123` |
| business_name | 业务名称 | `"员工 张三 的离职合同"` |
| handler_id | 责任人ID | `5`（第一个审批人的ID） |
| handler_name | 责任人姓名 | `"李四"` |
| deadline_date | 截止日期 | `2025-12-31`（离职日期当月最后一天） |
| status | 状态 | `'overdue'`（已超期） |
| remark | 备注 | `"员工离职日期：2025-12-15，截止日期前未完成离职合同审批"` |

## 使用说明

### 1. 填写离职日期

在人员档案管理中，编辑员工信息时可以填写"离职日期"字段。

### 2. 提交离职合同

在合同管理中，为离职员工创建"解除协议合同"并提交审批。

### 3. 自动检查

系统会在每月最后一天自动检查，无需人工干预。

### 4. 查看考核记录

在"考勤考核 > 考核记录"中可以查看所有考核记录，包括离职合同相关的记录。

## 测试方法

### 手动测试

1. 创建一个测试员工，填写离职日期
2. 为该员工创建离职合同但不完成审批
3. 运行测试脚本：
   ```bash
   php test_resignation_check.php
   ```

### 执行检查命令

```bash
php artisan check:resignation-contracts
```

### 查看日志

检查执行日志：
```bash
tail -f storage/logs/laravel.log | grep "离职合同检查"
```

## 注意事项

1. **离职日期可以为空**：如果员工没有填写离职日期，不会被检查
2. **避免重复记录**：系统会检查是否已存在相同的考核记录，避免重复创建
3. **只检查当月及之前的离职**：只检查 `resignation_date <= 当月最后一天` 的员工
4. **需要审批实例**：离职合同必须关联审批实例才能获取审批人信息
