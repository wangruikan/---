## 📊 根本原因

### 响应拦截器的处理

在 `src/api/request.js` 中：

```javascript
// 响应拦截器
request.interceptors.response.use(
  async (response) => {
    const { data } = response
    
    // 直接返回 data
    return data  // ← 关键：返回的是 response.data
  }
)
```

### 数据流转

```
后端返回
  ↓
{success: true, data: {...}}
  ↓
响应拦截器 (return data)
  ↓
{success: true, data: {...}}  ← 这就是前端收到的
  ↓
前端使用
  ↓
response.success ✅ 正确
response.data.success ❌ 错误（undefined）
```

## 📋 统一的响应格式处理规则

### 在本项目中，所有API响应的处理方式

```javascript
// ✅ 标准写法
const response = await someApi()

// 判断成功
if (response.success) {
  // 获取消息
  const message = response.message
  // 获取数据
  const data = response.data
}

// ❌ 错误写法（不要这样写）
if (response.data.success) {  // 多了一层
  const data = response.data.data  // 多了一层
}
```

### 响应数据结构

**后端返回：**
```json
{
  "success": true,
  "message": "操作成功",
  "data": {...}
}
```

**前端收到（经过拦截器）：**
```javascript
{
  success: true,
  message: "操作成功",
  data: {...}
}
```

**使用方式：**
```javascript
response.success  // ✅ 判断成功
response.message  // ✅ 获取消息
response.data     // ✅ 获取数据
```
