# 📊 公积金补差测试方案

## 前提条件

- 员工：wrk（或其他测试员工）
- 公积金配置：标准配置
- 公司比例：假设 12% (0.12)
- 个人比例：假设 12% (0.12)
- 补差期间：2025-01 至 2025-09（9个月，假设10月测试）

---

## 方案1：基数不变，上下限提高（最常见）

### 场景
```
员工公积金基数：5000（一直没变）

第1次设置上下限：1500 - 25000
第2次修改上下限：3000 - 30000
```

### 手动设置数据

```sql
-- 1. 设置员工基数
UPDATE employees 
SET housing_fund_base = 5000 
WHERE name = 'wrk';

-- 2. 查找公积金配置ID
SELECT id, region_id, config_name, min_base_amount, max_base_amount
FROM housing_fund_configs
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');

-- 3. 设置配置的新旧上下限
UPDATE housing_fund_configs 
SET 
    old_min_base_amount = 1500,
    old_max_base_amount = 25000,
    old_limits_updated_at = '2025-09-01 10:00:00',
    new_min_base_amount = 3000,
    new_max_base_amount = 30000,
    new_limits_updated_at = NOW(),
    min_base_amount = 3000,
    max_base_amount = 30000
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');
```

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：5000
旧上下限：1500 - 25000
判断：1500 ≤ 5000 ≤ 25000 ✅
约束后旧基数：5000
```

#### 新基数（用新上下限约束）
```
原始基数：5000（本月没有调基，使用旧基数）
新上下限：3000 - 30000
判断：3000 ≤ 5000 ≤ 30000 ✅
约束后新基数：5000
```

#### 补差计算
```
基数差额 = 5000 - 5000 = 0

结果：❌ 不生成补差（约束后基数相同）
```

## 方案2：基数低于新下限（会触发补差）

### 场景
```
员工公积金基数：2500

第1次设置上下限：1500 - 25000
第2次修改上下限：3000 - 30000
```

### 手动设置数据

```sql
UPDATE employees SET housing_fund_base = 2500 WHERE name = 'wrk';

UPDATE housing_fund_configs 
SET 
    old_min_base_amount = 1500,
    old_max_base_amount = 25000,
    new_min_base_amount = 3000,
    new_max_base_amount = 30000,
    min_base_amount = 3000,
    max_base_amount = 30000
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');
```

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：2500
旧上下限：1500 - 25000
判断：1500 ≤ 2500 ≤ 25000 ✅
约束后旧基数：2500
```

#### 新基数（用新上下限约束）
```
原始基数：2500
新上下限：3000 - 30000
判断：2500 < 3000 ❌（低于下限）
约束后新基数：3000（使用下限）
```

#### 补差计算
```
基数差额 = 3000 - 2500 = 500

公积金：
- 公司：500 × 12% × 9个月 = 540
- 个人：500 × 12% × 9个月 = 540

总计：
- 公司合计：540
- 个人合计：540
- 总额：1080
```

### 验证SQL
```sql
SELECT 
    employee_name,
    old_base AS '旧基数',
    new_base AS '新基数',
    compensation_months AS '月数',
    company_total AS '公司合计',
    personal_total AS '个人合计',
    total_amount AS '总额'
FROM insurance_compensation_records
WHERE employee_name = 'wrk'
  AND compensation_type = 'housing_fund';
```

**预期结果**：
- old_base: 2500.00
- new_base: 3000.00 ✅
- compensation_months: 9
- company_total: 540.00
- personal_total: 540.00
- total_amount: 1080.00

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：3000（本月之前最近一次，2024-01）
旧上下限：1500 - 25000
判断：1500 ≤ 3000 ≤ 25000 ✅
约束后旧基数：3000
```

#### 新基数（用新上下限约束）
```
原始基数：5000（本月有调基，2025-10）
新上下限：3000 - 30000
判断：3000 ≤ 5000 ≤ 30000 ✅
约束后新基数：5000
```

#### 补差计算
```
基数差额 = 5000 - 3000 = 2000

公积金：
- 公司：2000 × 12% × 9个月 = 2160
- 个人：2000 × 12% × 9个月 = 2160

总计：
- 公司合计：2160
- 个人合计：2160
- 总额：4320
```

## 方案4：基数高于新上限（约束后降低）

### 场景
```
员工公积金基数：28000

第1次设置上下限：1500 - 30000
第2次修改上下限：3000 - 25000（上限降低了）
```

### 手动设置数据

```sql
UPDATE employees SET housing_fund_base = 28000 WHERE name = 'wrk';

UPDATE housing_fund_configs 
SET 
    old_min_base_amount = 1500,
    old_max_base_amount = 30000,
    new_min_base_amount = 3000,
    new_max_base_amount = 25000,
    min_base_amount = 3000,
    max_base_amount = 25000
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');
```

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：28000
旧上下限：1500 - 30000
判断：1500 ≤ 28000 ≤ 30000 ✅
约束后旧基数：28000
```

#### 新基数（用新上下限约束）
```
原始基数：28000
新上下限：3000 - 25000
判断：28000 > 25000 ❌（超出上限）
约束后新基数：25000（使用上限）
```

#### 补差计算
```
基数差额 = 25000 - 28000 = -3000（负数）

公积金：
- 公司：-3000 × 12% × 9个月 = -3240
- 个人：-3000 × 12% × 9个月 = -3240

总计：
- 公司合计：-3240（负数，表示应退款）
- 个人合计：-3240
- 总额：-6480
```

## 🧪 快速测试（使用Tinker）

```bash
php artisan tinker
```

```php
// 1. 查找员工和配置
$employee = App\Models\Employee::where('name', 'wrk')->first();
$config = App\Models\HousingFundConfig::find($employee->housing_fund_config_id);

echo "员工: {$employee->name}" . PHP_EOL;
echo "配置: {$config->config_name}" . PHP_EOL;
echo "当前基数: {$employee->housing_fund_base}" . PHP_EOL;

// 2. 清理旧数据
DB::table('insurance_compensation_records')
    ->where('employee_id', $employee->id)
    ->where('compensation_type', 'housing_fund')
    ->delete();

DB::table('housing_fund_configs')
    ->where('id', $config->id)
    ->update([
        'old_min_base_amount' => null,
        'old_max_base_amount' => null,
        'new_min_base_amount' => null,
        'new_max_base_amount' => null,
    ]);

// 3. 第1次设置上下限
$service = app(App\Services\BaseLimitUpdateService::class);
$result = $service->updateLimits($config, 1500, 25000);
print_r($result);

// 4. 第2次修改上下限（触发补差）
$result = $service->updateLimits($config, 3000, 30000);
if ($result['should_compensate']) {
    $compService = app(App\Services\BaseLimitCompensationService::class);
    $compResult = $compService->calculateHousingFundCompensation(
        $config->id,
        $result['old_min'],
        $result['old_max'],
        $result['new_min'],
        $result['new_max']
    );
    print_r($compResult);
}

// 5. 查看补差记录
$comp = App\Models\InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->where('compensation_type', 'housing_fund')
    ->first();

if ($comp) {
    echo "\n========== 补差记录 ==========" . PHP_EOL;
    echo "期间: {$comp->compensation_start_month} ~ {$comp->compensation_end_month}" . PHP_EOL;
    echo "月数: {$comp->compensation_months} 个月" . PHP_EOL;
    echo "旧基数: {$comp->old_base}" . PHP_EOL;
    echo "新基数: {$comp->new_base}" . PHP_EOL;
    echo "公司合计: {$comp->company_total}" . PHP_EOL;
    echo "个人合计: {$comp->personal_total}" . PHP_EOL;
    echo "总额: {$comp->total_amount}" . PHP_EOL;
}
```
