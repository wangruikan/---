## API接口说明

### 1. 获取我的入职登记表（小程序端）

**请求**
```
GET /api/mini/onboarding-form
Headers: Authorization: Bearer {token}
```

**响应**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "employee_id": 1,
    "registration_date": "2025-01-10",
    "name": "张三",
    "gender": "male",
    // ... 其他字段
  }
}
```

### 2. 提交入职登记表（小程序端）

**请求**
```
POST /api/mini/onboarding-form
Headers: Authorization: Bearer {token}
Content-Type: application/json

{
  "registration_date": "2025-01-10",
  "name": "张三",
  "gender": "male",
  "id_number": "110101199001011234",
  "declaration_agreed": true,
  // ... 其他字段
}
```

**响应**
```json
{
  "success": true,
  "message": "提交成功",
  "data": {
    // 提交后的数据
  }
}
```

### 3. 获取员工入职登记表（管理端）

**请求**
```
GET /api/employees/{id}/onboarding-form
Headers: Authorization: Bearer {token}
```

**响应**
```json
{
  "success": true,
  "data": {
    // 入职登记表数据
  }
}
```

## 表单字段说明

### 基本信息
- 登记日期（必填）
- 姓名（必填）
- 性别（必填）
- 民族
- 政治面貌
- 籍贯
- 出生年月
- 身份证号码（必填）
- 现居住地
- 户口所在地
- 婚姻状况
- 健康状况
- 身高（cm）
- 体重（kg）

### 教育信息
- 毕业学校
- 毕业时间
- 文化程度
- 所学专业
- 学位
- 技术职称

### 学习简历（可多行）
- 起止时间
- 在何学校学习
- 学习层次
- 证明人

### 工作经历（可多行）
- 起止时间
- 在何工作单位
- 主要工作内容
- 证明人

### 家庭情况（可多行）
- 姓名
- 关系
- 所在单位
- 联系电话

### 就业信息
- 岗位
- 求职地区
- 是否服从调配
- 联系地址
- 联系电话

### 其他
- 备注
- 声明同意（必填）
- 本人签名

## 使用流程

### 员工端（小程序）
1. 员工登录小程序
2. 进入"我的"页面
3. 点击"入职登记表"或"填写个人信息"
4. 填写所有必填字段
5. 可以动态添加学习简历、工作经历、家庭情况
6. 同意声明并签名
7. 提交表单

### 管理端（Web系统）
1. 登录管理系统
2. 进入"人员档案管理"
3. 找到目标员工
4. 点击"入职登记表"按钮
5. 查看完整的入职登记表信息

1. 参考 `miniprogram-onboarding-form-example.md` 文件中的示例代码

3. 在小程序的"我的"页面添加入职登记表入口
4. 确保API地址配置正确
5. 确保用户登录后可以获取token

## 注意事项

1. **数据验证**：后端已实现完整的字段验证，确保数据完整性
2. **数据隔离**：每个员工只能查看和编辑自己的入职登记表
3. **数据关联**：入职登记表与员工档案通过 `employee_id` 关联
4. **多行数据**：学习简历、工作经历、家庭情况使用JSON数组存储
5. **必填字段**：登记日期、姓名、性别、身份证号码、声明同意为必填项
