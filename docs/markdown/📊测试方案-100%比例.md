# 📊 测试方案 - 100%比例版本

## 前提条件

- **员工**：wrk
- **公司比例**：100% (1.0)
- **个人比例**：100% (1.0)
- **补差期间**：2025-01 至 2025-09（9个月，假设10月测试）

## 方案1：基数低于新下限（最简单验证）⭐ 推荐

### 场景
```
员工公积金基数：2500（一直没变）

第1次设置上下限：1500 - 25000
第2次修改上下限：3000 - 30000
```

### 设置数据（SQL）
```sql
-- 1. 清理旧数据
DELETE FROM insurance_compensation_records 
WHERE employee_name = 'wrk' 
  AND compensation_type = 'housing_fund';

-- 2. 设置员工基数
UPDATE employees SET housing_fund_base = 2500 WHERE name = 'wrk';

-- 3. 设置配置上下限
UPDATE housing_fund_configs 
SET 
    old_min_base_amount = 1500,
    old_max_base_amount = 25000,
    old_limits_updated_at = '2025-09-01 10:00:00',
    new_min_base_amount = 3000,
    new_max_base_amount = 30000,
    new_limits_updated_at = NOW(),
    min_base_amount = 3000,
    max_base_amount = 30000
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');
```

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：2500
旧上下限：1500 - 25000
约束后：2500（在范围内，不变）
```

#### 新基数（用新上下限约束）
```
原始基数：2500（本月没调基，使用旧基数）
新上下限：3000 - 30000
约束后：3000（低于下限，使用下限）✅
```

#### 补差计算
```
基数差额 = 3000 - 2500 = 500

公积金（100%比例）：
- 公司：500 × 100% × 9个月 = 4500
- 个人：500 × 100% × 9个月 = 4500

总计：
- 公司合计：4500.00
- 个人合计：4500.00
- 总额：9000.00
```

### 验证
```sql
SELECT 
    employee_name,
    old_base AS '旧基数',
    new_base AS '新基数',
    compensation_months AS '月数',
    company_total AS '公司',
    personal_total AS '个人',
    total_amount AS '总额'
FROM insurance_compensation_records
WHERE employee_name = 'wrk'
  AND compensation_type = 'housing_fund';
```

**预期结果**：
| 旧基数 | 新基数 | 月数 | 公司 | 个人 | 总额 |
|--------|--------|------|------|------|------|
| 2500.00 | 3000.00 | 9 | 4500.00 | 4500.00 | 9000.00 |

## 方案4：基数超出新上限

### 场景
```
员工基数：28000（一直没变）

第1次设置上下限：1500 - 30000
第2次修改上下限：3000 - 25000（上限降低）
```

### 设置数据
```sql
UPDATE employees SET housing_fund_base = 28000 WHERE name = 'wrk';
DELETE FROM base_adjustments WHERE employee_id = (SELECT id FROM employees WHERE name = 'wrk');

UPDATE housing_fund_configs 
SET 
    old_min_base_amount = 1500,
    old_max_base_amount = 30000,
    new_min_base_amount = 3000,
    new_max_base_amount = 25000,
    min_base_amount = 3000,
    max_base_amount = 25000
WHERE id = (SELECT housing_fund_config_id FROM employees WHERE name = 'wrk');
```

### 计算过程
```
旧基数：28000（用旧上限30000约束）→ 28000
新基数：28000（用新上限25000约束）→ 25000

基数差额 = 25000 - 28000 = -3000

公积金（100%比例）：
- 公司：-3000 × 100% × 9 = -27000
- 个人：-3000 × 100% × 9 = -27000

总计：-54000.00
```

## 🎯 快速验证公式（100%比例）

```
补差总额 = (新基数 - 旧基数) × 2 × 9

例如：
方案1：(3000 - 2500) × 2 × 9 = 500 × 18 = 9000 ✅
方案2：(3000 - 2000) × 2 × 9 = 1000 × 18 = 18000 ✅
方案3：(3000 - 5000) × 2 × 9 = -2000 × 18 = -36000 ✅
方案4：(25000 - 28000) × 2 × 9 = -3000 × 18 = -54000 ✅
```
