# Windows 服务器定时任务配置教程

## 📋 配置步骤（5分钟完成）

### 第1步：确认批处理文件

项目根目录下已有 `schedule-runner.bat` 文件，内容：
```batch
@echo off
cd /d E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
php artisan schedule:run >> NUL 2>&1
```

✅ 这个文件已经创建好了，无需修改！

---

### 第2步：打开"任务计划程序"

#### 方法1：通过搜索打开
1. 按 `Win` 键
2. 输入：`任务计划程序`
3. 点击打开

#### 方法2：通过运行打开
1. 按 `Win + R`
2. 输入：`taskschd.msc`
3. 点击"确定"

---

### 第3步：创建基本任务

1. 在右侧"操作"面板，点击 **"创建基本任务"**

![创建基本任务](https://via.placeholder.com/600x100?text=创建基本任务)

---

### 第4步：设置任务名称

- **名称**：`Laravel定时任务`
- **描述**：`每分钟执行一次Laravel的定时任务调度器`
- 点击 **"下一步"**

---

### 第5步：设置触发器

- 选择：**"每天"**
- 点击 **"下一步"**

---

### 第6步：设置开始时间

- **开始日期**：今天的日期（默认）
- **开始时间**：`00:00:00`（凌晨0点，或者当前时间）
- **间隔**：`1 天`
- 点击 **"下一步"**

---

### 第7步：选择操作

- 选择：**"启动程序"**
- 点击 **"下一步"**

---

### 第8步：设置程序路径

- **程序或脚本**：
  ```
  E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan\schedule-runner.bat
  ```
  
  💡 提示：点击"浏览"按钮，找到项目根目录下的 `schedule-runner.bat` 文件

- **起始于**（可选，留空即可）

- 点击 **"下一步"**

---

### 第9步：完成创建

- ✅ 勾选：**"当单击'完成'时，打开此任务属性的对话框"**
- 点击 **"完成"**

---

### 第10步：设置重复间隔（重要！）

在弹出的"属性"对话框中：

1. 切换到 **"触发器"** 选项卡
2. 双击刚才创建的触发器（或点击"编辑"）
3. ✅ 勾选：**"重复任务间隔"**
4. 设置间隔：**1 分钟**
5. 持续时间：**1 天**
6. 点击 **"确定"**

![重复间隔设置](https://via.placeholder.com/400x300?text=重复间隔设置)

---

### 第11步：其他设置（可选但推荐）

#### 在"常规"选项卡：
- ✅ 勾选：**"不管用户是否登录都要运行"**
- ✅ 勾选：**"使用最高权限运行"**

#### 在"条件"选项卡：
- ❌ 取消勾选：**"只有在计算机使用交流电源时才启动此任务"**
- ❌ 取消勾选：**"如果计算机改用电池电源，则停止"**

#### 在"设置"选项卡：
- ✅ 勾选：**"如果任务失败，按以下间隔重新启动"** → 设置为 **1 分钟**，重试 **3 次**
- ✅ 勾选：**"如果请求后任务还在运行，强行将其停止"** → 设置为 **1 小时**

点击 **"确定"** 保存

---

### 第12步：输入管理员密码

- 系统会要求输入管理员密码
- 输入密码后点击 **"确定"**

## ❓ 常见问题

### Q1：任务显示"上次运行结果：0x1"
**原因**：批处理文件路径错误或 PHP 环境变量未配置

**解决**：
1. 检查 `schedule-runner.bat` 文件路径是否正确
2. 确认 PHP 已加入系统环境变量（CMD 中输入 `php -v` 能显示版本）

### Q2：任务不执行
**原因**：未设置"重复间隔"

**解决**：返回第10步，确保设置了"重复任务间隔：1分钟"

### Q3：提示"找不到 php"
**原因**：PHP 未加入环境变量

**解决**：
修改 `schedule-runner.bat`，使用 PHP 完整路径：
```batch
@echo off
cd /d E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
C:\php\php.exe artisan schedule:run >> NUL 2>&1
```
（将 `C:\php\php.exe` 改为您实际的 PHP 路径）

### Q4：如何查看执行日志？
查看项目日志：
```
E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan\storage\logs\laravel.log
```

或修改 `schedule-runner.bat` 保存日志：
```batch
@echo off
cd /d E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
php artisan schedule:run >> schedule.log 2>&1
```
