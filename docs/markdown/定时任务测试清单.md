# 定时任务测试清单

## 📋 系统定时任务概览

本系统共配置了 **9个定时任务**，按执行时间分为每日任务、每月任务和月末任务。

---

## ⏰ 每日定时任务

### 1️⃣ 早上 08:00 - 检查参保入职超期情况
```bash
# 测试时间：修改系统时间为 2025-11-20 08:00:30
# 命令名称：assessment:check-insurance-deadlines
# 执行频率：每天早上8点
# 时区配置：Asia/Shanghai
```

**业务说明**：
- 检查员工参保入职是否超期
- 查找 `insurance_completion_time`（参保完成期限）已过期的员工
- 检查员工在 `InsuranceChange` 模块中的状态
- 如果不符合要求，为该员工生成考核记录

**测试要点**：
- 需要有员工的 `insurance_completion_time` 已过期
- 检查是否在 `assessment_records` 表中生成考核记录
- 考核记录关联到账套的经办人

**数据表影响**：
- `assessment_records` - 新增考核记录

---

### 2️⃣ 早上 09:00 - 处理员工调基
```bash
# 测试时间：修改系统时间为 2025-11-20 09:00:30
# 命令名称：base:process-employee-adjustments
# 执行频率：每天早上9点
# 时区配置：Asia/Shanghai
```

**业务说明**：
- 处理到达生效时间的基数调整
- 查找状态为 `pending` 且 `effective_date` 已到达的调整记录
- 调用 `$adjustment->apply()` 方法更新员工档案和参保人员表
- 成功处理后自动触发补差计算

**测试要点**：
- 需要有 `base_adjustments` 表中状态为 `pending` 且 `effective_date` 已到达的记录
- 检查员工的社保、医保、公积金基数是否更新
- 检查是否自动触发了补差计算

**数据表影响**：
- `base_adjustments` - 更新状态
- `employees` - 更新基数信息
- `insurance_personnel` - 更新参保基数

## 📅 每月定时任务

### 4️⃣ 每月 1 日 08:00 - 生成资料交付记录
```bash
# 测试时间：修改系统时间为 2025-12-01 08:00:30
# 命令名称：delivery:generate
# 执行频率：每月1日早上8点
# 时区配置：Asia/Shanghai
```

**业务说明**：
- 调用 `DocumentDeliveryService` 生成月度和季度资料交付记录
- 支持通过 `--type` 选项指定生成类型（monthly、quarterly 或 all）
- 默认生成所有类型的交付记录

**测试要点**：
- 检查 `document_deliveries` 表是否生成新的交付记录
- 月度交付每月生成，季度交付每3个月生成
- 验证交付记录的完整性

**数据表影响**：
- `document_deliveries` - 新增交付记录

---

### 5️⃣ 每月 15 日 09:00 - 合同提醒升级为考核

```bash
# 测试时间：修改系统时间为 2025-11-15 09:00:30

# 时区配置：Asia/Shanghai
```

- 检查是否生成了对应的考核记录
- 验证升级逻辑是否正确

**数据表影响**：
- `contract_reminders` - 更新状态
- `assessment_records` - 新增考核记录

### 8️⃣ 最后一天 22:00 - 检查员工合同签订情况
```bash
# 测试时间：修改系统时间为 2025-11-30 22:00:30
# 命令名称：contract:check-monthly
# 执行频率：每月最后一天晚上10点
# 时区配置：Asia/Shanghai
```

**业务说明**：
- 检查员工是否签订了相应的合同
- 劳动合同：员工在合同有效期内是否已签订
- 解除协议：离职员工是否已签订解除协议
- 退休解除协议：退休员工是否已签订退休解除协议

**测试要点**：
- 检查在职员工是否签订劳动合同
- 检查离职员工是否签订解除协议
- 检查退休员工是否签订退休解除协议
- 未签订则生成合同提醒记录

**数据表影响**：
- `employees` - 检查合同状态
- `employee_contracts` - 检查合同记录
- `contract_reminders` - 新增提醒记录

**注意事项**：
- 经办人硬编码为 `id=0, name='业务人员'`
- 需要确认这个设计是否符合业务需求

### 9️⃣ 最后一天 23:00 - 生成参保明细记录
```bash
# 测试时间：修改系统时间为 2025-11-30 23:00:30
# 命令名称：insurance:generate-monthly-details
```

**测试要点**：
- 需要有状态为 `active` 的参保人员
- 检查 `insurance_detail_records` 表是否生成当月记录
- 记录包含社保、医保、公积金等完整信息
- 验证数据快照的准确性

**数据表影响**：
- `insurance_personnel` - 读取活跃人员
- `insurance_detail_records` - 新增月度明细记录

## 💡 测试建议

### 准备工作
1. **备份数据库**：测试前备份，避免测试数据影响生产环境
2. **准备测试数据**：确保各个模块有相应的测试数据
3. **查看日志**：打开 `storage/logs/laravel.log` 监控执行日志
4. **检查权限**：确保相关用户和账套配置正确

### 测试顺序建议
1. 先测试每日任务（8:00, 9:00, 9:30）
2. 再测试每月任务（1日, 15日）
3. 最后测试月末任务（20:00, 21:00, 22:00, 23:00）

### 注意事项
1. **防重复执行**：同一时间点不要重复测试，可能不会再次执行
2. **时间跨越**：修改时间要跨过整点才能触发
3. **数据清理**：测试完成后清理测试产生的数据
4. **时间恢复**：测试完成后记得改回正确时间
