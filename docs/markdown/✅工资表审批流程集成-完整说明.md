# ✅ 工资表审批流程集成 - 完整说明

## 📋 概述

工资表审批已成功集成到现有的审批流程系统中，提交审批后会自动生成审批流程实例，并在审批管理的待办列表中显示。

---

## 🔄 工作流程

### 1. 提交审批流程

```
用户点击"提交审批"
    ↓
打开提交对话框（选择盖章方式、备注、上传附件）
    ↓
点击"提交审批"按钮
    ↓
① 创建审批记录 (salary_approvals)
    ↓
② 上传所有附件 (salary_approval_attachments)
    ↓
③ 创建审批流程实例 (approval_instances)
    ↓
④ 创建审批节点记录 (approval_records)
    ↓
⑤ 关联附件到审批实例 (approval_attachments)
    ↓
完成！审批出现在待办列表
```

### 2. 审批流程执行

- 第一级审批人在"审批管理"的待办列表中看到工资表审批
- 审批人可以：
  - 查看工资表详情
  - 查看上传的附件
  - 审批通过（进入下一级）
  - 审批拒绝（流程结束）
  - 添加审批意见
- 所有级别审批通过后，工资表审批状态变为"已审批"

| status | enum | 状态（pending/approved/rejected） |

| submitted_at | datetime | 提交时间 |
| approved_by | bigint | 审批人ID |
| approved_at | datetime | 审批时间 |
| rejection_reason | text | 拒绝原因 |
| remarks | text | 备注 |

**SQL 创建语句：**
```sql
ALTER TABLE `salary_approvals` 
ADD COLUMN `approval_instance_id` bigint(20) UNSIGNED DEFAULT NULL COMMENT '审批流程实例ID' AFTER `account_set_id`,
ADD INDEX `idx_approval_instance_id` (`approval_instance_id`);
```

### 2. 关联关系

```
salary_approvals
    ├─ approval_instance_id → approval_instances.id
    ├─ attachments → salary_approval_attachments
    └─ salaries → salaries.salary_approval_id

approval_instances
    ├─ business_type = '工资表审批'
    ├─ business_id = salary_approvals.id
    ├─ records → approval_records (审批节点)
    └─ attachments → approval_attachments (从salary_approval_attachments复制)
```

## 🚀 API 接口

### 1. 提交审批（创建审批记录）

**接口:** `POST /api/salary-approvals/submit`

**请求参数:**
```json
{
  "project_id": 1,
  "month": "2025-10",
  "approval_type": "online",  // online=线上, offline=线下
  "remarks": "备注信息",
  "current_account_set_id": 1
}
```

**响应:**
```json
{
  "success": true,
  "message": "工资表审批已提交",
  "data": {
    "id": 123,  // 审批记录ID
    "account_set_id": 1,
    "project_id": 1,
    "month": "2025-10",
    "status": "pending",
    ...
  }
}
```

### 2. 上传附件

**接口:** `POST /api/salary-approvals/attachments/upload`

**请求参数:** `multipart/form-data`
```
file: File对象
salary_approval_id: 123
current_account_set_id: 1
```

### 3. 完成提交（创建审批流程实例）⭐

**接口:** `POST /api/salary-approvals/complete-submission`

**请求参数:**
```json
{
  "salary_approval_id": 123,
  "current_account_set_id": 1
}
```

**功能:**
1. 检查审批记录是否存在
2. 检查是否已经创建过审批实例（防止重复）
3. 获取所有上传的附件
4. 调用 `ApprovalService::createApprovalInstance()`
   - 业务类型: `'工资表审批'`
   - 业务ID: `salary_approval_id`
   - 自动创建审批节点和附件关联
5. 更新 `salary_approvals.approval_instance_id`

**响应:**
```json
{
  "success": true,
  "message": "审批流程已创建",
  "data": {
    "approval": { ... },
    "instance": {
      "id": 456,  // 审批实例ID
      "business_type": "工资表审批",
      "business_id": 123,
      "current_step": 1,
      "total_steps": 2,
      "status": "pending"
    }
  }
}
```

### 2. 提交流程代码

```javascript
const handleConfirmSubmitApproval = async () => {
  if (fileList.value.length === 0) {
    ElMessage.warning('请先选择要上传的附件文件（至少1个）')
    return
  }

  try {
    // 1. 创建审批记录
    const response = await submitSalaryApproval({
      project_id: submitApprovalForm.project_id,
      month: submitApprovalForm.month,
      approval_type: submitApprovalForm.approval_type,
      remarks: submitApprovalForm.remarks
    })
    
    const approvalId = response.data.id
    
    // 2. 上传所有附件
    ElMessage.info('正在上传附件...')
    let uploadCount = 0
    for (const file of fileList.value) {
      const success = await uploadFileToServer(file, approvalId)
      if (success) uploadCount++
    }

    if (uploadCount === 0) {
      ElMessage.error('所有文件上传失败，请重试')
      return
    }

    // 3. 完成提交，创建审批流程实例 ⭐
    ElMessage.info('正在创建审批流程...')
    const completeResponse = await completeSalaryApprovalSubmission({
      salary_approval_id: approvalId,
      current_account_set_id: accountSetStore.currentAccountSetId
    })
    
    if (completeResponse.success) {
      ElMessage.success(`提交成功！已上传 ${uploadCount}/${fileList.value.length} 个文件，审批流程已创建`)
    }

    submitApprovalDialogVisible.value = false
    handleSearch()
  } catch (error) {
    ElMessage.error(error.response?.data?.message || '提交失败')
  }
}
```

## 🔧 后端核心代码

### 1. SalaryApprovalController::completeSubmission()

```php
public function completeSubmission(Request $request)
{
    $approval = SalaryApproval::with(['attachments', 'project'])->find($request->salary_approval_id);

    // 检查是否已经有审批流程实例
    if ($approval->approval_instance_id) {
        return response()->json([
            'success' => false,
            'message' => '该审批已经创建了审批流程'
        ], 422);
    }

    // 获取附件列表
    $attachments = $approval->attachments->map(function ($att) {
        return [
            'path' => $att->file_path,
            'name' => $att->filename,
            'size' => $att->file_size,
            'type' => $att->mime_type,
        ];
    })->toArray();

    // 创建审批流程实例
    $instance = $this->approvalService->createApprovalInstance(
        $approval->account_set_id,
        '工资表审批',  // 业务类型
        $approval->id, // 业务ID
        $approval->submitted_by,
        $attachments,
        false // 不跳过发起人
    );

    // 更新审批记录，关联审批实例
    $approval->update([
        'approval_instance_id' => $instance->id
    ]);

    return response()->json([
        'success' => true,
        'message' => '审批流程已创建',
        'data' => ['approval' => $approval, 'instance' => $instance]
    ]);
}
```

## 🐛 常见问题

### Q1: 提交审批后，待办列表没有显示？

**A:** 检查以下几点：
1. 是否执行了 SQL 添加 `approval_instance_id` 字段？
2. 是否清除了缓存？
3. 查看日志 `storage/logs/laravel.log`，搜索"工资表审批流程创建"
4. 检查账套是否配置了审批人（账套管理 → 审批级别）

### Q2: 提示"该账套未配置审批人"？

**A:** 需要在账套管理中设置审批级别：
1. 进入"账套管理"
2. 编辑当前账套
3. 添加审批人并设置审批级别（第一级、第二级等）
4. 保存后重新提交

### Q3: 上传附件失败？

**A:** 检查以下几点：
1. 文件大小是否超过 50MB
2. `public/salary_approvals/` 目录权限是否正确（755）
3. 查看浏览器控制台的错误信息
4. 查看后端日志 `storage/logs/laravel.log`

### Q4: 创建审批流程失败？

**A:** 查看日志中的详细错误信息：
```bash
tail -f storage/logs/laravel.log | grep "创建工资表审批流程失败"
```

常见原因：
- 审批记录不存在
- 已经创建过审批实例
- 附件数据不完整
- 账套审批人配置错误
