## 🚀 部署步骤

### 步骤 1：备份数据库（推荐）

```bash
mysqldump -u用户名 -p 数据库名 > backup_$(date +%Y%m%d).sql
```

### 步骤 2：执行数据库迁移

由于之前的迁移文件已经执行过，需要先回滚再重新执行：

```bash
# 回滚上一次迁移
php artisan migrate:rollback --step=1

# 重新执行迁移
php artisan migrate
```

或者手动执行SQL：

```sql
-- 删除旧字段
ALTER TABLE reimbursements DROP COLUMN IF EXISTS received_invoice_date;

-- 添加新字段
ALTER TABLE reimbursements 
ADD COLUMN received_invoice VARCHAR(255) NULL COMMENT '收到发票(是/否)' AFTER project,
ADD COLUMN record_status VARCHAR(255) NULL COMMENT '状态(是/否)' AFTER invoice_date,
ADD COLUMN accounting_status VARCHAR(255) NULL COMMENT '入账(是/否)' AFTER record_status;
```

### 步骤 3：清除缓存

```bash
php artisan cache:clear
php artisan config:clear
php artisan route:clear
```

### 步骤 4：前端重新编译

```bash
npm run build
```

### 步骤 5：刷新浏览器

强制刷新：`Ctrl + Shift + R` 或 `Cmd + Shift + R`

## ✨ 功能特性

### 1. **税率自动计算**
- 选择税率后自动计算扣税额、不含税金额、税金
- 选择"免税"时税率为0%
- 选择"-"时不计算税金

### 2. **项目动态加载**
- 页面加载时自动获取项目列表
- 支持搜索过滤
- 与项目管理同步

### 3. **下拉选择标准化**
- 类目、发票类型、收到发票、状态、入账都改为下拉选择
- 保证数据一致性
- 方便后续统计分析

## ⚠️ 注意事项

1. **数据兼容性**
   - 旧数据的received_invoice_date需要手动迁移或清理
   - 新字段record_status和accounting_status默认为NULL

2. **税率计算**
   - 选择"-"时，taxRate为null，不会触发计算
   - 选择"免税"时，taxRate为0，计算结果也为0

3. **项目选择**
   - 确保项目管理中有可用项目
   - 如果没有项目，下拉框会是空的

## 📊 数据迁移建议

如果有旧数据需要迁移：

```sql
-- 将旧的日期数据转换为是/否
UPDATE reimbursements 
SET received_invoice = CASE 
  WHEN received_invoice_date IS NOT NULL THEN '是' 
  ELSE '否' 
END
WHERE received_invoice IS NULL;
```
