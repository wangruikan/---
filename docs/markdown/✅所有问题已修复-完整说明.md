## 🎯 功能说明

### 基数约束规则

```
员工基数 < 地区下限 → 按地区下限计算 ✅
地区下限 ≤ 员工基数 ≤ 地区上限 → 按员工基数计算 ⭕
员工基数 > 地区上限 → 按地区上限计算 ✅
```

### 险种约束来源

| 险种 | 上下限来源 |
|------|------------|
| 社保 | 社保地区 min/max |
| 医保 | 医保地区 min/max |
| 公积金 | 公积金配置 min/max |
| 大额医疗（按基数核算） | **复用社保地区** min/max |

### 计算示例

```
原始社保基数：5500
社保地区上下限：222 ~ 333
约束后基数：333（按上限）✅

金额计算：
  养老保险（100%）：333 × 1.0 = 333.00 ✅
  类型"11"（10.01%）：333 × 0.1001 = 33.33 ✅
  大额医疗（18%）：333 × 0.18 = 59.94 ✅（复用社保上下限）
```

## ⚠️ 使用说明

### 1. 设置地区基数上下限

1. 进入"社保与医保管理"
2. 编辑社保地区，设置：
   - 最低基数：例如 3000
   - 最高基数：例如 30000
3. 编辑医保地区，设置上下限
4. 保存

### 2. 导入员工参保

1. 确保员工档案中已绑定社保/医保地区
2. 进入"参保增减管理"
3. 点击"新增变更"
4. 选择员工
5. 系统自动读取员工的地区ID和保险配置

### 3. 确认参保

1. 点击"确认处理"
2. 系统自动应用基数约束
3. 生成保险明细时使用约束后的基数

### 4. 查看明细

1. 进入"参保增减明细"
2. 查看社保明细标签页
3. 应该能看到：
   - ✅ 社保类型列（养老保险等）
   - ✅ 医保类型列（医保1等）
   - ✅ 大额医疗列
   - ✅ 所有金额按约束后的基数计算
