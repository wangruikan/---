# ✅ 工资汇总功能 - 部署说明

## 📋 功能概述

工资汇总模块会在工资表审批完成后**自动生成**汇总记录，用于统计和查看已审批的工资数据。

---

## 🎯 核心特性

### 1. **自动生成**
- ✅ 工资表审批通过后自动触发
- ✅ 无需手动创建
- ✅ 支持单项目和多项目工资表

### 2. **按项目分组**
- 单项目工资表 → 生成 1 条汇总记录
- 多项目工资表 → 按项目生成多条汇总记录

### 3. **汇总数据包含**
- 工资期间（YYYY-MM）
- 工资周期（如：19-30）
- 项目名称
- 员工人数
- 应发合计
- 实发合计
- 单位保险合计
- 个人保险合计（不含大额医疗个人部分）
- 个税合计
- 审批时间

## 🔧 实现逻辑

### 触发时机

```
工资表提交审批
  ↓
审批流程（节点1 → 节点2 → ... → 最终节点）
  ↓
最终审批人通过
  ↓
【自动触发】生成工资汇总记录
```

### 生成逻辑

```php
审批通过后：
1. 获取该工资表的所有员工工资记录
2. 按项目ID分组
3. 为每个项目生成一条汇总记录：
   - 员工人数 = COUNT(员工)
   - 应发合计 = SUM(gross_salary)
   - 实发合计 = SUM(net_salary)
   - 单位保险合计 = SUM(company_insurance_total)
   - 个人保险合计 = SUM(personal_insurance_total) // 不含大额医疗
   - 个税合计 = SUM(tax_payable_or_refundable)
```

## 📊 数据说明

### 个人保险合计计算（已优化）

**之前：**
```
个人保险合计 = 社保个人 + 公积金个人 + 大额医疗个人
```

**现在：**
```
个人保险合计 = 社保个人 + 公积金个人
```

**大额医疗：**
- ✅ 单位部分：仍然计入单位保险合计
- ❌ 个人部分：不再计入个人保险合计
- ❌ 个人部分：不再显示在工资表列中
