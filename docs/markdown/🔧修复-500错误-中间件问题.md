# 🔧 修复 500 错误 - 中间件问题

## 🐛 问题描述

访问发票管理模块时出现 500 错误：

```
invoice-projects:1 
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
加载数据失败 AxiosError
```

## 🔍 问题原因

错误不是发票模块本身的问题，而是 **`DailyAssessmentCheck` 中间件** 的问题。

### 错误日志

```
SQLSTATE[42S22]: Column not found: 1054 Unknown column 'insurance_completion_time' in 'where clause' 
(Connection: mysql, SQL: select * from `employees` where `insurance_completion_time` is not null and `insurance_completion_time` <= 2025-11-02 00:00:00)
```

### 根本原因

1. `DailyAssessmentCheck` 中间件在每个API请求时都会执行
2. 中间件查询 `employees` 表的 `insurance_completion_time` 字段
3. 但是 `employees` 表中**没有这个字段**
4. 导致SQL查询失败，返回500错误

## ✅ 解决方案

### 修改的文件

`app/Http/Middleware/DailyAssessmentCheck.php`

### 修改内容

在 `performDailyCheck()` 方法中添加字段检查：

```php
/**
 * 执行每日检查
 */
private function performDailyCheck()
{
    $today = Carbon::today();

    // ✅ 添加：检查 employees 表是否有 insurance_completion_time 字段
    try {
        $hasColumn = \Schema::hasColumn('employees', 'insurance_completion_time');
        if (!$hasColumn) {
            \Log::info('employees表中没有insurance_completion_time字段，跳过每日考核检查');
            return;
        }
    } catch (\Exception $e) {
        \Log::error('检查insurance_completion_time字段失败', ['error' => $e->getMessage()]);
        return;
    }

    // 查找所有参保完成时间已到期的员工
    $employees = Employee::whereNotNull('insurance_completion_time')
        ->where('insurance_completion_time', '<=', $today)
        ->get();
    
    // ... 后续逻辑
}
```

### 修改说明

1. ✅ 在查询前检查字段是否存在
2. ✅ 如果字段不存在，记录日志并直接返回
3. ✅ 如果检查过程出错，也记录日志并返回
4. ✅ 防止中间件错误影响其他模块

## 🔍 根本原因分析

### 为什么会出现这个问题？

`DailyAssessmentCheck` 中间件可能是之前为某个特定功能（参保考核）开发的，但：

1. 数据库表可能还没有完全创建
2. 或者是老代码，字段名已经改变
3. 或者是该功能还在开发中

### 为什么影响发票模块？

因为这个中间件在 **所有API请求** 时都会执行（注册在全局中间件中），所以：

1. 访问任何API都会触发
2. 中间件执行失败导致整个请求失败
3. 返回 500 错误
