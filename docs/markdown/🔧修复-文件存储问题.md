# 🔧 修复文件存储问题 - 附件无法访问

## ❌ 问题描述

**症状**：
- ✅ 生成 Excel 成功
- ✅ 附件列表显示文件
- ❌ 点击下载后文件是空白的
- ❌ 浏览器显示 404 或文件无法访问

**原因**：
文件保存到了 `storage/app/` 目录，但没有使用 Laravel 的 `public` 磁盘，导致文件无法通过 Web 访问。

---

## ✅ 解决方案

### 1. 修改代码使用 `public` 磁盘

#### 修改前（错误）
```php
// 保存到 storage/app/invoice_attachments/
Storage::makeDirectory('invoice_attachments');
$fullPath = storage_path('app/' . $filepath);
$writer->save($fullPath);

// URL 无法访问
'url' => Storage::url($filepath)
```

#### 修改后（正确）
```php
// 保存到 storage/app/public/invoice_attachments/
Storage::disk('public')->makeDirectory('invoice_attachments');
$fullPath = storage_path('app/public/' . $filepath);
$writer->save($fullPath);

// URL 可以访问
'url' => Storage::disk('public')->url($filepath)
```

---

### 2. 创建符号链接

Laravel 需要创建从 `public/storage` 到 `storage/app/public` 的符号链接。

#### Windows 系统（以管理员身份运行）

**方法1：使用 Artisan 命令**
```bash
cd E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
php artisan storage:link
```

**方法2：手动创建符号链接（如果 Artisan 失败）**
```powershell
# 以管理员身份运行 PowerShell
cd E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan

# 删除旧的符号链接（如果存在）
Remove-Item -Path "public\storage" -Force -ErrorAction SilentlyContinue

# 创建新的符号链接
New-Item -ItemType SymbolicLink -Path "public\storage" -Target "storage\app\public"
```

**方法3：使用 mklink（CMD，以管理员身份运行）**
```cmd
cd /d E:\project\re_li_zi_yuan\re_li_zi_yuan(1)\re_li_zi_yuan
mklink /D public\storage storage\app\public
```

## ⚠️ 常见问题

### Q1: `php artisan storage:link` 报错
**A**: 以管理员身份运行 PowerShell 或 CMD

### Q2: 符号链接创建成功，但还是 404
**A**: 检查 `.env` 文件中的 `FILESYSTEM_DISK` 配置：
```env
FILESYSTEM_DISK=public
```

### Q3: 文件下载后是空白的
**A**: 
1. 检查文件是否真的生成了（查看 `storage/app/public/invoice_attachments/`）
2. 检查文件大小是否为 0
3. 查看 Laravel 日志：`storage/logs/laravel.log`

### Q4: 权限问题
**A**: 确保 `storage` 目录有写权限：
```bash
# Windows 不需要特殊权限
# Linux/Mac 需要：
chmod -R 775 storage
```
