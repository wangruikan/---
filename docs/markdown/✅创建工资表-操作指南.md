# ✅ 创建工资表 - 操作指南

## 📋 问题说明

运行查询时报错：
```sql
1054 - Unknown column 's.gross_salary' in 'field list'
```

**原因**：`salaries` 表还没有创建

---

## 🔧 解决方案

### 步骤 1：创建 salaries 表

执行 `create_salaries_table.sql` 文件，该脚本会创建完整的工资表结构。

**方法一：在数据库工具中执行**
1. 打开 Navicat/phpMyAdmin/MySQL Workbench
2. 打开 `create_salaries_table.sql` 文件
3. 选择正确的数据库
4. 点击"运行"或"执行"

**方法二：命令行执行**
```bash
mysql -u your_username -p your_database < create_salaries_table.sql
```

## 📊 表结构说明

创建的 `salaries` 表包含以下字段：

### 基本信息
- `id` - 主键
- `account_set_id` - 账套ID
- `employee_id` - 员工ID
- `project_id` - 项目ID
- `month` - 工资期间（格式：2025-10）

### 工资构成
- `basic_salary` - 基本工资
- `allowance` - 津贴
- `overtime_pay` - 加班费
- `bonus` - 奖金
- `gross_salary` - 应发工资

### 扣款项
- `social_security` - 社保个人部分
- `housing_fund` - 公积金个人部分
- `special_deduction` - 专项扣除

### 税务相关
- `taxable_income` - 应纳税所得额
- `personal_tax` - 个人所得税
- `actual_tax` - 实际个税

### 实发工资
- `net_salary` - 实发工资
- `paid_salary` - 实际发放金额

### 状态管理
- `status` - 状态（draft, submitted, approved, paid, rejected）

### 审批流程
- `submitted_by` - 提交人ID
- `approved_by` - 审批人ID
- `submitted_at` - 提交时间
- `approved_at` - 审批时间
- `paid_at` - 发放时间
- `rejection_reason` - 拒绝原因

### 时间戳
- `created_at` - 创建时间
- `updated_at` - 更新时间

## 🔍 索引优化

表已创建以下索引：

1. **唯一索引**：`uk_salary (account_set_id, employee_id, project_id, month)`
   - 确保同一员工在同一项目的同一期间只有一条记录

2. **普通索引**：
   - `idx_account_set_id` - 按账套查询
   - `idx_employee_id` - 按员工查询
   - `idx_project_id` - 按项目查询
   - `idx_month` - 按期间查询
   - `idx_status` - 按状态查询
   - `idx_project_month` - 按项目+期间组合查询（最常用）

## ⚠️ 注意事项

1. **唯一约束**：同一账套下，同一员工在同一项目的同一期间只能有一条工资记录
2. **状态流转**：draft → submitted → approved → paid（或 rejected）
3. **数据完整性**：建议启用外键约束（脚本中已注释，按需启用）
