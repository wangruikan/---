# 修复项目列表加载错误

## ❌ 错误信息
```
TypeError: Cannot read properties of null (reading 'id')
at ConfigList.vue:474
```

## 🔍 问题原因

在 `ConfigList.vue` 和 `DeliveryList.vue` 中，`loadProjects()` 方法直接将 API 返回的 `res.data` 赋值给 `projectList`。

但是 `getProjects` API 可能返回分页数据（`{data: [...], total: ...}`）而不是直接的数组，导致在模板中循环时出错。

## ✅ 已修复

修改了两个文件的 `loadProjects` 方法，添加了数据格式判断：

### 修复后的代码
```javascript
const loadProjects = async () => {
  try {
    const res = await getProjects({
      current_account_set_id: accountSetStore.currentAccountSetId
    })
    if (res.success) {
      // 确保 res.data 是数组，如果是分页数据则取 data 属性
      if (Array.isArray(res.data)) {
        projectList.value = res.data
      } else if (res.data && Array.isArray(res.data.data)) {
        projectList.value = res.data.data
      } else {
        projectList.value = []
      }
    }
  } catch (error) {
    console.error('Load projects error:', error)
    projectList.value = []
  }
}
```

### 修改的文件
- ✅ `src/views/DocumentDelivery/ConfigList.vue`
- ✅ `src/views/DocumentDelivery/DeliveryList.vue`
