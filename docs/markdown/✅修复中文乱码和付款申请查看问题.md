# ✅ 修复中文乱码和付款申请查看问题

## 🐛 问题1：PDF中文乱码

### 原因
jsPDF默认不支持中文字体，直接使用会显示乱码。

### 解决方案
使用 `html2canvas` 将HTML渲染为图片，然后插入PDF。

## 🐛 问题2：付款申请查看不了

### 可能的原因

1. **后端API返回错误**
2. **数据格式不匹配**
3. **权限问题**

### 排查步骤

#### 步骤1：查看浏览器控制台错误

1. 打开付款申请列表页面
2. 按 **F12** 打开开发者工具
3. 点击"查看"按钮
4. 查看 **Console** 标签页的错误信息
5. 查看 **Network** 标签页的API请求

#### 步骤2：检查API响应

在Network标签页中：
1. 找到 `/api/payment-applications/11` 这样的请求
2. 查看 **Response** 标签
3. 看看返回的数据是什么

**如果返回错误**：
- `success: false` - 后端逻辑错误
- `404` - 数据不存在
- `500` - 服务器错误

#### 步骤3：检查后端日志

查看Laravel日志文件：
```
storage/logs/laravel.log
```

查找最新的错误信息。

## 🔧 常见错误及解决方法

### 错误1：`付款申请不存在`

**原因**：数据库中没有这条记录

**解决**：
1. 检查数据库 `payment_requests` 表
2. 确认ID是否存在
3. 可能需要重新创建付款申请

### 错误2：`success: false, message: "该汇总流程已发起过付款申请"`

**原因**：重复发起付款申请

**解决**：
```sql
-- 删除已有的付款申请记录
DELETE FROM payment_requests WHERE insurance_summary_id = 11;
```

然后重新发起。

### 错误3：`Undefined property: stdClass::$type_name`

**原因**：后端返回的数据缺少 `type_name` 字段

**解决**：检查后端 `PaymentApplicationController@show` 方法
