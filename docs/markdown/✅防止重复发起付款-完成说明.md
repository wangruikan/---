# 防止重复发起付款功能 - 完成说明

## ✅ 功能已完成！

我已经添加了防止重复发起付款的功能。现在如果已经发起过付款申请，将不再显示"发起付款"按钮，而是显示付款申请的当前状态。

## 🎯 实现的功能

### 1. 后端检查

**文件：** `app/Http/Controllers/SalaryController.php`

**修改内容：**
- 在返回工资表列表时，检查是否已发起过付款申请
- 添加 `has_payment_request` 字段（是否已发起付款）
- 添加 `payment_request_status` 字段（付款申请状态）

```php
// 检查是否已经发起过付款申请
$paymentRequest = \App\Models\PaymentRequest::where('salary_approval_id', $approval->id)
                                            ->where('payment_type', 'salary')
                                            ->first();
if ($paymentRequest) {
    $item->has_payment_request = true;
    $item->payment_request_status = $paymentRequest->status;
} else {
    $item->has_payment_request = false;
    $item->payment_request_status = null;
}
```

### 2. 前端显示逻辑

**文件：** `src/views/Salaries/index.vue`

**修改内容：**
- 只有未发起付款时才显示"发起付款"按钮
- 已发起付款时显示付款申请的状态标签

```vue
<!-- 未发起付款：显示发起付款按钮 -->
<el-button 
  v-if="!scope.row.has_payment_request"
  link 
  type="primary" 
  size="small" 
  @click="handleCreatePayment(scope.row)"
>
  发起付款
</el-button>

<!-- 已发起付款：显示付款状态 -->
<el-tag v-if="scope.row.payment_request_status === 'pending'" type="warning">
  付款待审批
</el-tag>
<el-tag v-if="scope.row.payment_request_status === 'approved'" type="success">
  付款已批准
</el-tag>
<el-tag v-if="scope.row.payment_request_status === 'rejected'" type="danger">
  付款已驳回
</el-tag>
<el-tag v-if="scope.row.payment_request_status === 'paid'" type="info">
  已付款
</el-tag>
```

## 🔄 状态流转

```
工资表生成
    ↓
提交审批
    ↓
审批通过 → 显示"已审批"标签 + "发起付款"按钮
    ↓
发起付款申请
    ↓
付款待审批 → 显示"已审批"标签 + "付款待审批"标签
    ↓
付款审批通过 → 显示"已审批"标签 + "付款已批准"标签
    ↓
付款完成 → 显示"已审批"标签 + "已付款"标签
```
