## 🎯 核心功能

### ✨ 考勤审批限制

```
生成工资表时：
1. 选择期间（如：2025-10）
2. 系统自动查询该期间考勤已审批的项目
3. 只能选择考勤已审批的项目 ✅
4. 未审批项目置灰不可选
```

### 📊 工资表管理

| 期间 | 项目 | 人数 | 应发合计 | 实发合计 | 状态 |
|------|------|------|---------|---------|------|
| 2025-10 | 项目A | 50 | ¥250,000 | ¥200,000 | 草稿 |

### 🔄 状态流转

```
草稿 (draft)
  ↓ 提交审批
已提交 (submitted)
  ↓ 审批通过
已审批 (approved)
  ↓ 标记发放
已发放 (paid)
```

## 🚀 API接口（待实现）

### 前端已调用的接口

1. **获取项目列表（带审批状态）**
```
GET /api/payroll/projects-with-approval?period=2025-10
```

2. **获取工资表列表**
```
GET /api/salaries?month=2025-10&project_id=1&status=draft
```

3. **生成工资表**
```
POST /api/salaries/generate
Body: { month: '2025-10', project_id: 1 }
```

4. **获取工资明细**
```
GET /api/salaries/details?project_id=1&month=2025-10
```

5. **提交审批**
```
POST /api/salaries/submit
Body: { project_id: 1, month: '2025-10' }
```

6. **审批通过**
```
POST /api/salaries/approve
Body: { project_id: 1, month: '2025-10' }
```

7. **审批拒绝**
```
POST /api/salaries/reject
Body: { project_id: 1, month: '2025-10', reason: '原因' }
```

8. **标记发放**
```
POST /api/salaries/pay
Body: { project_id: 1, month: '2025-10' }
```

9. **删除**
```
DELETE /api/salaries?project_id=1&month=2025-10
```

创建 `app/Http/Controllers/SalaryController.php`：
- `index()` - 获取工资表列表

### 2. 核心逻辑

**生成工资表**：
```php
1. 验证考勤是否已审批
2. 查询该项目所有员工
3. 为每个员工生成一条 salaries 记录
4. 自动计算工资：
   - 应发 = 基本工资 + 津贴 + 加班费 + 奖金
   - 读取社保公积金（从 insurance_detail_records）
   - 计算个税
   - 实发 = 应发 - 社保 - 公积金 - 个税
```

**状态流转**：
```php
更新该项目该月所有员工的工资状态
```

## 🔍 使用流程

### 管理员操作

1. **生成工资表**
   ```
   薪金计算 → 生成工资表 → 选择期间 → 选择项目 → 确定
   系统为该项目所有员工生成工资记录（草稿状态）
   ```

2. **提交审批**
   ```
   查看明细 → 提交审批
   状态变为：已提交
   ```

3. **审批通过**
   ```
   管理员/审批人 → 审批通过
   状态变为：已审批
   ```

4. **标记发放**
   ```
   已审批 → 标记发放
   状态变为：已发放
   ```
