# ✅ 测试 - 自动创建基数调差记录

## 测试目标
验证当新员工的劳动合同审批通过后，系统是否自动在基数调差管理中创建已生效的记录。

## 前置条件
1. ✅ 已部署最新代码
2. ✅ 系统中至少有一个项目
3. ✅ 系统中至少有一个审批流程

## 测试步骤

### 步骤 1：创建新员工
1. 进入"人员档案管理"页面
2. 点击"新增员工"按钮
3. 点击"示例填充"按钮（自动填充所有数据）
4. 确认填充
5. 验证基数信息已填充：
   - 社保基数：5000
   - 医保基数：5000
   - 公积金基数：5000
   - 大额医疗基数：5000
6. 点击"保存"按钮
7. ✅ 员工创建成功

### 步骤 2：添加劳动合同
1. 在员工列表中找到刚创建的员工
2. 点击"编辑"按钮
3. 切换到"合同管理"标签页
4. 点击"添加劳动合同"
5. 上传合同文件
6. 点击"保存"
7. ✅ 合同添加成功

### 步骤 3：提交合同审批
1. 在合同列表中找到刚添加的合同
2. 点击"提交审批"按钮
3. 选择审批流程
4. 点击"确定"
5. ✅ 审批提交成功

### 步骤 4：审批通过
1. 进入"审批管理"页面
2. 找到刚提交的合同审批
3. 点击"审批"按钮
4. 选择"通过"
5. 填写审批意见（可选）
6. 点击"确定"
7. ✅ 审批通过

### 步骤 5：验证基数调差记录
1. 进入"基数调差管理"页面
2. 在员工列表中找到刚创建的员工
3. 验证以下内容：

#### ✅ 应该看到的内容
- **员工姓名**：张三（或您填写的姓名）
- **员工标签**：已调整
- **社保基数**：
  - 当前：¥0
  - 调整后：¥5000
- **医保基数**：
  - 当前：¥0
  - 调整后：¥5000
- **公积金基数**：
  - 当前：¥0
  - 调整后：¥5000
- **大额医疗基数**：
  - 当前：¥0
  - 调整后：¥5000
- **生效时间**：当前日期
- **状态**：已生效（绿色标签）

#### ❌ 不应该看到的内容
- 暂无数据
- 状态为"待生效"

### 步骤 6：查看调整历史
1. 点击该员工的"历史"按钮
2. 验证历史记录中有一条记录
3. 验证记录内容：
   - 社保基数：¥0 → ¥5000
   - 医保基数：¥0 → ¥5000
   - 公积金基数：¥0 → ¥5000
   - 大额医疗基数：¥0 → ¥5000
   - 状态：已生效
   - 创建时间：合同审批通过的时间
   - 实际生效时间：合同审批通过的时间

## 预期结果

### ✅ 成功标准
1. 员工创建成功，基数信息已保存
2. 劳动合同添加成功
3. 合同审批提交成功
4. 合同审批通过
5. **基数调差管理中自动创建了该员工的记录**
6. **记录状态为"已生效"**
7. **旧基数为 0，新基数为 5000**
8. **生效时间为当前日期**
9. 历史记录中有一条记录

### ❌ 失败情况
如果出现以下情况，说明功能有问题：
1. 基数调差管理中没有该员工的记录
2. 记录状态为"待生效"而不是"已生效"
3. 基数值不正确
4. 生效时间不正确

## 故障排查

### 问题 1：基数调差管理中没有记录
**可能原因**：
1. 后端代码未部署
2. 合同审批未真正通过
3. 员工未填写基数信息

**排查步骤**：
1. 检查后端日志：
   ```bash
   Get-Content -Path "storage\logs\laravel.log" -Tail 100 | Select-String -Pattern "自动创建基数调差"
   ```
2. 检查是否有错误日志：
   ```bash
   Get-Content -Path "storage\logs\laravel.log" -Tail 100 | Select-String -Pattern "ERROR"
   ```
3. 检查员工表中的基数字段是否有值：
   ```sql
   SELECT id, name, social_security_base, medical_insurance_base, housing_fund_base, large_medical_base 
   FROM employees 
   WHERE name = '张三';
   ```

### 问题 2：记录状态为"待生效"
**可能原因**：
1. 前端代码未更新
2. 浏览器缓存未清除

**解决方案**：
1. 清除浏览器缓存
2. 强制刷新页面（Ctrl + F5）
3. 重新构建前端：`npm run build`

### 问题 3：基数值不正确
**可能原因**：
1. 示例填充功能未正确填充基数
2. 员工保存时基数未保存

**排查步骤**：
1. 检查员工表中的基数值
2. 检查基数调差表中的基数值
3. 对比两者是否一致

## 重要修改说明

### 后端修改
- **文件**：`app/Http/Controllers/BaseAdjustmentController.php`
- **修改**：第 110 行，将查询条件从 `['pending', 'applied']` 改为 `['pending', 'applied', 'effective']`
- **原因**：自动创建的记录状态为 `effective`，需要包含在查询中

### 前端修改
- **文件**：`src/views/BaseAdjustment/index.vue`
- **修改**：
  - 第 206 行：状态显示逻辑支持 `effective` 状态
  - 第 501 行：历史记录状态显示逻辑支持 `effective` 状态
- **原因**：前端需要能够识别和显示 `effective` 状态
