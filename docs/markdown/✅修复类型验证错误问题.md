# ✅ 修复类型验证错误问题

## 🐛 问题描述

用户在编辑社保/医保地区时报错：
```json
{
  "success": false,
  "message": "验证失败",
  "errors": {
    "min_base_amount": ["The min base amount field is required."],
    "max_base_amount": ["The max base amount field is required."]
  }
}
```

## 🔍 问题原因

在之前的实现中，我们将基数上下限从"类型级别"移到了"地区级别"。但是在社保/医保类型的添加和更新方法中，**验证规则仍然要求 `min_base_amount` 和 `max_base_amount` 为必填字段**。

这导致前端提交类型数据时，因为没有这两个字段而验证失败。

## 🔧 修复内容

### 修改的文件

1. **`app/Http/Controllers/SocialSecurityController.php`**
2. **`app/Http/Controllers/MedicalInsuranceController.php`**

### 修改的方法

#### 社保控制器（4处修改）

1. **`addType()` 方法 - 验证规则**
   ```php
   // 修改前
   $validator = Validator::make($request->all(), [
       'name' => 'required|string|max:100',
       'min_base_amount' => 'required|numeric|min:0',  // ❌ 移除
       'max_base_amount' => 'required|numeric|min:0',  // ❌ 移除
       'employee_ratio' => 'required|numeric|min:0|max:1',
       'company_ratio' => 'required|numeric|min:0|max:1',
   ]);
   
   // 修改后
   $validator = Validator::make($request->all(), [
       'name' => 'required|string|max:100',
       'employee_ratio' => 'required|numeric|min:0|max:1',
       'company_ratio' => 'required|numeric|min:0|max:1',
   ]);
   ```

2. **`addType()` 方法 - 创建逻辑**
   ```php
   // 修改前
   $type = SocialSecurityType::create([
       'region_id' => $regionId,
       'name' => $request->name,
       'min_base_amount' => $request->min_base_amount,  // ❌ 移除
       'max_base_amount' => $request->max_base_amount,  // ❌ 移除
       'employee_ratio' => $request->employee_ratio,
       'company_ratio' => $request->company_ratio,
       'created_by' => $user->id,
   ]);
   
   // 修改后
   $type = SocialSecurityType::create([
       'region_id' => $regionId,
       'name' => $request->name,
       'employee_ratio' => $request->employee_ratio,
       'company_ratio' => $request->company_ratio,
       'created_by' => $user->id,
   ]);
   ```

3. **`updateType()` 方法 - 验证规则**（同 addType）

4. **`updateType()` 方法 - 更新逻辑**
   ```php
   // 修改前
   $type->update([
       'name' => $request->name,
       'min_base_amount' => $request->min_base_amount,  // ❌ 移除
       'max_base_amount' => $request->max_base_amount,  // ❌ 移除
       'employee_ratio' => $request->employee_ratio,
       'company_ratio' => $request->company_ratio,
   ]);
   
   // 修改后
   $type->update([
       'name' => $request->name,
       'employee_ratio' => $request->employee_ratio,
       'company_ratio' => $request->company_ratio,
   ]);
   ```

#### 医保控制器（4处修改）

与社保控制器完全相同的修改：
1. `addType()` 方法的验证规则
2. `addType()` 方法的创建逻辑
3. `updateType()` 方法的验证规则
4. `updateType()` 方法的更新逻辑

## 🎯 业务逻辑

### 当前架构

```
社保地区
  ├── min_base_amount（地区级别的下限）
  ├── max_base_amount（地区级别的上限）
  └── 社保类型列表
        ├── 类型1（养老保险）- 只有费率，无上下限
        ├── 类型2（医疗保险）- 只有费率，无上下限
        └── ...

医保地区
  ├── min_base_amount（地区级别的下限）
  ├── max_base_amount（地区级别的上限）
  └── 医保类型列表
        ├── 类型1（职工医保）- 只有费率，无上下限
        ├── 类型2（补充医保）- 只有费率，无上下限
        └── ...
```

### 计算逻辑

当计算保险金额时：
1. 获取员工的基数
2. 获取地区的上下限
3. 应用约束（低于下限按下限，高于上限按上限）
4. 使用约束后的基数 × 类型费率 = 金额
