# ✅ 负数补差显示 - 修复完成

## 🐛 问题描述

工资表中：
- ✅ 公积金补差显示正常（正数：¥3,600.00）
- ❌ 社保补差不显示（负数：-¥4,950.45）

## 🔍 根本原因

前端代码中使用了 `v-if="getCompensationAmount(...) > 0"` 条件判断：

```vue
<!-- ❌ 修复前 - 只显示正数 -->
<span v-if="getCompensationAmount(row, 'social_security', 'company') > 0" 
      style="color: #F56C6C;">
  {{ formatMoney(getCompensationAmount(row, 'social_security', 'company')) }}
</span>
<span v-else>-</span>
```

**问题**：
- 社保补差金额：`-4950.45`（负数，表示基数下调）
- 条件 `> 0` → false
- 显示 `-` 而不是 `-¥4,950.45` ❌

## ✅ 修复方案

将条件改为 `!== 0`，只要不为0就显示（包括负数）：

```vue
<!-- ✅ 修复后 - 显示正数和负数 -->
<span v-if="getCompensationAmount(row, 'social_security', 'company') !== 0" 
      style="color: #F56C6C;">
  {{ formatMoney(getCompensationAmount(row, 'social_security', 'company')) }}
</span>
<span v-else>-</span>
```

## 🎯 负数的业务含义

### 为什么会有负数补差？

当基数**下调**时，会产生负数补差：

```
示例：
旧基数: ¥3,500
新基数: ¥3,000
差额: -¥500（下调了）

补差月数: 9个月
补差金额: -¥500 × 比例 × 9个月 = -¥4,950.45（负数）
```

### 负数的含义

- **正数补差**（如公积金 ¥3,600）：基数上调，需要**补缴**
- **负数补差**（如社保 -¥4,950.45）：基数下调，需要**退款/冲抵**

### 会计处理

```
10月工资计算：
  基本工资: ¥8,100.00
  社保补差: -¥4,950.45  ← 退款，增加实发
  公积金补差: ¥3,600.00  ← 补缴，减少实发
  净补差: -¥1,350.45
  实发工资: ¥8,100 - 正常扣款 - ¥1,350.45 = ...
```

## 🔄 条件对比

| 条件 | 正数 (3600) | 负数 (-4950.45) | 零 (0) | 说明 |
|------|------------|----------------|--------|------|
| `> 0` | ✅ 显示 | ❌ 不显示 | ❌ 不显示 | 只显示正数 |
| `!== 0` | ✅ 显示 | ✅ 显示 | ❌ 不显示 | 显示正数和负数 |
| 无条件 | ✅ 显示 | ✅ 显示 | ✅ 显示 ¥0.00 | 全部显示 |

我们使用 `!== 0` 是最合理的：
- ✅ 显示有补差的情况（正数或负数）
- ✅ 不显示没有补差的情况（零）
