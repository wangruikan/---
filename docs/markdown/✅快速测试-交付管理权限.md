## 📝 如何设置用户审批级别

### 方法1：通过系统界面（推荐）

1. 使用管理员账号登录
2. 进入「账套管理」
3. 选择某个账套，点击「管理成员」
4. 为用户设置「审批级别」：
   - 1 = 经办
   - 2 = 初审
   - 3 = 复审
   - 4 = 终审

### 方法2：直接修改数据库

```sql
-- 查看当前用户的审批级别
SELECT 
    u.id,
    u.name,
    asu.account_set_id,
    asu.approval_level,
    asu.approval_level_name
FROM users u
JOIN account_set_users asu ON u.id = asu.user_id
WHERE u.email = 'test@example.com';

-- 设置用户为"经办"（级别1，无权限）
UPDATE account_set_users
SET approval_level = 1,
    approval_level_name = '经办'
WHERE user_id = 1  -- 替换为实际用户ID
  AND account_set_id = 1;  -- 替换为实际账套ID

-- 设置用户为"初审"（级别2，有权限）
UPDATE account_set_users
SET approval_level = 2,
    approval_level_name = '初审'
WHERE user_id = 2
  AND account_set_id = 1;
```

## 🔍 调试技巧

### 1. 检查后端权限API

在浏览器开发者工具 → Network 中查看：

```
Request:
GET /api/delivery-configs/check-access?current_account_set_id=1

Response (有权限):
{
  "success": true,
  "has_access": true
}

Response (无权限):
{
  "success": true,
  "has_access": false
}
```

### 2. 检查用户审批级别

在浏览器 Console 中执行：

```javascript
// 查看当前账套ID
console.log('当前账套:', accountSetStore.currentAccountSetId)

// 手动调用权限检查
fetch('/api/delivery-configs/check-access?current_account_set_id=1', {
  headers: {
    'Authorization': 'Bearer ' + localStorage.getItem('token')
  }
})
.then(r => r.json())
.then(data => console.log('权限检查结果:', data))
```

### 3. 数据库查询验证

```sql
-- 查看所有用户的审批级别
SELECT 
    u.id as user_id,
    u.name,
    u.email,
    asu.account_set_id,
    asu.approval_level,
    asu.approval_level_name,
    CASE 
        WHEN asu.approval_level IN (2, 3, 4) THEN '有权限'
        ELSE '无权限'
    END as delivery_access
FROM users u
JOIN account_set_users asu ON u.id = asu.user_id
ORDER BY asu.account_set_id, asu.approval_level;
```

## 🐛 常见问题

### Q1: 登录后看不到「交付配置」菜单项

**原因**：
- 用户的 approval_level 为 1 或 NULL（这是正常的，经办人员本来就看不到）
- 只有审批级别 2、3、4 的用户才能看到配置菜单

**解决**：
```sql
-- 检查用户级别
SELECT * FROM account_set_users 
WHERE user_id = ? AND account_set_id = ?;

-- 如果需要授予配置权限，设置为审批人员
UPDATE account_set_users 
SET approval_level = 2, approval_level_name = '初审'
WHERE user_id = ? AND account_set_id = ?;
```

### Q2: 设置了级别但菜单还是不显示

**解决步骤**：
1. 清除浏览器缓存（Ctrl + Shift + Delete）
2. 刷新页面（F5 或 Ctrl + R）
3. 重新登录
4. 检查浏览器 Console 是否有错误

### Q3: 切换账套后菜单没有更新

**检查**：
- 浏览器 Console 是否有错误
- Network 是否成功调用了 `/api/delivery-configs/check-access`
- 响应数据是否正确

**临时解决**：刷新页面
