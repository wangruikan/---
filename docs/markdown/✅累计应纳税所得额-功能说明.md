# 累计应纳税所得额 - 功能说明

## 📝 字段说明

**字段名**：累计应纳税所得额  
**数据库字段**：`taxable_income`  
**作用**：用于计算个人所得税的累计应纳税所得额

---

## 🎯 计算公式

```
累计应纳税所得额 = 累计收入 
                - 累计减除费用 
                - 累计专项扣除 
                - 累计专项附加扣除
```

**详细说明**：
- **累计收入**：当年1月到当前月的应发工资合计
- **累计减除费用**：5000 × 实际工作月数
- **累计专项扣除**：社保公积金个人部分的累计值
- **累计专项附加扣除**：6项扣除（子女教育等）的累计值

---

## 💡 计算示例

### 案例1：全年正常工作的员工（2025年9月）

**数据**：
```
累计收入（1-9月）：¥90,000
累计减除费用：¥45,000（5000 × 9个月）
累计专项扣除：¥12,600（社保¥7,200 + 公积金¥5,400）
累计专项附加扣除：¥36,000（每月¥4,000 × 9个月）
```

**计算**：
```
累计应纳税所得额 = 90,000 - 45,000 - 12,600 - 36,000
                = -3,600元
```

**结果**：累计应纳税所得额为负数，按0处理，不需要缴税 ✅

---

### 案例2：高收入员工（2025年9月）

**数据**：
```
累计收入（1-9月）：¥270,000
累计减除费用：¥45,000（5000 × 9个月）
累计专项扣除：¥18,000（社保¥10,800 + 公积金¥7,200）
累计专项附加扣除：¥18,000（每月¥2,000 × 9个月）
```

**计算**：
```
累计应纳税所得额 = 270,000 - 45,000 - 18,000 - 18,000
                = 189,000元
```

**结果**：累计应纳税所得额为¥189,000，需要按此金额计算个税 ✅

---

### 案例3：年中入职的员工（2025年5月入职，计算9月工资）

**数据**：
```
累计收入（5-9月）：¥50,000
累计减除费用：¥25,000（5000 × 5个月，从5月开始）
累计专项扣除：¥7,000（社保¥4,000 + 公积金¥3,000）
累计专项附加扣除：¥10,000（每月¥2,000 × 5个月）
```

**计算**：
```
累计应纳税所得额 = 50,000 - 25,000 - 7,000 - 10,000
                = 8,000元
```

**结果**：累计应纳税所得额为¥8,000 ✅

## 🔄 后端计算逻辑

**文件**：`app/Http/Controllers/SalaryController.php`（第736-744行）

```php
// 累计应纳税所得额
$cumulativeTaxableIncome = $cumulativeIncomeWithCurrent     // 累计收入
    - $cumulativeBasicDeduction                             // 累计减除费用
    - $cumulativeSocialSecurity                             // 累计社保
    - $cumulativeHousingFund                                // 累计公积金
    - $cumulativeSpecialDeduction;                          // 累计专项附加扣除（6项）

// 如果累计应纳税所得额为负数，则为0
$cumulativeTaxableIncome = max(0, $cumulativeTaxableIncome);
```

**说明**：
- 如果计算结果为负数，按0处理
- 这个值用于后续计算税率和速算扣除数

## 📊 在个税计算中的应用

### 个税计算流程

```
1. 计算累计应纳税所得额 ← 这个字段
   ↓
2. 根据累计应纳税所得额确定税率和速算扣除数
   ↓
3. 计算应纳税额 = 累计应纳税所得额 × 税率 - 速算扣除数
   ↓
4. 计算本月应扣税额 = 本月应纳税额 - 已扣税额
```

### 个税税率表（年度综合所得）

| 累计应纳税所得额 | 税率 | 速算扣除数 |
|---------------|------|-----------|
| 不超过36,000元 | 3% | 0 |
| 超过36,000至144,000元 | 10% | 2,520 |
| 超过144,000至300,000元 | 20% | 16,920 |
| 超过300,000至420,000元 | 25% | 31,920 |
| 超过420,000至660,000元 | 30% | 52,920 |
| 超过660,000至960,000元 | 35% | 85,920 |
| 超过960,000元 | 45% | 181,920 |

**示例**：
```
累计应纳税所得额：¥189,000

查表：
- 超过144,000元，不超过300,000元
- 税率：20%
- 速算扣除数：¥16,920

应纳税额 = 189,000 × 20% - 16,920 = 37,800 - 16,920 = ¥20,880
```

## 🎯 重要特性

### 1. 负数处理
```php
$cumulativeTaxableIncome = max(0, $cumulativeTaxableIncome);
```

如果计算结果为负数（扣除项大于收入），则按0处理。

**示例**：
```
累计收入：¥50,000
累计减除费用：¥45,000
累计专项扣除：¥12,600
累计专项附加扣除：¥5,000

计算结果：50,000 - 45,000 - 12,600 - 5,000 = -12,600
最终结果：¥0（不是负数）
```

### 2. 累计计算
这个字段是**累计值**，不是当月值：
- ✅ 反映的是1月到当前月的累计情况
- ✅ 用于年度个税预扣预缴
- ✅ 符合国家个税累计预扣法的要求

### 3. 与税率的关系
累计应纳税所得额直接决定了税率：
- 0元：税率0%
- 1-36,000元：税率3%
- 36,001-144,000元：税率10%
- 144,001-300,000元：税率20%
- ...
