# 审批撤回功能说明

## 功能概述

本功能为审批管理系统添加了两个重要特性：
1. **自动跳过第一个审批人**：如果第一个审批人就是发起人本人，系统会自动跳过该步骤
2. **撤回功能**：发起人可以在特定条件下撤回自己发起的审批

## 一、自动跳过第一个审批人

### 功能说明
当创建审批流程时，系统会自动检测第一个审批人是否是发起人本人。如果是，则：
- 自动将第一步标记为"已审批"
- 自动推进到第二个审批人
- 记录日志方便追踪

### 实现位置
- **后端**：`app/Services/ApprovalService.php` 的 `createApprovalInstance` 方法
- **逻辑**：在创建审批记录时自动检测并处理

### 示例场景
```
审批流程配置：
- 第1级：张三（发起人）
- 第2级：李四
- 第3级：王五

当张三发起审批时：
✓ 第1级自动通过（张三）
→ 直接进入第2级（李四待审批）
```

## 二、撤回功能

### 功能说明
发起人可以撤回自己发起的审批流程，但需要满足以下条件：

### 撤回条件
1. **必须是发起人**：只有发起人本人才能撤回
2. **审批状态为待审批**：流程必须还在进行中（status = 'pending'）
3. **第二个审批人未审批**：第二个审批人的状态不能是"已审批"
4. **流程在第二步或之前**：不能已经进入第三步或更后

### 不可撤回的情况
- ❌ 第二个审批人已经审批
- ❌ 审批已完成（approved）
- ❌ 审批已驳回（rejected）
- ❌ 流程已进入第三步或更后

### 使用方式

#### 前端操作
1. 进入"审批管理"页面
2. 切换到"我发起的"标签页
3. 找到需要撤回的审批记录
4. 如果满足撤回条件，会显示"撤回"按钮（黄色）
5. 点击"撤回"按钮，确认后即可撤回

#### API 接口
```http
POST /api/approvals/{instanceId}/withdraw
Headers:
  Authorization: Bearer {token}
  X-Account-Set-Id: {accountSetId}
```

**成功响应**：
```json
{
  "success": true,
  "message": "审批已撤回",
  "data": { ... }
}
```

**失败响应示例**：
```json
{
  "success": false,
  "message": "第二个审批人已经审批，无法撤回"
}
```

### 撤回后的状态变化
- 审批实例状态：`pending` → `withdrawn`
- 所有待审批记录：`pending/waiting` → `withdrawn`
- 业务数据状态：根据业务类型更新为 `withdrawn`

## 五、注意事项

1. **撤回是不可逆的**：撤回后审批流程终止，无法恢复
2. **业务数据状态**：撤回后业务数据状态会更新为 `withdrawn`
3. **权限控制**：只有发起人才能撤回，其他人无权限
4. **时机限制**：只能在第二个审批人审批前撤回
5. **日志记录**：所有撤回操作都会记录在日志中

## 六、常见问题

### Q1: 为什么我看不到撤回按钮？
A: 可能的原因：
- 不在"我发起的"标签页
- 审批状态不是"待审批"
- 第二个审批人已经审批
- 流程已进入第三步或更后

### Q2: 撤回后可以重新发起吗？
A: 可以。撤回后可以重新发起新的审批流程。

### Q3: 第一个审批人自动跳过后，能看到审批记录吗？
A: 可以。在审批流程中会显示第一步已自动通过，状态为"已审批"。

### Q4: 撤回功能适用于所有业务类型吗？
A: 是的。撤回功能适用于所有审批流程（合同、工资、报销、发票等）。
