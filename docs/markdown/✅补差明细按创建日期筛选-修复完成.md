# ✅ 补差明细按创建日期筛选 - 修复完成

## 🐛 问题描述

在参保增减页面选择月份筛选时：
- ✅ 普通参保明细可以正常筛选
- ❌ 社保补交明细和公积金补交明细无法筛选

## 🔍 问题分析

### 用户需求

根据补差记录的**创建日期月份**筛选，而不是根据补差的时间段筛选。

### 示例说明

```
补差记录:
  ID: 27
  类型: 社保补差
  补差期间: 2025-01 ~ 2025-09  ← 这是补差涉及的月份范围
  创建时间: 2025-10-25         ← 这是记录创建的时间
  创建月份: 2025-10             ← 按这个筛选！
```

**期望行为**：
- 选择 **2025-10** 月 → ✅ 显示（因为创建于10月）
- 选择 **2025-09** 月 → ❌ 不显示（虽然在补差期间内，但不是创建月份）

## ✅ 修复方案

### 1. 后端修改 - 社保补差筛选

```php
// ❌ 修复前 - 按补差时间段筛选
if ($month) {
    $query->where(function($q) use ($month) {
        $q->where('compensation_start_month', '<=', $month)
          ->where('compensation_end_month', '>=', $month);
    });
}

// ✅ 修复后 - 按创建日期筛选
if ($month) {
    $query->whereYear('created_at', '=', substr($month, 0, 4))
          ->whereMonth('created_at', '=', substr($month, 5, 2));
}
```

**SQL 逻辑**：
```sql
-- 月份参数: '2025-10'
WHERE YEAR(created_at) = 2025
  AND MONTH(created_at) = 10
```

## 🔄 与工资表的关系

⚠️ **注意**：工资表的补差查询逻辑不同！

- **参保增减补差明细**：按**创建日期**筛选（本次修复）
- **工资表补差数据**：按**补差时间段**筛选（保持原样）

### 工资表的逻辑

```php
// SalaryController.php - calculateCompensation
$compensations = InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->where(function($q) use ($month) {
        $q->where('compensation_start_month', '<=', $month)
          ->where('compensation_end_month', '>=', $month);
    })
    ->get();
```

**工资表场景**：
- 生成 2025-09 月工资表
- 需要查找在 2025-09 有效的补差记录
- 所以按补差时间段筛选是正确的

**参保增减场景**：
- 查看 2025-10 月创建的补差记录
- 需要按创建日期筛选
- 所以按 `created_at` 筛选是正确的
