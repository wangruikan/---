#### 1.2 添加发起付款申请接口

**文件：** `app/Http/Controllers/SalaryController.php`

**新增方法：**
```php
/**
 * 发起工资付款申请
 */
public function createPaymentRequest(Request $request)
{
    $validator = Validator::make($request->all(), [
        'project_id' => 'required|exists:projects,id',
        'month' => 'required|date_format:Y-m',
    ]);

    if ($validator->fails()) {
        return response()->json([
            'success' => false,
            'message' => '验证失败',
            'errors' => $validator->errors()
        ], 422);
    }

    $currentAccountSetId = $request->header('X-Account-Set-Id') ?: $request->input('current_account_set_id');

    DB::beginTransaction();
    try {
        // 1. 查找工资表审批记录
        $salaryApproval = SalaryApproval::where('project_id', $request->project_id)
                                       ->where('month', $request->month)
                                       ->where('account_set_id', $currentAccountSetId)
                                       ->where('status', 'approved')
                                       ->first();

        if (!$salaryApproval) {
            return response()->json([
                'success' => false,
                'message' => '未找到已审批的工资表'
            ], 404);
        }

        // 2. 检查是否已经发起过付款申请
        $existingRequest = PaymentRequest::where('salary_approval_id', $salaryApproval->id)->first();
        if ($existingRequest) {
            return response()->json([
                'success' => false,
                'message' => '该工资表已发起过付款申请'
            ], 422);
        }

        // 3. 计算付款金额（所有员工的实发工资总和）
        $totalAmount = Salary::where('project_id', $request->project_id)
                            ->where('month', $request->month)
                            ->where('account_set_id', $currentAccountSetId)
                            ->sum('net_salary');

        // 4. 创建付款申请
        $paymentRequest = PaymentRequest::create([
            'payment_type' => 'salary',
            'account_set_id' => $currentAccountSetId,
            'salary_approval_id' => $salaryApproval->id,
            'amount' => $totalAmount,
            'status' => 'pending',
            'submitted_by' => $request->user()->id,
            'submitted_at' => now(),
            'remarks' => '工资付款申请 - ' . $salaryApproval->month,
        ]);

        DB::commit();

        \Log::info('工资付款申请已创建', [
            'payment_request_id' => $paymentRequest->id,
            'salary_approval_id' => $salaryApproval->id,
            'amount' => $totalAmount
        ]);

        return response()->json([
            'success' => true,
            'message' => '付款申请已发起',
            'data' => $paymentRequest
        ]);
    } catch (\Exception $e) {
        DB::rollBack();
        \Log::error('创建工资付款申请失败', ['error' => $e->getMessage()]);
        
        return response()->json([
            'success' => false,
            'message' => '发起付款申请失败：' . $e->getMessage()
        ], 500);
    }
}
```

**路由：**
```php
// routes/api.php
Route::prefix('salaries')->group(function () {
    // ... 其他路由 ...
    Route::post('/create-payment-request', [SalaryController::class, 'createPaymentRequest']);
});
```

### 3. 数据修复SQL

#### 3.1 更新已审批的工资表状态

**文件：** `update_approved_salary_status.sql`

```sql
-- 查看需要更新的记录
SELECT 
    sa.id AS salary_approval_id,
    sa.status AS current_status,
    ai.id AS instance_id,
    ai.status AS instance_status,
    ai.completed_at
FROM salary_approvals sa
JOIN approval_instances ai ON sa.approval_instance_id = ai.id
WHERE ai.business_type = '工资表审批'
AND ai.status = 'approved'
AND sa.status != 'approved';

-- 更新工资表审批状态为已批准
UPDATE salary_approvals sa
JOIN approval_instances ai ON sa.approval_instance_id = ai.id
SET 
    sa.status = 'approved',
    sa.approved_at = ai.completed_at
WHERE ai.business_type = '工资表审批'
AND ai.status = 'approved'
AND sa.status != 'approved';

-- 验证更新结果
SELECT 
    sa.id,
    sa.project_id,
    sa.month,
    sa.status,
    sa.approved_at,
    ai.status AS instance_status
FROM salary_approvals sa
JOIN approval_instances ai ON sa.approval_instance_id = ai.id
WHERE ai.business_type = '工资表审批'
ORDER BY sa.id DESC;
```

### 场景：从工资表生成到付款完成

```
1. 生成工资表
   └─ 工资管理 → 生成工资表 → 选择项目和月份

3. 审批流程（自动跳过发起人）
   └─ 第二步审批人审批 → 第三步审批人审批

5. 发起付款申请⭐
   └─ 工资管理 → 点击"发起付款" → 确认

6. 付款审批
   └─ 付款申请 → 审批人审批 → 财务付款

7. 标记为已付款
   └─ 付款申请 → 标记已付款 → 完成
```

## 🔍 常见问题

### Q1: 工资表审批通过了，但还是显示"待审批"？

**A:** 执行SQL更新脚本：
```sql
UPDATE salary_approvals sa
JOIN approval_instances ai ON sa.approval_instance_id = ai.id
SET 
    sa.status = 'approved',
    sa.approved_at = ai.completed_at
WHERE ai.business_type = '工资表审批'
AND ai.status = 'approved'
AND sa.status != 'approved';
```

### Q2: 点击"发起付款"没有反应？

**A:** 检查：
1. 浏览器控制台是否有错误
2. Network 标签中 API 请求是否成功
3. 后端日志 `storage/logs/laravel.log`

### Q3: 提示"未找到已审批的工资表"？

**A:** 检查：
1. 工资表审批状态是否为 'approved'
2. project_id 和 month 是否正确
3. account_set_id 是否匹配

### Q4: 提示"该工资表已发起过付款申请"？

**A:** 这是正常的，一个工资表只能发起一次付款申请。如需重新发起，先删除已有的付款申请记录。
