# 付款类型筛选功能 - 完成说明

## ✅ 功能已完成！

在付款申请列表页面，添加了"付款类型"筛选功能，可以快速筛选工资付款或保险付款申请。

## 🎯 实现的功能

### 1. 付款类型筛选框

✅ **位置**：筛选区域第一个选项

✅ **选项**：
- **全部类型**（默认）：显示所有付款申请
- **工资付款**：只显示工资付款申请
- **保险付款**：只显示保险付款申请

✅ **功能**：
- 可清空重置
- 与月份、状态筛选联合使用
- 实时查询后端数据

### 2. 智能查询优化

后端会根据选择的付款类型，智能查询对应的数据表：

**选择"工资付款"时**：
- ✅ 只查询 `payment_requests` 表中 `payment_type = 'salary'` 的记录
- ✅ 跳过旧的 `payment_applications` 表查询
- ✅ 提升查询效率

**选择"保险付款"时**：
- ✅ 查询 `payment_applications` 表（旧系统）
- ✅ 查询 `payment_requests` 表中 `payment_type = 'insurance'` 的记录（新系统）
- ✅ 合并显示

**选择"全部类型"时**：
- ✅ 查询所有表的所有类型
- ✅ 合并显示所有付款申请

## 🔄 筛选逻辑

### 前端逻辑
```javascript
// 用户选择付款类型后，点击查询按钮
handleSearch() → loadApplicationList()

// 发送请求到后端
{
  current_account_set_id: 1,
  payment_type: 'salary',  // 或 'insurance' 或空
  month: '2025-10',        // 可选
  status: 'pending',       // 可选
  page: 1,
  per_page: 15
}
```

### 后端逻辑
```php
// 1. 判断付款类型
if (payment_type == 'salary') {
    // 只查询新系统的工资付款
    查询 payment_requests (payment_type = 'salary')
    
} elseif (payment_type == 'insurance') {
    // 查询新旧系统的保险付款
    查询 payment_applications (所有)
    查询 payment_requests (payment_type = 'insurance')
    合并结果
    
} else {
    // 查询所有
    查询 payment_applications (所有)
    查询 payment_requests (所有)
    合并结果
}

// 2. 按创建时间排序
// 3. 分页返回
```
