# ✅ 动态签名位置功能完成

## 问题背景

之前使用固定坐标（X=95, Y=153）来放置签名，但这样有个问题：

**如果PDF内容增多，"本人签名："文字的位置会下移，而签名图片位置是固定的，就会错位。**

## 解决方案

### 新方案：动态计算签名位置

**核心思路：** 签名始终距离页面底部固定距离，无论内容多少都不会错位。

```
┌─────────────────┐
│                 │
│   PDF 内容      │  ← 内容可多可少
│   （可变）      │
│                 │
├─────────────────┤
│ 本人签名：____  │  ← 固定距底部60点
└─────────────────┘
      ↑
  签名始终在这里
```

## 实现方式

### 1. 后端返回签名位置信息

**文件：** `app/Http/Controllers/EmployeeController.php`

**返回数据：**
```json
{
  "pdf_base64": "...",
  "signature_base64": "...",
  "signature_position": {
    "x": 95,
    "y": 60,
    "width": 33,
    "height": 13,
    "from_bottom": true  ← 关键：标记Y是从底部计算
  }
}
```

**签名位置参数：**
- `x`: 95 - 从左边距离
- `y`: 60 - **距离页面底部的距离**
- `width`: 33 - 签名宽度
- `height`: 13 - 签名高度
- `from_bottom`: true - Y坐标从底部计算

### 2. 前端动态计算Y坐标

**文件：** `src/views/Employees/index.vue`

**计算逻辑：**
```javascript
// 获取PDF页面高度
const { width, height } = lastPage.getSize()

// 如果from_bottom为true，从页面高度减去Y值
const sigY = position.from_bottom 
  ? height - position.y   // 从底部计算：842 - 60 = 782
  : position.y            // 直接使用Y值
```

## 工作流程

```
后端生成PDF
    ↓
返回签名位置：Y=60（距底部）
    ↓
前端获取PDF高度：height=842点
    ↓
计算实际Y坐标：842 - 60 = 782
    ↓
在(95, 782)位置绘制签名
    ↓
✅ 签名永远在底部固定位置
```

## 如何调整签名位置

### 向右/左移动

修改后端 `EmployeeController.php` 中的 `x` 值：

```php
$signaturePosition = [
    'x' => 100,  // 向右移动（从95改为100）
    // 或
    'x' => 90,   // 向左移动（从95改为90）
    'y' => 60,
    // ...
];
```

### 向上/下移动

修改后端 `EmployeeController.php` 中的 `y` 值：

```php
$signaturePosition = [
    'x' => 95,
    'y' => 70,   // 距底部更远（向上移动）
    // 或
    'y' => 50,   // 距底部更近（向下移动）
    // ...
];
```

### 调整大小

修改后端的 `width` 和 `height`：

```php
$signaturePosition = [
    'x' => 95,
    'y' => 60,
    'width' => 40,   // 增大宽度
    'height' => 16,  // 增大高度
    // ...
];
```

## 兼容性说明

- ✅ **向后兼容** - 如果后端没返回 `signature_position`，前端使用默认值
- ✅ **容错处理** - 如果 `from_bottom` 为 false 或 undefined，直接使用Y值
- ✅ **统一标准** - 测试按钮和批量导出使用相同逻辑
