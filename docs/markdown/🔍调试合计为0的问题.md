# 🔍 调试合计为 0 的问题

## 📋 操作步骤

### 1️⃣ 删除旧数据

```sql
DELETE FROM salaries WHERE month = '2025-10';
```

### 2️⃣ 重新生成工资表

在前端点击"生成工资表"按钮

### 3️⃣ 查看日志

打开 `storage/logs/laravel.log`，搜索：

```
生成工资表 - 保险合计
```

应该看到类似这样的日志：

```json
{
  "employee_id": 123,
  "employee_name": "wrk",
  "company_social_security": 6051.15,
  "company_housing_fund": 1500.0,
  "company_large_medical": 900.0,
  "company_insurance_total": 8451.15,     // ✅ 应该有值
  "personal_social_security": 6054.05,
  "personal_housing_fund": 1500.0,
  "personal_large_medical": 1650.0,
  "personal_insurance_total": 9204.05     // ✅ 应该有值
}
```

### 4️⃣ 如果合计为 0

**可能原因：**

1. **`company_social_security` 为 0**
   - 检查：是否有社保数据
   - 检查：`insurance_personnel` 表中是否有该员工的记录
   - 检查：`social_security_types` JSON 是否有数据

2. **`company_housing_fund` 为 0**
   - 检查：`housing_fund_params` JSON 是否有数据
   - 检查：`employee_housing_fund_base` 是否有值

3. **`company_large_medical` 为 0**
   - 检查：`large_medical_insurance_config` JSON 是否有数据
   - 检查：`large_medical_insurance_enabled` 是否为 true

### 5️⃣ 检查 insurance_personnel 表

```sql
SELECT 
    id,
    employee_id,
    employee_name,
    employee_social_security_base,
    employee_housing_fund_base,
    employee_large_medical_base,
    large_medical_insurance_enabled,
    social_security_types,
    housing_fund_params,
    large_medical_insurance_config
FROM insurance_personnel
WHERE employee_id = (SELECT id FROM employees WHERE name = 'wrk')
AND status = 'active';
```

**应该看到：**
- `employee_social_security_base`: 有值（如 1009.00）
- `employee_housing_fund_base`: 有值（如 4000.00）
- `large_medical_insurance_enabled`: 1
- `social_security_types`: JSON 字符串
- `housing_fund_params`: JSON 字符串
- `large_medical_insurance_config`: JSON 字符串

### 6️⃣ 如果 insurance_personnel 表没有数据

说明该员工还没有参保记录！

**需要先在"参保增减管理"中为该员工办理参保：**

1. 进入"参保增减管理"
2. 点击"位生成员工记录"或添加员工
3. 填写参保信息
4. 点击"确认处理"
5. 然后再回到工资管理生成工资表

## 🔑 关键检查点

| 检查项 | 预期值 | 如果为空/0 |
|--------|--------|-----------|
| `insurance_personnel` 记录存在 | 有记录 | 先办理参保 |
| `employee_social_security_base` | > 0 | 设置社保基数 |
| `employee_housing_fund_base` | > 0 | 设置公积金基数 |
| `social_security_types` | JSON数组 | 配置社保险种 |
| `housing_fund_params` | JSON对象 | 配置公积金比例 |
| `large_medical_insurance_config` | JSON对象 | 配置大额医疗 |
| `large_medical_insurance_enabled` | 1 | 启用大额医疗 |

## 📊 正确的数据流程

```
1. 参保增减管理
   ↓ 添加员工 + 配置保险
   
2. insurance_personnel 表
   ↓ 存储参保信息
   
3. 生成工资表
   ↓ 从 insurance_personnel 读取
   
4. calculateMonthlyInsurance()
   ↓ 计算各项保险金额
   
5. Salary::create()
   ↓ 存入 salaries 表
   
6. 前端显示
   ↓ 从 salaries 表读取并显示
```

## ⚠️ 常见错误

### 错误1：直接生成工资表，没有先参保

**现象：** 合计为 0，所有保险金额都是 0

**解决：** 先在"参保增减管理"中办理参保

### 错误2：参保信息不完整

**现象：** 某些保险有值，某些为 0

**解决：** 检查 `insurance_personnel` 表，补全缺失的基数和配置

### 错误3：使用了旧数据

**现象：** 修改了代码但合计还是 0

**解决：** 删除 `salaries` 表的旧数据，重新生成
