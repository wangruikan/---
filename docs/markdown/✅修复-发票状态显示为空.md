# ✅ 修复 - 发票状态显示为空

## 🔍 问题根因

状态列显示为空的**真正原因**是：

模型中定义了 `getStatusTextAttribute()` 访问器方法，但**没有添加到 `$appends` 属性**中，导致序列化时不会自动包含 `status_text` 字段。

## 📋 问题分析

### Laravel 模型序列化规则

```php
class InvoiceApplication extends Model
{
    // 定义了访问器
    public function getStatusTextAttribute() {
        return '正常';
    }
    
    // ❌ 但是没有添加到 $appends
    // 结果：API返回的JSON中不包含 status_text 字段
    
    // ✅ 需要添加 $appends
    protected $appends = ['status_text'];
    // 结果：API返回的JSON中会自动包含 status_text 字段
}
```

## ✅ 解决方案

### 文件：`app/Models/InvoiceApplication.php`

添加 `$appends` 属性：

```php
protected $casts = [
    'year' => 'integer',
    'month' => 'integer',
    'attachments' => 'array',
    'submitted_at' => 'datetime',
    'created_at' => 'datetime',
    'updated_at' => 'datetime',
];

// ✅ 添加这个属性
protected $appends = [
    'status_text',  // 将 getStatusTextAttribute() 的结果包含在序列化输出中
];
```

## 🎯 修复后的效果

### API 返回的数据格式

**修复前**：
```json
{
  "id": 1,
  "status": "normal",
  "application_no": "INV202511035281"
  // ❌ 缺少 status_text 字段
}
```

**修复后**：
```json
{
  "id": 1,
  "status": "normal",
  "status_text": "正常",  // ✅ 自动添加
  "application_no": "INV202511035281"
}
```

### 前端显示

```vue
<el-tag :type="getStatusType(row.status)">
  {{ row.status_text }}  <!-- ✅ 现在有值了 -->
</el-tag>
```

## 🔧 如果还是不显示

### 检查1：确认数据库中status有值

执行SQL：
```sql
SELECT id, application_no, status, created_at 
FROM invoice_applications 
ORDER BY created_at DESC 
LIMIT 5;
```

如果 `status` 字段为 `NULL`：
```sql
-- 修复现有数据
UPDATE invoice_applications 
SET status = 'normal' 
WHERE status IS NULL;
```

### 检查2：确认前端有正确的映射

```javascript
// src/views/InvoiceApplications/index.vue
const getStatusType = (status) => {
  const typeMap = {
    draft: '',
    normal: 'primary',    // ← 确认存在
    pending: 'warning',
    approved: 'success',
    rejected: 'danger',
    red_flushed: 'danger'
  }
  return typeMap[status] || ''
}
```
