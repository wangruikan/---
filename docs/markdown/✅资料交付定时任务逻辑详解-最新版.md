# 资料交付定时任务逻辑详解（最新版）

## 📋 定时任务总览

资料交付管理有 **2 个定时任务**：

1. **生成交付记录** (`delivery:generate`) - 每月1日早上8点
2. **检查未交付记录** (`delivery:check-pending`) - 每月最后一天晚上8点

---

## 1️⃣ 生成交付记录任务 (`delivery:generate`)

### 🕐 执行时间
**每月1日早上8点**

### 🎯 主要功能
在每月1日自动生成新的资料交付记录。

---

### 📊 执行逻辑

#### A. 月度交付项目（每月生成）

**执行频率**：每月1日都执行

```
步骤1：查询配置
├─ 查询所有 delivery_cycle = 'monthly' 且 is_active = 1 的配置
└─ 获取所有"按月交付"的项目

步骤2：生成交付记录
├─ 为每个配置生成当月的交付记录
├─ delivery_period = '2025-10'（当前月份）
├─ status = 'pending'（待交付）
├─ handler_id = 经办人ID
├─ config_id = 关联的配置ID
└─ required_documents = 配置中的资料清单

步骤3：防重复检查
├─ 生成前检查：该项目的当月记录是否已存在
├─ 如果已存在 → 跳过（防止重复生成）
└─ 如果不存在 → 创建新记录

步骤4：发送提醒
├─ 向经办人发送"新周期提醒"
├─ 提醒类型：reminder_type = 'new_period'
└─ 记录到 document_delivery_reminders 表
```

**示例**：
```
2025年10月1日执行：
- 项目A：生成 2025-10 月度交付记录
- 项目B：生成 2025-10 月度交付记录
```

---

#### B. 季度交付项目（固定日期生成）

**⚠️ 重要变更**：季度交付采用固定日期生成，不再计算3个月周期。

**执行频率**：仅在以下日期执行
- ✅ **1月1日** → 生成 Q1（第一季度）
- ✅ **4月1日** → 生成 Q2（第二季度）
- ✅ **7月1日** → 生成 Q3（第三季度）
- ✅ **10月1日** → 生成 Q4（第四季度）
- ❌ 其他日期（如2月1日、3月1日等）→ 跳过季度生成

**执行逻辑**：
```
步骤1：检查日期
├─ 判断今天是否是 1/1、4/1、7/1、10/1
├─ 如果不是 → 跳过季度交付生成，直接返回
└─ 如果是 → 继续执行

步骤2：查询配置
├─ 查询所有 delivery_cycle = 'quarterly' 且 is_active = 1 的配置
└─ 获取所有"按季度交付"的项目

步骤3：生成交付记录
├─ 为每个配置生成当季的交付记录
├─ delivery_period = '2025-Q1'、'2025-Q2'、'2025-Q3'、'2025-Q4'
├─ status = 'pending'
├─ handler_id = 经办人ID
└─ 其他字段同月度

步骤4：防重复检查
├─ 生成前检查：该项目的当季记录是否已存在
├─ 如果已存在 → 跳过
└─ 如果不存在 → 创建新记录

步骤5：发送提醒
└─ 向经办人发送"新周期提醒"
```

**关键代码**：
```php
// 检查是否是季度第一天
private function isQuarterFirstDay(Carbon $date)
{
    $month = $date->month;
    $day = $date->day;
    
    // 只在 1/1、4/1、7/1、10/1 返回 true
    return $day === 1 && in_array($month, [1, 4, 7, 10]);
}
```

**示例场景**：
```
日期         月度交付    季度交付    说明
-----------------------------------------------------------
1月1日       生成✅      生成✅      生成 2025-01 + 2025-Q1
2月1日       生成✅      跳过❌      只生成 2025-02
3月1日       生成✅      跳过❌      只生成 2025-03
4月1日       生成✅      生成✅      生成 2025-04 + 2025-Q2
5月1日       生成✅      跳过❌      只生成 2025-05
...
10月1日      生成✅      生成✅      生成 2025-10 + 2025-Q4
11月1日      生成✅      跳过❌      只生成 2025-11
```

## 2️⃣ 检查未交付记录任务 (`delivery:check-pending`)

### 🕐 执行时间
**每月最后一天晚上8点**

### 🎯 主要功能
检查本月/本季度还未交付的记录，发送提醒给经办人。

---

### 📊 执行逻辑

#### A. 月度交付检查

**检查范围**：当月的月度交付记录

```
步骤1：计算当月期间
├─ current_period = '2025-10'（当前月份）

步骤2：查询未交付记录
├─ 查询条件：
│   ├─ delivery_cycle = 'monthly'
│   ├─ delivery_period = '2025-10'
│   └─ status = 'pending'
└─ 找到所有本月还未提交的交付记录

步骤3：发送提醒
├─ 遍历每条未交付记录
├─ 向经办人发送"未交付提醒"
├─ 提醒类型：reminder_type = 'not_submitted'
└─ 记录到 document_delivery_reminders 表

步骤4：记录日志
└─ 记录未交付数量和详细信息
```

**示例**：
```
10月31日晚上8点执行：

查询结果：
- 项目A的 2025-10 记录：status = 'submitted' → 已提交，跳过
- 项目B的 2025-10 记录：status = 'pending' → 未提交，发送提醒
- 项目C的 2025-10 记录：status = 'pending' → 未提交，发送提醒

提醒内容：
"【资料交付提醒】项目B 2025年10月的资料尚未交付，请尽快提交。"
```

---

#### B. 季度交付检查

**⚠️ 重要变更**：季度交付改为每月底检查本季度的记录。

**检查范围**：当季的季度交付记录（每月都检查）

```
步骤1：计算当季期间
├─ 根据当前月份计算所在季度
├─ 1-3月 → current_period = '2025-Q1'
├─ 4-6月 → current_period = '2025-Q2'
├─ 7-9月 → current_period = '2025-Q3'
└─ 10-12月 → current_period = '2025-Q4'

步骤2：查询未交付记录
├─ 查询条件：
│   ├─ delivery_cycle = 'quarterly'
│   ├─ delivery_period = '2025-Q4'（当前季度）
│   └─ status = 'pending'
└─ 找到本季度所有还未提交的交付记录

步骤3：发送提醒
├─ 向经办人发送"未交付提醒"
├─ 提醒类型：reminder_type = 'not_submitted'
└─ 每月都会提醒，直到交付完成

步骤4：记录日志
└─ 记录本季度未交付数量
```

**示例场景**：
```
场景1：第四季度（10-12月）

10月1日：生成 2025-Q4 交付记录（status = pending）
10月31日：检查 2025-Q4 记录，status = pending → 发送提醒
11月30日：检查 2025-Q4 记录，status = pending → 再次提醒
12月15日：经办人提交资料，status = submitted
12月31日：检查 2025-Q4 记录，status = submitted → 已提交，不提醒

场景2：第一季度（1-3月）

1月1日：生成 2025-Q1 交付记录
1月31日：检查 2025-Q1，未提交 → 提醒
2月28日：检查 2025-Q1，未提交 → 再次提醒
3月1日：经办人提交
3月31日：检查 2025-Q1，已提交 → 不提醒
```

**提醒频率**：
- 📅 **月度交付**：每月底提醒1次
- 📅 **季度交付**：每月底提醒1次（直到提交）
  - Q1（1-3月）：最多提醒3次（1月底、2月底、3月底）
  - Q2（4-6月）：最多提醒3次（4月底、5月底、6月底）
  - Q3（7-9月）：最多提醒3次（7月底、8月底、9月底）
  - Q4（10-12月）：最多提醒3次（10月底、11月底、12月底）

## 🔧 定时任务配置

### Linux 服务器配置（Crontab）

```bash
# 编辑 crontab
crontab -e

# 添加以下行
# 每分钟执行 Laravel 调度器
* * * * * cd /path/to/project && php artisan schedule:run >> /dev/null 2>&1
```

### Laravel 任务调度配置

**文件**：`app/Console/Kernel.php`

```php
// 每月1日早上8点生成资料交付记录
$schedule->command('delivery:generate')
         ->monthlyOn(1, '08:00')
         ->timezone('Asia/Shanghai');

// 每月最后一天晚上20点检查未交付记录
$schedule->command('delivery:check-pending')
         ->monthlyOn(31, '20:00')
         ->timezone('Asia/Shanghai')
         ->when(function () {
             // 确保是月末最后一天
             return Carbon::now()->day === Carbon::now()->endOfMonth()->day;
         });
```

## ✅ 关键变更总结

### 与旧版本的区别

| 项目 | 旧逻辑 | 新逻辑 |
|------|--------|--------|
| 季度生成时机 | 每月1日检查是否满3个月 | 固定在 1/1、4/1、7/1、10/1 |
| 季度生成判断 | 计算项目创建时间或上次生成时间 | 仅判断当前日期 |
| 季度提醒时机 | 仅在距离生成满3个月时提醒 | 每月底都检查本季度记录 |
| 提醒频率 | 季度提醒1次 | 季度每月提醒（最多3次） |

### 优势

✅ **逻辑更简单**：不需要计算时间差，固定日期更易理解
✅ **更可预测**：明确知道哪天会生成季度记录
✅ **提醒更频繁**：季度交付每月都会提醒，避免遗忘
✅ **维护更方便**：代码逻辑清晰，易于调试
