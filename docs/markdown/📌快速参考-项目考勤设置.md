# 📌 项目考勤设置 - 快速参考

## 🚀 一分钟部署

### 1️⃣ 执行SQL（必须）
在 Navicat 中执行：`⚡⚡立即执行-添加考勤字段.sql`

### 2️⃣ 清除缓存（可选）
```bash
cd backend
php artisan config:clear

```

### 3️⃣ 完成！
刷新浏览器，开始使用新功能。

---

## 💡 功能说明

### 在哪里设置？
**项目管理** → 编辑项目 → **需要考勤表** 开关

### 有什么效果？
| 设置 | 效果 |
|-----|------|
| ✅ 需要考勤 | 必须先审批考勤表才能生成工资表 |
| ❌ 无需考勤 | 可直接生成工资表 |

---

## 📊 工资表生成规则

```
┌─────────────────────────────────────────┐
│         选择工资期间                      │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│    系统检查项目的"需要考勤表"设置         │
└─────────────────────────────────────────┘
                  ↓
        ┌─────────┴─────────┐
        ↓                   ↓
   需要考勤              无需考勤
        ↓                   ↓
   检查考勤审批         直接可生成
        ↓                   
  ┌─────┴─────┐
  ↓           ↓
已审批      未审批
  ↓           ↓
可生成      禁用
```

## 🎯 快速测试

### 测试1：无需考勤
1. 项目设置 → 关闭"需要考勤表"
2. 工资管理 → 生成工资表
3. 应该看到：项目名称 **(无需考勤)** ✅可选

### 测试2：需要考勤
1. 项目设置 → 开启"需要考勤表"
2. 工资管理 → 生成工资表
3. 应该看到：
   - 有考勤审批：项目名称 ✅可选
   - 无考勤审批：项目名称 **(考勤未审批)** ❌禁用

## ❓ 常见问题

**Q: 修改设置后，已有的工资表会变吗？**  
A: 不会。只影响新生成的工资表。

**Q: 默认新项目需要考勤吗？**  
A: 是的。默认需要考勤，保持原有逻辑。

**Q: 无需考勤的项目还能创建考勤表吗？**  
A: 可以。考勤设置只影响工资表生成条件。

**Q: 字段添加失败怎么办？**  
A: 可能已经存在，可以忽略错误。
