# 工资表应发工资导入功能 - 使用说明

## 📋 功能概述

现在工资表的应发工资采用**Excel导入**方式，支持通过身份证号精准匹配更新每个员工的应发工资。

---

## 🚀 使用流程

### 第1步：生成工资表
1. 点击"生成工资表"
2. 选择工资期间和项目
3. 点击"确定生成"
4. **此时应发工资为 ¥0.00**（正常情况）

### 第2步：准备Excel文件

**Excel格式要求：**
```
第1行：任意内容（会被忽略，可以是表名、说明等）
第2行：标题行（必须包含以下列）
第3行起：员工数据
```

**示例：**
| 行号 | 内容 |
|------|------|
| 1 | 2025年11月工资表（此行会被忽略）|
| 2 | 姓名 \| 身份证号码 \| 应发工资 |
| 3 | 张三 \| 110101199001011234 \| 8500.00 |
| 4 | 李四 \| 110101199002022345 \| 7200.00 |

### 第3步：导入Excel
1. 打开工资明细对话框（点击"查看明细"）
2. 点击"📤 导入应发工资Excel"按钮
3. 选择准备好的Excel文件
4. 系统自动匹配更新

---

## 📊 Excel列名支持

### 身份证列（支持以下任意一种）：
- `身份证`
- `身份证号`
- `身份证号码` ✅ （您的表格使用这个）
- `证件号`
- `证件号码`
- `ID` / `id`

### 应发工资列（支持以下任意一种）：
- `应发工资` ✅ （推荐）
- `应发`
- `工资`
- `金额`
- `应发金额`

**注意：** 列名会自动去除空格，不区分全角/半角空格

## ⚙️ 数据库准备

### 更新现有工资表的身份证号

**如果之前生成的工资表身份证号为空，请执行：**

```sql
-- 执行文件：立即执行-更新工资表身份证号.sql
```

这个SQL会：
- ✅ 从员工表读取身份证号和姓名
- ✅ 更新所有工资表中的空值
- ✅ 显示更新统计
- ✅ 检查哪些员工缺少身份证号

## 🔍 导入结果说明

### 导入成功提示：
```
导入完成：成功 15 条，失败 0 条
```

### 如果有失败记录：
```
导入完成：成功 14 条，失败 1 条

失败详情：
第5行：未找到身份证号为 110101199003033456 的员工工资记录
```

**失败原因可能是：**
1. 该身份证号的员工不在本项目
2. 该员工未生成本月工资记录
3. 身份证号输入错误
4. 员工表中身份证号与Excel不一致

## ✅ 验证和检查

### 检查员工是否有身份证号：
```sql
-- 查看所有员工的身份证号
SELECT id, name, id_number 
FROM employees 
ORDER BY id;

-- 查看缺少身份证号的员工
SELECT id, name 
FROM employees 
WHERE id_number IS NULL OR id_number = '';
```

### 检查工资表的身份证号：
```sql
-- 查看工资表数据
SELECT 
    id,
    employee_name,
    id_card,
    gross_salary,
    month,
    project_id
FROM salaries
WHERE month = '2025-11'
ORDER BY id;
```

## 🎯 常见问题

### Q1: 身份证号显示为空？
**解决方法：**
1. 执行SQL脚本：`立即执行-更新工资表身份证号.sql`
2. 或删除工资表重新生成
3. 确保员工表中已录入身份证号

### Q2: 导入失败"未找到身份证号为xxx的员工"？
**检查：**
1. 该员工是否属于当前项目
2. 是否已生成该月工资表
3. Excel中身份证号是否与系统一致（18位数字）

### Q3: Excel导入后应发工资还是0？
**检查：**
1. Excel第2行是否包含"身份证号码"和"应发工资"
2. Excel第3行起是否有数据
3. 查看后端日志确认匹配情况
