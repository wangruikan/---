## 方案1：基数不变，上下限提高（触发补差）

### 场景
```
员工基数：2800（一直没变）

第1次设置上下限：2360 - 26421
第2次修改上下限：3000 - 30000
```

### 计算过程

#### 旧基数（用旧上下限约束）
```
原始基数：2800
旧上下限：2360 - 26421
判断：2360 ≤ 2800 ≤ 26421 ✅
约束后旧基数：2800
```

#### 新基数（用新上下限约束）
```
原始基数：2800（本月没有调基，使用旧基数）
新上下限：3000 - 30000
判断：2800 < 3000 ❌（低于下限）
约束后新基数：3000（使用下限）
```

#### 补差计算
```
基数差额 = 3000 - 2800 = 200


老保险：
- 公司：200 × 100% × 9个月 = 1800
- 个人：200 × 100% × 9个月 = 1800

总计：
- 公司合计：1800
- 个人合计：1800
- 总额：3600
```

### 验证SQL
```sql
-- 手动设置测试数据
UPDATE social_security_regions 
SET old_min_base_amount = 2360,
    old_max_base_amount = 26421,
    new_min_base_amount = 3000,
    new_max_base_amount = 30000
WHERE id = (SELECT social_security_region_id FROM employees WHERE name = 'wrk');

-- 查看补差记录应该是
-- old_base: 2800
-- new_base: 3000
-- company_total: 1800
-- personal_total: 1800
-- total_amount: 3600
```
