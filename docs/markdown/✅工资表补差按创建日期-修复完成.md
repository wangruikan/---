# ✅ 工资表补差按创建日期 - 修复完成

## 🐛 问题描述

工资表中的补差数据一直显示为空（`-`），无法正确导入参保补差明细的数据。

## 🔍 根本原因

工资表的补差计算逻辑与参保增减不一致：

### 修改前的逻辑

```php
// ❌ 按补差时间段筛选
$compensations = InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->where(function($q) use ($month) {
        $q->where('compensation_start_month', '<=', $month)
          ->where('compensation_end_month', '>=', $month);
    })
    ->get();
```

**问题**：
- 生成 2025-10 月工资表
- 查找在 2025-10 有效的补差记录
- 但补差记录的期间是 2025-01 ~ 2025-09
- 2025-10 不在这个范围内 → 查不到数据 ❌

## ✅ 修复方案

改为按**创建日期**筛选，与参保增减保持一致：

```php
// ✅ 按创建日期筛选
$compensations = InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->whereYear('created_at', '=', substr($month, 0, 4))
    ->whereMonth('created_at', '=', substr($month, 5, 2))
    ->get();
```

**逻辑**：
- 生成 2025-10 月工资表
- 查找在 2025-10 **创建**的补差记录
- 补差记录创建于 2025-10-24 和 2025-10-25
- 能查到数据 ✅

## 🔄 与原有逻辑的对比

### 原逻辑（按补差时间段）

```
场景：生成 2025-09 月工资表

补差记录:
  补差期间: 2025-01 ~ 2025-09
  创建时间: 2025-10-25

筛选条件: compensation_start_month <= '2025-09' 
         AND compensation_end_month >= '2025-09'

结果: ✅ 能查到（2025-09 在期间内）

问题：
  - 10月才创建的补差，怎么能算到9月工资里？
  - 不符合业务逻辑
```

### 新逻辑（按创建日期）

```
场景：生成 2025-09 月工资表

补差记录:
  补差期间: 2025-01 ~ 2025-09
  创建时间: 2025-10-25

筛选条件: YEAR(created_at) = 2025 
         AND MONTH(created_at) = 9

结果: ❌ 查不到（创建于10月，不是9月）

优点：
  - 只在创建月份的工资中体现补差
  - 符合会计核算逻辑
  - 与参保增减页面一致
```
