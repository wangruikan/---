# 月末合同检查功能完成

## 📋 功能概述

实现了完整的月末合同检查和提醒升级系统，包括：

1. **月末检查** - 每月最后一天检查员工合同情况
2. **提醒记录** - 生成合同相关提醒记录
3. **15号复查** - 每月15号复查提醒记录
4. **升级考核** - 未解决的提醒升级为考核记录

## 🗂️ 数据库结构

### contract_reminders 表
```sql
- id: 主键
- account_set_id: 账套ID
- employee_id: 员工ID
- reminder_type: 提醒类型（labor_contract/termination_agreement/retirement_agreement）
- employee_name: 员工姓名
- contract_start_date: 合同开始日期
- contract_end_date: 合同结束日期
- termination_date: 离职日期
- retirement_date: 退休日期
- status: 状态（pending/resolved/escalated）
- description: 描述
- handler_id: 处理人ID
- handler_name: 处理人姓名
- reminder_date: 提醒日期
- escalation_date: 升级日期
- is_escalated: 是否已升级
- assessment_record_id: 关联考核记录ID
- remark: 备注
```

## 🔧 核心功能

### 1. 月末合同检查 (`CheckMonthlyContracts`)

**检查逻辑：**

#### 劳动合同检查
- 当前时间在合同有效期内（contract_start_date 到 contract_end_date）
- 检查是否有劳动合同（contract_type = 'labor'）
- 合同创建时间必须在合同有效期内
- 如果没有，生成提醒记录

#### 解除协议检查
- 员工已离职（有 termination_date）
- 当前时间大于合同结束时间
- 检查是否有解除协议合同（contract_type = 'termination'）
- 合同创建时间必须在合同结束日期后
- 如果没有，生成提醒记录

#### 退休解除协议检查
- 员工已退休（is_retired = true）
- 当前时间大于合同结束时间
- 检查是否有退休解除协议合同（contract_type = 'retirement'）
- 合同创建时间必须在合同结束日期后
- 如果没有，生成提醒记录

### 2. 15号复查升级 (`EscalateContractReminders`)

**处理逻辑：**
- 检查上月末生成的待处理提醒记录
- 重新验证是否已上传相关合同
- 已解决：标记为 resolved
- 未解决：升级为考核记录，标记为 escalated

## ⏰ 定时任务调度

```php
// 每月最后一天晚上22点检查员工合同签订情况
$schedule->command('contract:check-monthly')
         ->monthlyOn(31, '22:00')
         ->timezone('Asia/Shanghai')
         ->when(function () {
             return Carbon::now()->day === Carbon::now()->endOfMonth()->day;
         });

// 每月15号早上9点检查合同提醒记录，未解决的升级为考核
$schedule->command('contract:escalate-reminders')
         ->monthlyOn(15, '09:00')
         ->timezone('Asia/Shanghai');
```

## 🎯 业务流程

### 月末流程（每月最后一天）
1. 系统自动执行合同检查
2. 检查所有员工的合同情况
3. 生成提醒记录（不计入考核）
4. 通知业务人员处理

### 15号流程（每月15号）
1. 系统检查上月末的提醒记录
2. 验证是否已上传相关合同
3. 已解决：标记为已解决
4. 未解决：升级为考核记录

## 🧪 测试命令

### 手动执行月末检查
```bash
# 检查所有账套
php artisan contract:check-monthly

# 检查指定账套
php artisan contract:check-monthly --account-set-id=1

# 检查指定日期
php artisan contract:check-monthly --date=2024-11-30
```

### 手动执行15号复查
```bash
# 检查所有账套
php artisan contract:escalate-reminders

# 检查指定账套
php artisan contract:escalate-reminders --account-set-id=1

# 检查指定日期
php artisan contract:escalate-reminders --date=2024-11-15
```

## ⚠️ 注意事项

1. **时间判断严格** - 严格按照合同有效期和创建时间判断
2. **避免重复** - 同一员工同一类型同一日期不会重复创建提醒
3. **状态管理** - 提醒记录有完整的状态流转
4. **考核升级** - 只有未解决的提醒才会升级为考核
5. **工作日计算** - 考核截止日期按工作日计算
