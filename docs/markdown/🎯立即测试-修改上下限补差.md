$employee = App\Models\Employee::where('name', 'wrk')->first();

### 测试场景：第2次修改上下限（生成补差）

```php
// 第2次修改上下限
$result = $service->updateLimits($region, 3000, 30000);

echo "\n结果: " . PHP_EOL;
print_r($result);

// 如果需要补差，调用补差服务
if ($result['should_compensate']) {
    $compensationService = app(App\Services\BaseLimitCompensationService::class);
    $compResult = $compensationService->calculateSocialSecurityCompensation(
        $region->id,
        $result['old_min'],
        $result['old_max'],
        $result['new_min'],
        $result['new_max']
    );
    echo "\n补差结果: " . PHP_EOL;
    print_r($compResult);
}

// 查看补差记录
$comps = App\Models\InsuranceCompensationRecord::where('employee_id', $employee->id)
    ->where('compensation_type', 'social_security')
    ->get();

foreach ($comps as $comp) {
    echo "\n========== 补差记录 ==========" . PHP_EOL;
    echo "期间: " . $comp->compensation_start_month . " ~ " . $comp->compensation_end_month . PHP_EOL;
    echo "月数: " . $comp->compensation_months . " 个月" . PHP_EOL;
    echo "旧基数: " . $comp->old_base . PHP_EOL;
    echo "新基数: " . $comp->new_base . PHP_EOL;
    echo "总额: " . $comp->total_amount . PHP_EOL;
}
```

**预期结果**：
```
should_compensate => true
old上下限: 2360 - 26421
new上下限: 3000 - 30000
补差记录数: 1
期间: 2025-01 ~ 2025-09
月数: 9 个月
```

## 📊 SQL验证

```sql
-- 查看地区的新旧上下限
SELECT 
    id,
    name,
    old_min_base_amount AS '旧下限',
    old_max_base_amount AS '旧上限',
    old_limits_updated_at AS '旧修改时间',
    new_min_base_amount AS '新下限',
    new_max_base_amount AS '新上限',
    new_limits_updated_at AS '新修改时间'
FROM social_security_regions
WHERE id = (SELECT social_security_region_id FROM employees WHERE name = 'wrk');

-- 查看补差记录
SELECT 
    employee_name AS '员工',
    old_base AS '旧基数',
    new_base AS '新基数',
    compensation_start_month AS '起始',
    compensation_end_month AS '结束',
    compensation_months AS '月数',
    total_amount AS '总额',
    created_at AS '创建时间',
    updated_at AS '更新时间'
FROM insurance_compensation_records
WHERE employee_name = 'wrk'
  AND compensation_type = 'social_security'
ORDER BY updated_at DESC;
```
